Lesson 2 — Composite Node Segment Tree

Max Subarray Sum · sum · pref · suff · best · The Crossing Case

🧩 Problem Definition

You are given an array a of n integers (possibly negative). Support both:

1. Point Set

Replace a[pos] with a new value x.

2. Max Subarray Sum Query

Return max( a[i] + a[i+1] + … + a[j] ) over all l ≤ i ≤ j ≤ r — the best contiguous subarray sum inside [l, r].

This is the dynamic version of the classic Kadane problem. Brute force per query is O(n) with Kadane on a[l..r]; we want O(log n).

💡 Why a Composite Node?

The simplest segment tree stores one value per node — say, the sum. But "max subarray inside a range" cannot be computed from two children's max subarrays alone. Here is why:

The crossing case

Left  child : a = [..., +3]    best inside = 3
Right child : a = [+5, ...]    best inside = 5

Parent's best is NOT max(3, 5) = 5.
It could be 3 + 5 = 8 — a subarray
that crosses the boundary.

What we need to know

To check the crossing case we must know the best suffix of the left and the best prefix of the right. So each node must additionally store those.

And to maintain those when that node becomes a child of someone else, we need its total sum as well.

📐 The Four Values Stored at Each Node

Each node for interval [l, r] stores a struct:

FieldMeaning
sumTotal sum of all values in [l, r].
prefMaximum sum of any non-empty prefix of [l, r].
suffMaximum sum of any non-empty suffix of [l, r].
bestMaximum sum of any non-empty contiguous subarray inside [l, r].

At a leaf with value x, all four are simply x.

⚙️ The Merge Rule — A.merge(B)

Let A be the left child's node, B the right child's. The parent's four values are:

parent.sum

parent.sum = A.sum + B.sum

parent.pref

parent.pref = max(
    A.pref,
    A.sum + B.pref
)

Either the best prefix lives inside A, or it eats all of A and continues into B.

parent.suff

parent.suff = max(
    B.suff,
    B.sum + A.suff
)

Either inside B, or eats all of B going backward into A.

parent.best

parent.best = max(
    A.best,
    B.best,
    A.suff + B.pref
)

Three sources: entirely in A, entirely in B, or crossing the boundary.

🛡️ Invariants

✅ Invariant — Per-node contract

For every node interval [l, r]: sum = sum of values, pref/suff = best non-empty prefix/suffix sum, best = best non-empty contiguous subarray sum inside [l, r].

✅ Invariant — Merge invariant

If A and B satisfy the contract for disjoint adjacent intervals, then merge(A, B) as defined above satisfies the contract for their concatenation.

✅ Invariant — Identity invariant

The identity node satisfies identity ⊕ X = X ⊕ identity = X for every valid X. For max-subarray this requires sum = 0 and pref = suff = best = −∞ (see next section).

⚠️ The Identity is Tricky

For sum, identity is just 0. For max-subarray, identity must be a node such that combining it with any real node returns the real node unchanged. Substituting into the merge rules:

parent.pref = max(IDLEFT.pref, IDLEFT.sum + B.pref)
            = max(IDLEFT.pref, IDLEFT.sum + B.pref)
            should equal B.pref

So we need IDLEFT.pref ≤ B.pref for every legal B (force −∞), and IDLEFT.sum + B.pref = B.pref (force sum = 0). Same analysis for the other three fields.

⚠️ Pitfall — using 0 for pref/suff/best

If pref = 0, an array of all negatives would have best ≥ 0 — wrong, because the best non-empty subarray of [-5, -3] is -3, not 0. The identity must therefore be the "empty" sentinel: sum = 0, but all max-fields = −∞.

🧮 Proof of the Merge

We prove the merge rule by case analysis on the location of the optimal subarray S inside [L, R], where [L, R] is the union of A's interval and B's interval.

  1. S lies entirely inside A. Then by A's best invariant, sum(S) ≤ A.best, and equality is achievable. Contribution to parent.best: A.best.
  2. S lies entirely inside B. Symmetric. Contribution: B.best.
  3. S crosses the boundary. Then S decomposes as a suffix of A followed by a prefix of B. By A's suff invariant, the suffix is at most A.suff; by B's pref invariant, the prefix is at most B.pref. Equality is achievable by taking the optimal suffix of A and the optimal prefix of B. Contribution: A.suff + B.pref.

Taking the max over the three cases yields the merge rule for best. The rules for pref, suff, and sum follow similarly, with prefix/suffix not allowed to be empty (matching the non-empty requirement). ∎

⏱ Complexity

Identical to Lesson 1. Build O(n); point update and range query O(log n). The constant factor is slightly larger because each merge does four max/sum operations instead of one.

💻 Master Template — Max Subarray Sum

Full C++ Template click to expand/collapse 
#include <bits/stdc++.h>
using namespace std;

struct MaxSubarraySegTree {
    struct Node {
        long long sum;
        long long pref;
        long long suff;
        long long best;
    };

    int n = 0;
    int base = 1;
    vector<Node> tree;

    static Node make_node(long long x) {
        return {x, x, x, x};
    }

    static Node identity() {
        const long long NEG = -(1LL << 60);
        return {0, NEG, NEG, NEG};
    }

    static Node merge_nodes(const Node& a, const Node& b) {
        Node res;
        res.sum = a.sum + b.sum;
        res.pref = max(a.pref, a.sum + b.pref);
        res.suff = max(b.suff, b.sum + a.suff);
        res.best = max({a.best, b.best, a.suff + b.pref});
        return res;
    }

    MaxSubarraySegTree() {}
    MaxSubarraySegTree(const vector<long long>& a) { init(a); }

    void init(const vector<long long>& a) {
        n = (int)a.size();
        base = 1;
        while (base < max(1, n)) base <<= 1;
        tree.assign(2 * base, Node());
        build(0, 0, base - 1, a);
    }

    void build(int i, int l, int r, const vector<long long>& a) {
        if (l == r) {
            tree[i] = (l < n) ? make_node(a[l]) : identity();
            return;
        }
        int left = 2*i + 1, right = left + 1;
        int mid = l + (r - l) / 2;
        build(left, l, mid, a);
        build(right, mid + 1, r, a);
        tree[i] = merge_nodes(tree[left], tree[right]);
    }

    void point_set(int pos, long long value) {
        point_set(pos, value, 0, 0, base - 1);
    }
    void point_set(int pos, long long value, int i, int l, int r) {
        if (pos < l || r < pos) return;
        if (l == r) { tree[i] = make_node(value); return; }
        int left = 2*i + 1, right = left + 1;
        int mid = l + (r - l) / 2;
        point_set(pos, value, left, l, mid);
        point_set(pos, value, right, mid + 1, r);
        tree[i] = merge_nodes(tree[left], tree[right]);
    }

    Node query(int ql, int qr) const {
        return query(ql, qr, 0, 0, base - 1);
    }
    Node query(int ql, int qr, int i, int l, int r) const {
        if (qr < l || r < ql) return identity();
        if (ql <= l && r <= qr) return tree[i];
        int left = 2*i + 1, right = left + 1;
        int mid = l + (r - l) / 2;
        Node a = query(ql, qr, left, l, mid);
        Node b = query(ql, qr, right, mid + 1, r);
        return merge_nodes(a, b);
    }

    long long max_subarray_sum(int l, int r) const {
        return query(l, r).best;
    }
};

🔎 Line-by-Line Walkthrough

make_node(x) and identity()

make_node(x) sets all four fields to x because a single element forms a length-1 prefix, suffix, and subarray with sum x. identity() uses NEG = −2⁶⁰ as a safe stand-in for −∞ (large enough to defeat any real answer, far from LLONG_MIN so that X + NEG doesn't overflow).

merge_nodes(a, b)

Computes the four parent fields exactly as derived. Each field is one max of two or three sums, all O(1).

build / point_set / query

Structurally identical to Lesson 1 — the only change is the type of value flowing through the recursion. The recursive skeleton is uniform across the entire segment-tree family.

🎛️ Variants of Composite Nodes

Variant 1 — Longest run of consecutive equal values

Store: longestRun, prefRun, suffRun, leftVal, rightVal, len.

Useful for problems like "longest streak of 1s after toggles".

Variant 2 — Sum + count of zeros

Store: sum and cntZero. Useful when you need to ignore zeros or count them separately.

Variant 3 — Bracket matching

Store: unmatchedOpen, unmatchedClose. Merge cancels A's unmatched opens with B's unmatched closes. Classic application: CF 380C.

Variant 4 — Maximum subarray with at most one element removed

Extension of this lesson: store another field for "best with one drop allowed". Used in many GSS variants.

🧯 Edge Cases

Edge caseBehavior
All negative arraybest = the single largest (least negative) element.
Single element{x, x, x, x}; best = x even if negative.
Empty query (would mean ql > qr)Don't allow — answer is undefined. Caller must ensure ql ≤ qr.
Adding identity to identitysum = 0, others remain −∞. Stable.

⚠️ Common Pitfalls

PitfallFix
Using 0 for pref/suff/best in identityAll-negative arrays return 0. Use −2⁶⁰.
Using LLONG_MIN for −∞A.sum + B.pref overflows. Use −2⁶⁰ or smaller safe constant.
Swapping pref/suff in best calculationIt must be A.suff + B.pref (left's tail meets right's head).
Forgetting that the operation is not commutativeYou must preserve left/right order in merge. merge(A, B) ≠ merge(B, A) in general.

🎯 Decision Guide

Use a composite-node segment tree whenever a single number per node isn't enough. The recipe:

  1. Write down what the question asks at the full interval.
  2. Decide what extra information the children must expose so the parent can answer the question.
  3. Make sure the merge of two valid nodes is itself valid — this is the associativity check.
  4. Pick an identity that absorbs cleanly.

🏁 Final Summary

Composite-node segment trees teach the central segment-tree lesson: "what does a node store, and how do children combine?" Once you can answer those two questions, you can build a segment tree for almost any associative aggregate, even one as non-obvious as max subarray sum.

The same skeleton from Lesson 1 — recursion + the three query cases — applies verbatim. The only changes are the Node struct, the identity, and merge_nodes.
