Tree Queries: Ancestors, Paths, and Subtrees

LCA · Binary Lifting · Euler Tour Flattening · Euler Tour + RMQ · Tarjan Offline · Distances · k-th Ancestor · k-th Node on Path

🧩 1. Problem Definition

We are given a rooted tree with n nodes. We use the global convention 0-based nodes, root 0. We want to answer many queries fast, across three families:

👪 Ancestor Queries

LCA(u, v) — deepest common ancestor.
kth ancestor of u.
isAncestor(u, v).

🛤 Path Queries

distEdges(u, v), distWeight(u, v), kth node on path u → v.

🌳 Subtree Queries

sum / min / xor over subtree(u), point/subtree updates, ancestor tests.

LCA — the unifier

The Lowest Common Ancestor of two nodes u and v is the deepest node that is an ancestor of both. A node is allowed to be its own ancestor.

Example tree

        0
      /   \
     1     2
    / \
   3   4

Sample LCAs

LCA(3, 4) = 1
LCA(3, 2) = 0
LCA(1, 4) = 1
LCA(2, 2) = 2

LCA is the spine of tree query problems. Distance, k-th node on path, ancestor tests, offline tree queries, and many subtree decompositions all reduce to LCA plus a few helper arrays.

Global conventions (used everywhere in this chapter)

0-based nodes root = 0 --u; --v; on input if judge is 1-based cout << answer + 1 on output if judge is 1-based Safety: up[root][j] = root

The convention up[root][j] = root is critical — it avoids invalid ancestor access (no -1 sentinel arithmetic), simplifies the lifting loop, and lets isAncestor work even when u is the root.

📚 2. Tree Vocabulary

Every technique in this chapter rests on five quantities that we compute during a single DFS from the root.

Ancestor, descendant

An ancestor of u is any node on the path from root to u (including u itself).
A descendant of u is any node in subtree(u) (including u itself).

💡 3. Intuition from Scratch

3.1 The "two people climbing" picture (for LCA)

Imagine two people standing at u and v. To find their LCA they perform a two-phase ritual:

  1. The deeper person climbs upward until both are on the same depth.
  2. From there, both climb together, one edge at a time.
  3. The first node where they meet is the LCA.

Walk-through: LCA(3, 2)

path 3 → root: 3 → 1 → 0
path 2 → root: 2 → 0

first common node = 0
LCA(3, 2) = 0

Why this is slow

If the tree is the chain 0 — 1 — 2 — ... — 99999, one query already costs O(n). With q queries the total is O(n · q) — unusable.

Fix: jump in powers of two instead of one edge at a time. That's binary lifting.

3.2 The "subtree is a contiguous block" picture (for subtree queries)

When a DFS enters node u, it must visit every descendant before leaving. So if we stamp each node with a discovery time tin[u], the subtree of u occupies a contiguous interval [tin[u], tout[u]] in DFS order.

This single observation reduces every subtree question to an array range question — and array range questions we already know how to solve with Fenwick trees, segment trees, prefix sums, and so on.

3.3 The big picture in one sentence

DFS gives us depth and timestamps · Binary lifting gives us fast ancestor jumps · Flattening turns subtrees into ranges.

🧠 5. Formal Abstraction

We define everything precisely. From now on every algorithm is just a mechanical consequence of these definitions.

DEF depth(u) = number of edges on the unique path from root to u. depth(root) = 0.
DEF distRoot(u) = sum of edge weights on the unique path from root to u. distRoot(root) = 0. Stored as long long.
DEF ancestor(u) = u, parent(u), parent(parent(u)), ... up to the root. (u itself counts.)
DEF LCA(u, v) = the unique ancestor of both u and v that has the largest depth.
DEF tin(u), tout(u) are DFS timestamps such that subtree(u) = {x : tin(u) ≤ tin(x) ≤ tout(u)}.
DEF up[u][j] = the 2j-th ancestor of u. For root, up[root][j] = root for all j.

Distance, in two lines

distEdges(u, v) = depth(u) + depth(v) − 2·depth(LCA(u, v))
distWeight(u, v) = distRoot(u) + distRoot(v) − 2·distRoot(LCA(u, v))

🔭 6. Key Observations

  1. DFS makes subtrees contiguous. When DFS enters u, it cannot leave until every descendant is visited. So {tin(x) : x ∈ subtree(u)} = [tin(u), tout(u)].
  2. Ancestor test = interval containment. u is an ancestor of v iff tin(u) ≤ tin(v) ≤ tout(v) ≤ tout(u). No DSU, no climbing, just two integer comparisons.
  3. Every non-negative integer is a sum of distinct powers of two. So any "climb k edges upward" decomposes into at most ⌈log₂ k⌉ power-of-two jumps. This is the engine behind binary lifting and behind sparse-table RMQ.
  4. Doubling recurrence. 2j = 2j−1 + 2j−1, so up[u][j] = up[ up[u][j−1] ][j−1] — every entry is built from two smaller entries in O(1).
  5. Path u→v splits at the LCA. Every simple path in a tree decomposes as u → LCA(u,v) → v. So every "what happens on path u→v" question reduces to "what happens on path root→u" plus "what happens on path root→v" minus what's on path root→LCA (twice).
  6. The full Euler Tour turns LCA into RMQ. Between the first occurrences of u and v in the full Euler Tour, the shallowest node visited is exactly LCA(u, v). So LCA collapses to a Range Minimum Query on a depth array.
  7. Offline batches are free. If all queries are known up front, DFS + DSU answers each in amortized O(α(n)) — Tarjan's algorithm.

🐢 7. Naive Climb (Baseline)

Before optimizing, let's nail the obvious algorithm. It is correct, easy to remember, and useful as a sanity check during stress-testing.

Naive LCA pseudocode read me first 
while depth[u] > depth[v]:
    u = parent[u]

while depth[v] > depth[u]:
    v = parent[v]

while u != v:
    u = parent[u]
    v = parent[v]

return u

This is the literal "two people climbing" story. The first two loops equalize depth; the last loop walks them up together.

Time per query: O(height) Worst case O(n) on a chain Useful as a brute force for testing

⚡ 8. Binary Lifting

Instead of remembering only parent[u], we precompute, for every node, its 20-th, 21-th, 22-th, ... ancestor:

up[u][0] = parent[u]              // jump  1 edge
up[u][1] = grandparent[u]         // jump  2 edges
up[u][2] = great-grand-grand[u]   // jump  4 edges
up[u][3] = ...                    // jump  8 edges
up[u][j] = ...                    // jump 2^j edges

The doubling recurrence

To jump 2j edges, jump 2j−1 twice:

up[u][j] = up[ up[u][j−1] ][j−1]

This is O(1) per entry, so the entire table costs O(n log n).

How many levels do we need?

The loop computes LOG = floor(log2(n)) + 1 for n > 0. Therefore the available columns are 0..LOG−1, and the largest stored jump is 2LOG−1, the highest power of two not exceeding n. Since any climb length is at most n − 1, its binary representation uses only these bit positions, so every reachable ancestor can be represented as a sum of stored jumps.

LOG = 1;
while ((1 << LOG) <= max(1, n)) LOG++;

For n = 100000, LOG = 17, so columns 0..16 are available. The largest stored jump is 216 = 65536, and together these bits can represent any climb length up to 99999. The table size is n · LOG ≈ 1.7M ints.

Climbing exactly k edges with binary lifting

Write k in binary. For every set bit at position j, replace u by up[u][j]:

for (int j = 0; j < LOG; j++)
    if (k & (1 << j))
        u = up[u][j];

Example: k = 13 = 0b1101 = 8 + 4 + 1. So jump up[u][0] (1), then up[u][2] (4), then up[u][3] (8). Three jumps, in any order, give the 13-th ancestor.

🧭 8b. The LCA Query Loop

Once the lifting table exists, answering LCA(u, v) is three lines of reasoning:

  1. If u is an ancestor of v, then LCA = u. Symmetrically for v.
  2. Otherwise, lift u upward in powers of two, taking each jump only when the destination is still NOT an ancestor of v.
  3. When all bits are processed, u sits exactly one edge below the LCA. Return up[u][0].
int lca(int u, int v) const {
    if (isAncestor(u, v)) return u;
    if (isAncestor(v, u)) return v;

    for (int j = LOG - 1; j >= 0; j--) {
        if (!isAncestor(up[u][j], v)) {
            u = up[u][j];
        }
    }
    return up[u][0];
}

We sweep from large bits to small bits. This greediness is the same trick as binary search: large jumps first, then refine. We never "overshoot" because we only commit to a jump that keeps us strictly below the LCA.

🔐 9. Invariants

Every correctness argument later in this chapter chains together the four invariants below. Memorize them as the contract of the data structure.

I1 · DFS When dfs(u, p) begins, three facts already hold: (a) p is the parent of u, (b) depth[u] is final, (c) distRoot[u] is final. Children are processed only after these three facts are stable.
I2 · Lifting After preprocessing, for every u and every j ∈ [0, LOG), up[u][j] is the 2j-th ancestor of u. For the root, up[root][j] = root (sentinel-free).
I3 · Timestamps tin[u] is set on entry, tout[u] on exit, and for every pair (u, v):
u is an ancestor of v  ⇔  tin[u] ≤ tin[v] and tout[v] ≤ tout[u].
I4 · LCA Lift Inside the lifting loop, the moving pointer u always satisfies "up[u][j] is NOT an ancestor of v" before every committed jump. When the loop ends, u is exactly one edge below LCA(u, v), so up[u][0] = LCA(u, v).

📐 10. Proof of Correctness

10.1 The lifting table is correct induction on j

Claim. For every u and every j ≥ 0, up[u][j] stores the 2j-th ancestor of u.

Base case (j = 0). By definition, up[u][0] = parent[u], which is the 20=1-st ancestor.

Inductive step. Suppose the claim holds for j − 1. Let mid = up[u][j − 1] — by hypothesis this is the 2j−1-th ancestor of u. Then up[mid][j − 1] is the 2j−1-th ancestor of mid, again by hypothesis. Climbing 2j−1 twice from u ends 2j−1 + 2j−1 = 2j edges above u. So up[u][j] = up[ up[u][j−1] ][j−1] is the 2j-th ancestor. ∎

10.2 isAncestor is correct subtree = interval

During DFS we never leave u before every node in subtree(u) is discovered. So tin[u] is assigned before any tin[x] with x ∈ subtree(u) \ {u}, and tout[u] is assigned after every tout[x] in that subtree. Therefore:

v ∈ subtree(u) ⇔ tin[u] ≤ tin[v] and tout[v] ≤ tout[u].

Conversely, if v ∉ subtree(u), then either DFS has not yet entered u when it enters v (so tin[v] < tin[u]) or DFS has already left u (so tout[u] < tin[v]), in either case violating the condition. ∎

10.3 The LCA loop is correct big-to-small bit greedy

If u is an ancestor of v, the deepest common ancestor is u; symmetrically for v. Otherwise neither is an ancestor of the other, so the true LCA w is a strict ancestor of both on different branches.

The loop sweeps j from high to low and commits the jump u ← up[u][j] only when up[u][j] is not an ancestor of v. By induction on the loop counter, after processing bit j, the current u is the highest descendant of the original u that is still a strict descendant of the true LCA w and whose 20..j−1 ancestors are all strict descendants of w.

After the last bit (j = 0), no further power-of-two jump keeps us strictly below w, so u sits exactly one edge below w. Hence up[u][0] = w = LCA(u, v). ∎

10.4 Distance formula path splits at LCA

Let w = LCA(u, v). The unique simple path from u to v in a tree is u → w → v. Its edge count is (depth[u] − depth[w]) + (depth[v] − depth[w]) = depth[u] + depth[v] − 2·depth[w]. Replacing depth by distRoot gives the weighted version. ∎

10.5 k-th node on path u → v decompose into two kth-ancestor calls

Let w = LCA(u, v), upLen = depth[u] − depth[w], downLen = depth[v] − depth[w], totalLen = upLen + downLen. With our convention k = 0 means u itself.

If k ≤ upLen, the answer is on the upward leg u → w: it is the k-th ancestor of u. Otherwise the answer is on the downward leg w → v; instead of descending (which we cannot do efficiently), we walk upward from v a distance of totalLen − k — same node. ∎

Naming contract

Every variant in this chapter — directed, weighted, offline, RMQ-based — uses the same names: adj, up, depth, distRoot, tin, tout, subtreeSize. Variants only change which fields are populated and how they are combined; the mental model is the same.

📄 13. Master Template — TreeLCA

This is the reusable struct. Copy-paste-and-go for any contest that needs ancestor or path queries on a static rooted tree.

TreeLCA — full C++ click to expand / collapse 
#include <bits/stdc++.h>
using namespace std;

struct TreeLCA {
    int n = 0;
    int LOG = 0;
    int timer = 0;

    vector<vector<pair<int, long long>>> adj;
    vector<vector<int>> up;

    vector<int> depth;
    vector<int> tin;
    vector<int> tout;
    vector<int> subtreeSize;

    vector<long long> distRoot;

    TreeLCA() {}

    TreeLCA(int n) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        LOG = 1;
        while ((1 << LOG) <= max(1, n)) {
            LOG++;
        }

        adj.assign(n, {});
        up.assign(n, vector<int>(LOG, 0));

        depth.assign(n, 0);
        tin.assign(n, 0);
        tout.assign(n, 0);
        subtreeSize.assign(n, 0);
        distRoot.assign(n, 0);

        timer = 0;
    }

    void addEdge(int u, int v, long long w = 1) {
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    void dfs(int u, int p) {
        tin[u] = timer++;
        subtreeSize[u] = 1;

        up[u][0] = p;

        for (int j = 1; j < LOG; j++) {
            up[u][j] = up[up[u][j - 1]][j - 1];
        }

        for (auto [v, w] : adj[u]) {
            if (v == p) {
                continue;
            }

            depth[v] = depth[u] + 1;
            distRoot[v] = distRoot[u] + w;

            dfs(v, u);

            subtreeSize[u] += subtreeSize[v];
        }

        tout[u] = timer - 1;
    }

    void build(int root = 0) {
        timer = 0;
        depth[root] = 0;
        distRoot[root] = 0;

        dfs(root, root);
    }

    bool isAncestor(int u, int v) const {
        return tin[u] <= tin[v] && tout[v] <= tout[u];
    }

    int kthAncestor(int u, int k) const {
        if (k < 0 || k > depth[u]) {
            return -1;
        }

        for (int j = 0; j < LOG; j++) {
            if (k & (1 << j)) {
                u = up[u][j];
            }
        }

        return u;
    }

    int lca(int u, int v) const {
        if (isAncestor(u, v)) {
            return u;
        }

        if (isAncestor(v, u)) {
            return v;
        }

        for (int j = LOG - 1; j >= 0; j--) {
            if (!isAncestor(up[u][j], v)) {
                u = up[u][j];
            }
        }

        return up[u][0];
    }

    int distEdges(int u, int v) const {
        int w = lca(u, v);
        return depth[u] + depth[v] - 2 * depth[w];
    }

    long long distWeight(int u, int v) const {
        int w = lca(u, v);
        return distRoot[u] + distRoot[v] - 2LL * distRoot[w];
    }

    int kthNodeOnPath(int u, int v, int k) const {
        int w = lca(u, v);

        int upLen = depth[u] - depth[w];
        int downLen = depth[v] - depth[w];
        int totalLen = upLen + downLen;

        if (k < 0 || k > totalLen) {
            return -1;
        }

        if (k <= upLen) {
            return kthAncestor(u, k);
        }

        return kthAncestor(v, totalLen - k);
    }
};

🔬 14. Line-by-Line Code Explanation

14.1 Computing LOG

LOG = 1;
while ((1 << LOG) <= max(1, n)) LOG++;

The loop computes LOG = floor(log2(n)) + 1 for n > 0, so columns 0..LOG−1 are available. The largest stored jump is 2LOG−1, the highest power of two not exceeding n. Since any reachable climb length is at most n − 1, its binary representation uses only these bit positions and every reachable ancestor can be reached by a sum of stored jumps. The max(1, n) guards against the pathological n = 0 case. For n = 100000, LOG = 17, so the largest jump is 216 = 65536 and these bits together represent any climb up to 99999.

14.2 Weighted adjacency list

vector<vector<pair<int, long long>>> adj;

Each edge stores {neighbor, weight}. For unweighted trees we just pass w = 1 (the default) and ignore distRoot. Using the same struct for both eliminates an entire family of "I forgot to swap structs" bugs.

14.3 DFS entry timestamp

tin[u] = timer++;

The first thing we do on entering u. tin[u] is the position of u in DFS preorder. This is also the index where u's value lives in the flat array used for subtree queries.

14.4 Root parent convention

dfs(root, root);
// → up[root][0] = root, so up[root][j] = root for all j

This is the safety net. With up[root][j] = root, the lifting loop in kthAncestor can fall off the top without crashing — it just sits at the root. The k > depth[u] guard returns -1 explicitly when the user really asked for an ancestor that doesn't exist.

14.5 Building the lifting table during DFS

up[u][0] = p;
for (int j = 1; j < LOG; j++) {
    up[u][j] = up[up[u][j - 1]][j - 1];
}

We exploit the order of DFS: when we are at u, the entire row up[p][*] for the parent is already complete (filled when DFS entered p). So up[u][j] can be derived in O(1). This is why we fill the table inside DFS rather than in a separate pass — it saves both code and cache misses.

14.6 Recording tout on exit

tout[u] = timer - 1;

We set tout to timer − 1 (the most recent tin ever assigned). Because we set tin with timer++, this is exactly the tin of the last descendant of u — closing the interval [tin[u], tout[u]]. Some texts use a separate exit counter; both are correct, but our version makes the flat array exactly n long (perfect for Fenwick trees).

14.7 isAncestor — interval containment

bool isAncestor(int u, int v) const {
    return tin[u] <= tin[v] && tout[v] <= tout[u];
}

Direct application of Invariant I3. Two integer comparisons, no climbing, no DSU. This is also why isAncestor is O(1) even though lca is O(log n).

14.8 kthAncestor — bit-scan climb

for (int j = 0; j < LOG; j++) {
    if (k & (1 << j)) {
        u = up[u][j];
    }
}

We iterate over the bits of k. For each set bit, we replace u by its 2j-th ancestor. The order does not matter because tree ancestry composes commutatively; we scan low-to-high purely as a convention.

14.9 The LCA loop — the "stay-below" pattern

for (int j = LOG - 1; j >= 0; j--) {
    if (!isAncestor(up[u][j], v)) {
        u = up[u][j];
    }
}
return up[u][0];

This sweeps bits high-to-low, the opposite direction from kthAncestor. The asymmetry is essential: we are searching for the largest jump that still keeps us below the LCA, so we must try big jumps first. The "stay-below" check is the loop's invariant I4. When all bits are exhausted, u's next single step is the LCA.

14.10 Distance formulas

int  distEdges (int u,int v) const { int w=lca(u,v); return depth[u]+depth[v]-2*depth[w]; }
long long distWeight(int u,int v) const { int w=lca(u,v); return distRoot[u]+distRoot[v]-2LL*distRoot[w]; }

The 2LL on the weighted version forces long long arithmetic. Edge weights up to 109 over a chain of 105 nodes already overflow int.

14.11 k-th node on path — the upLen/downLen split

int w = lca(u, v);
int upLen   = depth[u] - depth[w];
int downLen = depth[v] - depth[w];
int totalLen = upLen + downLen;

if (k <= upLen) return kthAncestor(u, k);
return kthAncestor(v, totalLen - k);

If k is on the upward leg, climb directly from u. If it is on the downward leg, climb upward from v by totalLen − k edges — the same node, reached from the other side. This trick is what avoids ever having to descend the tree, which would be expensive in general.

14.12 Iterative buildIterative() — drop-in replacement for deep chains

The recursive dfs() above is short and clear but relies on the process call stack. On adversarial inputs (e.g. a chain of n = 105+ nodes) C++'s default stack can overflow long before n is exhausted. The snippet below is a faithful, in-place replacement that produces the exact same tin, tout, depth, distRoot, subtreeSize, and up[][] arrays as the recursive version, but uses an explicit stack of frames instead. The inclusive tin/tout convention is preserved: tout[u] = timer - 1 at exit, matching the recursive code byte-for-byte.

To use it, leave the recursive dfs() in place for pedagogy and simply call solver.buildIterative(0) instead of solver.build(0) in the main function. Nothing else changes.

void buildIterative(int root = 0) {
    timer = 0;
    depth[root] = 0;
    distRoot[root] = 0;

    struct Frame {
        int u, p, idx;
    };

    vector<Frame> st;
    st.push_back({root, root, 0});

    tin[root] = timer++;
    subtreeSize[root] = 1;
    up[root][0] = root;
    for (int j = 1; j < LOG; j++) {
        up[root][j] = up[up[root][j - 1]][j - 1];
    }

    while (!st.empty()) {
        Frame &f = st.back();
        int u = f.u;

        if (f.idx < (int)adj[u].size()) {
            auto [v, w] = adj[u][f.idx++];
            if (v == f.p) {
                continue;
            }

            depth[v] = depth[u] + 1;
            distRoot[v] = distRoot[u] + w;

            tin[v] = timer++;
            subtreeSize[v] = 1;

            up[v][0] = u;
            for (int j = 1; j < LOG; j++) {
                up[v][j] = up[up[v][j - 1]][j - 1];
            }

            st.push_back({v, u, 0});
        } else {
            tout[u] = timer - 1;

            if (u != f.p) {
                subtreeSize[f.p] += subtreeSize[u];
            }

            st.pop_back();
        }
    }
}

The structure is the classical "explicit recursion-stack" pattern. Each frame remembers which child index it is currently processing. When the frame's child index reaches the size of the adjacency list, the frame closes: we set tout[u] and fold subtreeSize[u] into the parent before popping. The order of operations — stamp tin and fill up[][] at push time, stamp tout and accumulate subtreeSize at pop time — mirrors what the recursive DFS does before and after the recursive call, so the contract on every produced array is identical.

Stack-safe for deep chains Same tin/tout/depth/distRoot/up as recursive Drop-in: just call buildIterative(root)

🪞 15. Euler Tour — Subtree Flattening

Binary lifting answers ancestor and path queries. A different but complementary idea — tree flattening — handles subtree queries by reducing them to array range queries.

The construction

During DFS, append the value of each node to a flat array at position tin[u]:

tin[u] = timer++;
flat[tin[u]] = value[u];

for (int v : children(u)) dfs(v);

tout[u] = timer - 1;

Because DFS visits all descendants of u before backtracking, subtree(u) maps to the contiguous slice flat[tin[u] .. tout[u]]. From this point on every subtree question is just an array range question.

🏔 16. Euler Tour + RMQ — LCA in O(1)

Binary lifting gives O(log n) per LCA query. For purely-LCA, purely-static problems we can do better: O(1) per query after O(n log n) preprocessing.

The full Euler Tour

Unlike the subtree flattening (where each node appears once), the full Euler Tour appends a node every time DFS enters it and every time DFS returns to it from a child. The array has length 2n − 1.

void dfs(int u, int p, int d) {
    depth[u] = d;
    if (first[u] == -1) first[u] = (int)euler.size();
    euler.push_back(u);
    eulerDepth.push_back(d);
    for (int v : adj[u]) if (v != p) {
        dfs(v, u, d + 1);
        euler.push_back(u);            // record u again on return
        eulerDepth.push_back(d);
    }
}

Why this turns LCA into a range minimum

Between the first occurrence of u and the first occurrence of v in the Euler array, DFS must climb from u up to their common ancestor and then descend back down to v. The shallowest node visited in that window is precisely LCA(u, v) — anything shallower would mean DFS left the common ancestor's subtree before reaching v, contradicting the fact that v is in that subtree.

LCA(u, v) = the node with minimum eulerDepth in euler[first[u] .. first[v]]

Range minimum on a static array is a classic sparse-table problem: O(n log n) build, O(1) query.

#include <bits/stdc++.h>
using namespace std;

struct EulerRMQLCA {
    int n;
    vector<vector<int>> adj;

    vector<int> first;
    vector<int> depth;
    vector<int> euler;
    vector<int> eulerDepth;

    vector<int> lg;
    vector<vector<int>> st;

    EulerRMQLCA(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;
        adj.assign(n, {});
        first.assign(n, -1);
        depth.assign(n, 0);
        euler.clear();
        eulerDepth.clear();
    }

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    void dfs(int u, int p, int d) {
        depth[u] = d;

        if (first[u] == -1) {
            first[u] = (int)euler.size();
        }

        euler.push_back(u);
        eulerDepth.push_back(d);

        for (int v : adj[u]) {
            if (v == p) {
                continue;
            }

            dfs(v, u, d + 1);

            euler.push_back(u);
            eulerDepth.push_back(d);
        }
    }

    int betterIndex(int i, int j) const {
        if (eulerDepth[i] < eulerDepth[j]) {
            return i;
        }
        return j;
    }

    void buildSparseTable() {
        int m = (int)euler.size();

        lg.assign(m + 1, 0);
        for (int i = 2; i <= m; i++) {
            lg[i] = lg[i / 2] + 1;
        }

        int K = lg[m] + 1;
        st.assign(K, vector<int>(m));

        for (int i = 0; i < m; i++) {
            st[0][i] = i;
        }

        for (int k = 1; k < K; k++) {
            for (int i = 0; i + (1 << k) <= m; i++) {
                int a = st[k - 1][i];
                int b = st[k - 1][i + (1 << (k - 1))];
                st[k][i] = betterIndex(a, b);
            }
        }
    }

    void build(int root = 0) {
        dfs(root, -1, 0);
        buildSparseTable();
    }

    int rmqIndex(int l, int r) const {
        int len = r - l + 1;
        int k = lg[len];

        int a = st[k][l];
        int b = st[k][r - (1 << k) + 1];

        return betterIndex(a, b);
    }

    int lca(int u, int v) const {
        int l = first[u];
        int r = first[v];

        if (l > r) {
            swap(l, r);
        }

        return euler[rmqIndex(l, r)];
    }

    int distEdges(int u, int v) const {
        int w = lca(u, v);
        return depth[u] + depth[v] - 2 * depth[w];
    }
};

⚠️ 17. Three Euler-Style Ideas — Don't Confuse Them

The word "Euler tour" gets reused for at least three different constructions. Mixing them up is the single most common source of WA on tree problems. They differ in array length, what is stored, and what they can answer.

17.1 Subtree flattening

Length: n.
Each node appears: once, at tin[u].
Use: subtree sum / update / min / xor via Fenwick or seg tree.
Example: [0, 1, 3, 4, 2] for the sample tree.

17.2 Full Euler Tour

Length: 2n − 1.
Each node appears: deg(u) + 1 times.
Use: LCA via RMQ on depth.
Example: [0, 1, 3, 1, 4, 1, 0, 2, 0].

17.3 Entry/exit timestamps

Length: 0 (no array!).
Each node has: tin[u], tout[u].
Use: ancestor test O(1), subtree interval bounds.
No physical Euler array is stored.

A useful mnemonic: (1) for subtree queries, (2) for LCA-as-RMQ, (3) for ancestor tests. They are friends, not the same person.

🧮 18. Tarjan Offline LCA

If all queries are known before we start, we can answer every LCA in amortized O(α(n)) using a single DFS plus a Disjoint Set Union (DSU). This is Tarjan's offline algorithm.

The mental model

Think of DFS sweeping over the tree. A subtree that DFS has fully finished collapses into a single DSU component. Inside that component we remember one designated representative — the ancestor of the component, which is the node that "owns" this finished subtree from the outside view.

When DFS is currently at node u and it sees a query (u, v):

— If v has already been visited, then LCA(u, v) = ancestor[find(v)].

Why? Because v is now inside some already-finished component. That component's ancestor is the topmost node DFS has not yet left after merging v's subtree into its parent's. That topmost node is on the active DFS path; the deepest such node that is also an ancestor of u is the LCA.

Tarjan Invariant After DFS finishes child subtree v of u, we unite(u, v) and set ancestor[find(u)] = u. So the merged component's official ancestor is always the highest still-open node — exactly what every later query against this subtree needs.

Algorithm sketch

dfs(u, p):
    ancestor[find(u)] = u
    visited[u] = true

    for each child v of u:
        distRoot[v] = distRoot[u] + w
        dfs(v, u)
        unite(u, v)
        ancestor[find(u)] = u

    for each query (u, v, id) attached to u:
        if visited[v]:
            w = ancestor[find(v)]
            answer[id] = distRoot[u] + distRoot[v] - 2 * distRoot[w]

The query is answered only when the other endpoint has been visited — this is the symmetric counterpart of the visited[] test. Each query is stored at both endpoints; the one we process is the second one DFS reaches.

Complexity

Build: O((n + q) · α(n)) Practically linear Memory: O(n + q) Requires ALL queries known in advance

#include <bits/stdc++.h>
using namespace std;

struct DSU {
    vector<int> parent;
    vector<int> sz;

    DSU(int n = 0) {
        init(n);
    }

    void init(int n) {
        parent.resize(n);
        sz.assign(n, 1);
        iota(parent.begin(), parent.end(), 0);
    }

    int find(int u) {
        if (parent[u] == u) {
            return u;
        }
        return parent[u] = find(parent[u]);
    }

    void unite(int u, int v) {
        u = find(u);
        v = find(v);

        if (u == v) {
            return;
        }

        if (sz[u] < sz[v]) {
            swap(u, v);
        }

        parent[v] = u;
        sz[u] += sz[v];
    }
};

struct TarjanOfflineLCA {
    int n;
    vector<vector<pair<int, long long>>> adj;
    vector<vector<pair<int, int>>> queries;

    vector<int> ancestor;
    vector<int> visited;
    vector<long long> distRoot;
    vector<long long> answer;

    DSU dsu;

    TarjanOfflineLCA(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        adj.assign(n, {});
        queries.assign(n, {});

        ancestor.assign(n, 0);
        visited.assign(n, 0);
        distRoot.assign(n, 0);

        dsu.init(n);
    }

    void addEdge(int u, int v, long long w = 1) {
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    void addQuery(int id, int u, int v) {
        queries[u].push_back({v, id});
        queries[v].push_back({u, id});
    }

    void dfs(int u, int p) {
        ancestor[dsu.find(u)] = u;
        visited[u] = 1;

        for (auto [v, w] : adj[u]) {
            if (v == p) {
                continue;
            }

            distRoot[v] = distRoot[u] + w;

            dfs(v, u);

            dsu.unite(u, v);
            ancestor[dsu.find(u)] = u;
        }

        for (auto [v, id] : queries[u]) {
            if (visited[v]) {
                int w = ancestor[dsu.find(v)];
                answer[id] = distRoot[u] + distRoot[v] - 2LL * distRoot[w];
            }
        }
    }

    vector<long long> solve(int root, int queryCount) {
        answer.assign(queryCount, 0);
        distRoot[root] = 0;
        dfs(root, root);
        return answer;
    }
};

⚠️ 20. Common Pitfalls / Bugs

20.1 Using subtree Euler order to compute LCA by RMQ

The subtree order has each node once. There is no "second occurrence after the children" to mark the climb. RMQ on this array does not yield the LCA — it yields garbage. RMQ LCA needs the full Euler Tour (each node deg(u)+1 times).

20.2 Returning u instead of up[u][0]

After the lifting loop in lca(u, v), the moving pointer is one edge below the LCA. Returning u gives a wrong answer that is correct in exactly half of the cases (when the test already returned from the initial isAncestor short circuit). This bug is famously elusive in stress tests because the easy cases pass.

20.3 k-th node indexing confusion

This tutorial uses k = 0 → u. Several judges (SPOJ QTREE2 among them) use k = 1 → u. Settle the convention once in the I/O block (typically --k; on input) and never touch kthNodeOnPath for the rest of the solution. Do not patch the discrepancy inside the data structure.

20.4 distRoot stored as int

Edge weights are typically up to 109. With n up to 105, distances can reach 1014. Always use long long. Note the 2LL in distWeight — without it, even with long long operands, the literal 2 can promote to int before multiplication in some edge cases.

20.5 Tarjan query answered before v is visited

Each query is stored at both endpoints. The correct one to answer is the one DFS reaches second — i.e. the moment we are at u and v is already marked visited. Skipping the guard leads to using uninitialized ancestor[] entries and reporting LCAs that don't exist.

20.6 Forgetting to set up[root][0] = root

If up[root][0] is left as something else (zero, garbage, −1), a kthAncestor call with k equal to depth[u] may walk off the end. The DFS in this template handles this by calling dfs(root, root) — the parent of the root is itself.

20.7 Recomputing depth on each lifting iteration

Reading depth[u] inside the lift loop instead of comparing with depth[v] once is harmless but slow. Worse is using depth instead of isAncestor as the "stay below LCA" test — that breaks when u and v are on different branches at the same depth.

Quick reflex rules

If the problem is static and the question is about ancestors or path distance, reach for TreeLCA first. If the question is about subtrees, flatten with DFS and switch to a Fenwick tree. If the question is only LCA, very many queries, no updates, consider the O(1)-query Euler-Tour + sparse table. If all queries are listed up front, Tarjan is the cheapest. Common confusion note

The word "Euler tour" appears in this chapter in two completely different roles: subtree flattening (each node once) and full Euler Tour for LCA RMQ (each node deg(u)+1 times). They share an ancestor in DFS theory but solve different problems. A third trap is mistakenly equating "Euler tour" with "Euler path" — the latter (visits every edge once) is an entirely different topic covered in chapter 13.

🪄 22. Variants & Extensions

22.1 Unweighted vs weighted

Use the same struct. For unweighted trees, pass w = 1 (the default in addEdge) and read distEdges instead of distWeight. The lifting table is identical because weights do not affect ancestry.

22.2 Distance in number of edges only

Skip distRoot entirely. Compute distance from depths alone: depth[u] + depth[v] − 2·depth[LCA]. Saves a vector of long longs.

22.3 Multiple roots / forest

Run DFS from each root. Use a single global timer so that tin/tout intervals never overlap across trees. isAncestor then correctly returns false for cross-tree pairs.

22.4 Path aggregates over min / max / xor

Store up[u][j] alongside a parallel array aggUp[u][j] that holds the min/max/xor of edge weights on the 2j-edge jump. Same recurrence: aggUp[u][j] = combine(aggUp[u][j−1], aggUp[ up[u][j−1] ][j−1]). Query the path's aggregate by accumulating aggUp as you lift, on both sides of the LCA.

22.5 Subtree updates that touch only descendants of u

Use the difference trick on the flat array: a Fenwick tree storing +x at tin[u] and −x at tout[u] + 1 turns "add x to every node in subtree(u)" into two point updates. Point query at tin[v] returns the accumulated value at v.

22.6 Persistent path counters (root-to-node)

Build a persistent segment tree where version x is the multiset of values on the path root → x. The values on path u → v are version[u] + version[v] − version[LCA] − version[parent[LCA]]. This is the technique behind Problem 6 below (SPOJ COT).

22.7 Directed trees (arborescence)

If edges are directed from parent to child, only push {child, w} into adj[parent]. The DFS becomes simpler — no if (v == p) check needed — and lifting works unchanged.

P1 — CSES Company Queries II

VJudge: https://vjudge.net/problem/CSES-1688

Problem story

You are given a company hierarchy tree where every employee has one boss. For pairs of employees (u, v), find their lowest common boss — the deepest employee who is a boss (direct or indirect) of both.

Abstraction

This is exactly LCA(u, v) on the company tree.

Why binary lifting fits

Queries are online (we must answer each before reading the next), there are many of them (up to 2·105), and the tree is static. Binary lifting gives O(log n) per query after O(n log n) preprocessing — easily fast enough.

Full solution click to expand 
#include <bits/stdc++.h>
using namespace std;

struct LCA {
    int n;
    int LOG;
    int timer;

    vector<vector<int>> adj;
    vector<vector<int>> up;
    vector<int> depth;
    vector<int> tin;
    vector<int> tout;

    LCA(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;
        LOG = 1;

        while ((1 << LOG) <= max(1, n)) {
            LOG++;
        }

        timer = 0;

        adj.assign(n, {});
        up.assign(n, vector<int>(LOG, 0));
        depth.assign(n, 0);
        tin.assign(n, 0);
        tout.assign(n, 0);
    }

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    void dfs(int u, int p) {
        tin[u] = timer++;

        up[u][0] = p;

        for (int j = 1; j < LOG; j++) {
            up[u][j] = up[up[u][j - 1]][j - 1];
        }

        for (int v : adj[u]) {
            if (v == p) {
                continue;
            }

            depth[v] = depth[u] + 1;
            dfs(v, u);
        }

        tout[u] = timer - 1;
    }

    void build(int root = 0) {
        timer = 0;
        depth[root] = 0;
        dfs(root, root);
    }

    bool isAncestor(int u, int v) const {
        return tin[u] <= tin[v] && tout[v] <= tout[u];
    }

    int lca(int u, int v) const {
        if (isAncestor(u, v)) {
            return u;
        }

        if (isAncestor(v, u)) {
            return v;
        }

        for (int j = LOG - 1; j >= 0; j--) {
            if (!isAncestor(up[u][j], v)) {
                u = up[u][j];
            }
        }

        return up[u][0];
    }
};

    ios::sync_with_stdio(false);
    cin.tie(nullptr);

        int parent;
        cin >> parent;
        --parent;

        solver.addEdge(parent, child);

    solver.build(0);

        int u, v;
        cin >> u >> v;
        --u;
        --v;

Problem story

Each node of a rooted tree has a value. Support two operations: update the value of a single node, and report the sum of values in the subtree of a node.

#include <bits/stdc++.h>
using namespace std;

struct Fenwick {
    int n;
    vector<long long> bit;

    Fenwick(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;
        bit.assign(n + 1, 0);
    }

    void add(int idx, long long delta) {
        idx++;

        while (idx <= n) {
            bit[idx] += delta;
            idx += idx & -idx;
        }
    }

    long long sumPrefix(int idx) const {
        idx++;

        long long result = 0;

        while (idx > 0) {
            result += bit[idx];
            idx -= idx & -idx;
        }

        return result;
    }

    long long rangeSum(int l, int r) const {
        if (l > r) {
            return 0;
        }

        long long result = sumPrefix(r);

        if (l > 0) {
            result -= sumPrefix(l - 1);
        }

        return result;
    }
};

    cin.tie(nullptr);

        cin >> value[i];
    }

    vector<vector<int>> adj(n);

    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;
        --u;
        --v;

        adj[u].push_back(v);
        adj[v].push_back(u);

    vector<long long> flat(n);

    int timer = 0;

    function<void(int, int)> dfs = [&](int u, int p) {

        flat[timer] = value[u];
        timer++;

        for (int v : adj[u]) {
            if (v == p) {
                continue;
            }

            dfs(v, u);
        }

        tout[u] = timer - 1;
    };

    dfs(0, -1);

    Fenwick fw(n);

    for (int i = 0; i < n; i++) {
        fw.add(i, flat[i]);

            cin >> s >> x;

P3 — Yosupo LCA

Original: https://judge.yosupo.jp/problem/lca

Problem story

Given a rooted tree, answer many static LCA queries.

Abstraction

Pure LCA, no updates, no path or subtree queries — and the constraints are large enough that even O(log n) per query matters. This is the textbook setting for the O(1)-per-query Euler-Tour + sparse-table technique.

Why Euler-Tour RMQ fits

Build once in O(n log n); every subsequent query is two array reads. The sparse table fits comfortably in memory for n ≤ 5·105.

#include <bits/stdc++.h>
using namespace std;

struct EulerRMQLCA {
    int n;

    vector<vector<int>> adj;

    vector<int> first;
    vector<int> depth;
    vector<int> euler;
    vector<int> eulerDepth;

    vector<int> lg;
    vector<vector<int>> st;

    EulerRMQLCA(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        adj.assign(n, {});
        first.assign(n, -1);
        depth.assign(n, 0);

        euler.clear();
        eulerDepth.clear();
    }

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    void dfs(int u, int p, int d) {
        depth[u] = d;

        if (first[u] == -1) {
            first[u] = (int)euler.size();
        }

        euler.push_back(u);
        eulerDepth.push_back(d);

        for (int v : adj[u]) {
            if (v == p) {
                continue;
            }

            dfs(v, u, d + 1);

            euler.push_back(u);
            eulerDepth.push_back(d);
        }
    }

    int betterIndex(int i, int j) const {
        if (eulerDepth[i] < eulerDepth[j]) {
            return i;
        }

        return j;
    }

    void buildSparseTable() {
        int m = (int)euler.size();

        lg.assign(m + 1, 0);

        for (int i = 2; i <= m; i++) {
            lg[i] = lg[i / 2] + 1;
        }

        int K = lg[m] + 1;
        st.assign(K, vector<int>(m));

        for (int i = 0; i < m; i++) {
            st[0][i] = i;
        }

        for (int k = 1; k < K; k++) {
            for (int i = 0; i + (1 << k) <= m; i++) {
                int a = st[k - 1][i];
                int b = st[k - 1][i + (1 << (k - 1))];

                st[k][i] = betterIndex(a, b);
            }
        }
    }

    void build(int root = 0) {
        dfs(root, -1, 0);
        buildSparseTable();
    }

    int rmqIndex(int l, int r) const {
        int len = r - l + 1;
        int k = lg[len];

        int a = st[k][l];
        int b = st[k][r - (1 << k) + 1];

        return betterIndex(a, b);
    }

    int lca(int u, int v) const {
        int l = first[u];
        int r = first[v];

        if (l > r) {
            swap(l, r);
        }

        return euler[rmqIndex(l, r)];
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    EulerRMQLCA solver(n);

    for (int child = 1; child < n; child++) {
        int parent;
        cin >> parent;

        solver.addEdge(parent, child);
    }

    solver.build(0);

    while (q--) {
        int u, v;
        cin >> u >> v;

        cout << solver.lca(u, v) << '\n';
    }

    return 0;
}

P4 — SPOJ QTREE2

VJudge: https://vjudge.net/problem/SPOJ-QTREE2

Problem story

You are given a weighted tree. The supported queries are DIST u v — return the weighted distance from u to v — and KTH u v k — return the k-th node on the path from u to v. The terminator is DONE.

Indexing note for this judge: SPOJ QTREE2 uses k = 1 meaning u itself. Our internal convention is k = 0 meaning u. So we apply --k immediately on input.

Abstraction

Two operations on the binary lifting structure: weighted distance and k-th node on path. Both are directly implemented in the master TreeLCA template.

Why binary lifting fits

Queries are online and interleaved; both query types reduce to one LCA plus one kthAncestor. Since there are no updates in this problem — we never modify the tree during queries — binary lifting is sufficient and there is no need for a path-decomposition data structure.

#include <bits/stdc++.h>
using namespace std;

struct TreeLCA {
    int n;
    int LOG;
    int timer;

    vector<vector<pair<int, long long>>> adj;
    vector<vector<int>> up;

    vector<int> depth;
    vector<int> tin;
    vector<int> tout;

    vector<long long> distRoot;

    TreeLCA(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        LOG = 1;
        while ((1 << LOG) <= max(1, n)) {
            LOG++;
        }

        timer = 0;

        adj.assign(n, {});
        up.assign(n, vector<int>(LOG, 0));
        depth.assign(n, 0);
        tin.assign(n, 0);
        tout.assign(n, 0);
        distRoot.assign(n, 0);
    }

    void addEdge(int u, int v, long long w) {
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    void dfs(int u, int p) {
        tin[u] = timer++;

        up[u][0] = p;

        for (int j = 1; j < LOG; j++) {
            up[u][j] = up[up[u][j - 1]][j - 1];
        }

        for (auto [v, w] : adj[u]) {
            if (v == p) {
                continue;
            }

            depth[v] = depth[u] + 1;
            distRoot[v] = distRoot[u] + w;

            dfs(v, u);
        }

        tout[u] = timer - 1;
    }

    void build(int root = 0) {
        timer = 0;
        depth[root] = 0;
        distRoot[root] = 0;
        dfs(root, root);
    }

    bool isAncestor(int u, int v) const {
        return tin[u] <= tin[v] && tout[v] <= tout[u];
    }

    int kthAncestor(int u, int k) const {
        if (k < 0 || k > depth[u]) {
            return -1;
        }

        for (int j = 0; j < LOG; j++) {
            if (k & (1 << j)) {
                u = up[u][j];
            }
        }

        return u;
    }

    int lca(int u, int v) const {
        if (isAncestor(u, v)) {
            return u;
        }

        if (isAncestor(v, u)) {
            return v;
        }

        for (int j = LOG - 1; j >= 0; j--) {
            if (!isAncestor(up[u][j], v)) {
                u = up[u][j];
            }
        }

        return up[u][0];
    }

    long long distWeight(int u, int v) const {
        int w = lca(u, v);
        return distRoot[u] + distRoot[v] - 2LL * distRoot[w];
    }

    int kthNodeOnPath(int u, int v, int k) const {
        int w = lca(u, v);

        int upLen = depth[u] - depth[w];
        int downLen = depth[v] - depth[w];
        int totalLen = upLen + downLen;

        if (k < 0 || k > totalLen) {
            return -1;
        }

        if (k <= upLen) {
            return kthAncestor(u, k);
        }

        return kthAncestor(v, totalLen - k);
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int tc;
    cin >> tc;

    while (tc--) {
        int n;
        cin >> n;

        TreeLCA solver(n);

        for (int i = 0; i < n - 1; i++) {
            int u, v;
            long long w;
            cin >> u >> v >> w;

            --u;
            --v;

            solver.addEdge(u, v, w);
        }

        solver.build(0);

        string command;

        while (cin >> command) {
            if (command == "DONE") {
                break;
            }

            if (command == "DIST") {
                int u, v;
                cin >> u >> v;

                --u;
                --v;

                cout << solver.distWeight(u, v) << '\n';
            } else {
                int u, v, k;
                cin >> u >> v >> k;

                --u;
                --v;
                --k;

                cout << solver.kthNodeOnPath(u, v, k) + 1 << '\n';
            }
        }
    }

    return 0;
}

P5 — HDU 2586 "How far away?" (Tarjan Offline)

VJudge: https://vjudge.net/problem/HDU-2586

Problem story

Given a weighted tree (a "village road network") and many distance queries between pairs of houses, answer all of them.

Abstraction

All queries are listed before processing. This is the canonical setting for Tarjan offline LCA: do one DFS, maintain a DSU, answer each query at the moment DFS sees its second endpoint.

Why Tarjan fits

Tarjan's amortized O((n + q) · α(n)) beats binary lifting's O((n + q) log n) in both speed and code length for offline batches. Distance is recovered with distRoot[u] + distRoot[v] − 2·distRoot[LCA], computed on the fly inside the DFS.

Full solution click to expand 
#include <bits/stdc++.h>
using namespace std;

struct DSU {
    vector<int> parent;
    vector<int> sz;

    DSU(int n = 0) {
        init(n);
    }

    void init(int n) {
        parent.resize(n);
        sz.assign(n, 1);

        iota(parent.begin(), parent.end(), 0);
    }

    int find(int u) {
        if (parent[u] == u) {
            return u;
        }

        return parent[u] = find(parent[u]);
    }

    void unite(int u, int v) {
        u = find(u);
        v = find(v);

        if (u == v) {
            return;
        }

        if (sz[u] < sz[v]) {
            swap(u, v);
        }

        parent[v] = u;
        sz[u] += sz[v];
    }
};

struct TarjanOfflineDistance {
    int n;

    vector<vector<pair<int, long long>>> adj;
    vector<vector<pair<int, int>>> queries;

    vector<int> ancestor;
    vector<int> visited;
    vector<long long> distRoot;
    vector<long long> answer;

    DSU dsu;

    TarjanOfflineDistance(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        adj.assign(n, {});
        queries.assign(n, {});

        ancestor.assign(n, 0);
        visited.assign(n, 0);
        distRoot.assign(n, 0);

        dsu.init(n);
    }

    void addEdge(int u, int v, long long w) {
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }

    void addQuery(int id, int u, int v) {
        queries[u].push_back({v, id});
        queries[v].push_back({u, id});
    }

    void dfs(int u, int p) {
        ancestor[dsu.find(u)] = u;
        visited[u] = 1;

        for (auto [v, w] : adj[u]) {
            if (v == p) {
                continue;
            }

            distRoot[v] = distRoot[u] + w;

            dfs(v, u);

            dsu.unite(u, v);
            ancestor[dsu.find(u)] = u;
        }

        for (auto [v, id] : queries[u]) {
            if (visited[v]) {
                int w = ancestor[dsu.find(v)];
                answer[id] = distRoot[u] + distRoot[v] - 2LL * distRoot[w];
            }
        }
    }

    vector<long long> solve(int root, int queryCount) {
        answer.assign(queryCount, 0);

        distRoot[root] = 0;
        dfs(root, root);

        return answer;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int tc;
    cin >> tc;

    while (tc--) {
        int n, q;
        cin >> n >> q;

        TarjanOfflineDistance solver(n);

        for (int i = 0; i < n - 1; i++) {
            int u, v;
            long long w;
            cin >> u >> v >> w;

            --u;
            --v;

            solver.addEdge(u, v, w);
        }

        for (int id = 0; id < q; id++) {
            int u, v;
            cin >> u >> v;

            --u;
            --v;

            solver.addQuery(id, u, v);
        }

        vector<long long> answer = solver.solve(0, q);

        for (long long x : answer) {
            cout << x << '\n';
        }
    }

    return 0;
}
Total: O((n + q) · α(n)) Memory: O(n + q)

P6 — SPOJ COT (k-th value on path)

VJudge: https://vjudge.net/problem/SPOJ-COT

Problem story

Each node of a rooted tree has a value. For a query (u, v, k), find the k-th smallest value among the values on the path from u to v.

Abstraction

Decompose the multiset of values on path u → v using persistent segment trees indexed by node. For each node x, build version rootVersion[x] by taking rootVersion[parent[x]] and adding value[x]. Then for any pair:

values(u → v) = rootVersion[u] + rootVersion[v] − rootVersion[LCA] − rootVersion[parent[LCA]]

The subtraction by parent[LCA] instead of by LCA itself is what keeps the LCA's own value counted exactly once instead of twice or zero times.

Why LCA + persistent seg tree fits

We need three things: (a) the LCA of u and v, (b) the multiset of values on each root-to-node path, and (c) a fast way to walk the segment tree four versions in parallel to do a k-th-smallest descent. Binary lifting answers (a), a persistent segment tree built during a DFS provides (b), and (c) is the classic "walk four versions in lockstep" k-th smallest trick.

using namespace std;

struct BinaryLifting {
    int n;
    int LOG;
    int timer;

    vector<vector<int>> adj;
    vector<vector<int>> up;

    vector<int> depth;
    vector<int> tin;
    vector<int> tout;
    vector<int> parent;

    BinaryLifting(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;

        LOG = 1;
        while ((1 << LOG) <= max(1, n)) {
            LOG++;
        }

        timer = 0;

        adj.assign(n, {});
        up.assign(n, vector<int>(LOG, 0));

        depth.assign(n, 0);
        tin.assign(n, 0);
        tout.assign(n, 0);
        parent.assign(n, -1);
    }

    void addEdge(int u, int v) {
        adj[u].push_back(v);
        adj[v].push_back(u);
    }

    void dfs(int u, int p) {
        tin[u] = timer++;

        parent[u] = (u == p ? -1 : p);
        up[u][0] = p;

        for (int j = 1; j < LOG; j++) {
            up[u][j] = up[up[u][j - 1]][j - 1];
        }

        for (int v : adj[u]) {
            if (v == p) {
                continue;
            }

            depth[v] = depth[u] + 1;
            dfs(v, u);
        }

        tout[u] = timer - 1;
    }

    void build(int root = 0) {
        timer = 0;
        depth[root] = 0;
        dfs(root, root);
    }

    bool isAncestor(int u, int v) const {
        return tin[u] <= tin[v] && tout[v] <= tout[u];
    }

    int lca(int u, int v) const {
        if (isAncestor(u, v)) {
            return u;
        }

        if (isAncestor(v, u)) {
            return v;
        }

        for (int j = LOG - 1; j >= 0; j--) {
            if (!isAncestor(up[u][j], v)) {
                u = up[u][j];
            }
        }

        return up[u][0];
    }
};

struct PersistentSegmentTree {
    struct Node {
        int left = 0;
        int right = 0;
        int sum = 0;
    };

    vector<Node> tree;

    PersistentSegmentTree(int maxNodes = 1) {
        tree.reserve(maxNodes);
        tree.push_back(Node());
    }

    int update(int old, int l, int r, int pos) {
        int cur = (int)tree.size();

        tree.push_back(tree[old]);
        tree[cur].sum++;

        if (l != r) {
            int mid = (l + r) / 2;

            if (pos <= mid) {
                tree[cur].left = update(tree[old].left, l, mid, pos);
            } else {
                tree[cur].right = update(tree[old].right, mid + 1, r, pos);
            }
        }

        return cur;
    }

    int kth(int a, int b, int c, int d, int l, int r, int k) const {
        if (l == r) {
            return l;
        }

        int leftCount =
            tree[tree[a].left].sum +
            tree[tree[b].left].sum -
            tree[tree[c].left].sum -
            tree[tree[d].left].sum;

        int mid = (l + r) / 2;

        if (k <= leftCount) {
            return kth(
                tree[a].left,
                tree[b].left,
                tree[c].left,
                tree[d].left,
                l,
                mid,
                k
            );
        }

        return kth(
            tree[a].right,
            tree[b].right,
            tree[c].right,
            tree[d].right,
            mid + 1,
            r,
            k - leftCount
        );
    }
};

    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    vector<int> value(n);
    vector<int> compressed;

    compressed.reserve(n);

    for (int i = 0; i < n; i++) {
        cin >> value[i];
        compressed.push_back(value[i]);
    }

    sort(compressed.begin(), compressed.end());
    compressed.erase(unique(compressed.begin(), compressed.end()), compressed.end());

    BinaryLifting lca(n);

    for (int i = 0; i < n - 1; i++) {
        int u, v;
        cin >> u >> v;

        --u;
        --v;

        lca.addEdge(u, v);
    }

    lca.build(0);

    PersistentSegmentTree pst(n * 20 + 5);
    vector<int> version(n, 0);

    function<void(int, int)> buildVersions = [&](int u, int p) {
        int id = (int)(
            lower_bound(compressed.begin(), compressed.end(), value[u]) -
            compressed.begin()
        );

        int parentVersion = (p == -1 ? 0 : version[p]);

        version[u] = pst.update(
            parentVersion,
            0,
            (int)compressed.size() - 1,
            id
        );

        for (int v : lca.adj[u]) {
            if (v == p) {
                continue;
            }

            buildVersions(v, u);
        }
    };

    buildVersions(0, -1);

    while (q--) {
        int u, v, k;
        cin >> u >> v >> k;

        --u;
        --v;

        int w = lca.lca(u, v);
        int pw = lca.parent[w];

        int parentVersionOfLca = (pw == -1 ? 0 : version[pw]);

        int id = pst.kth(
            version[u],
            version[v],
            version[w],
            parentVersionOfLca,
            0,
            (int)compressed.size() - 1,
            k
        );

        cout << compressed[id] << '\n';
    }

    return 0;
}

🎯 24. Final Mental Model

Now that we have walked the whole landscape, here is the one-screen recap that should stay in working memory during a contest.

Binary lifting — the workhorse

A DFS from the root computes parent, depth, distRoot, tin, and tout. Inside that same DFS we fill the table up[u][j] using the doubling recurrence. With that table in hand, every classical tree query — LCA, distance, k-th ancestor, k-th node on path, ancestor test — costs O(log n) or O(1). This is the right default for almost every contest problem that mentions "tree" and "queries" in the same sentence.

Subtree flattening — the bridge to data structures

Whenever the question is about a subtree rather than a path, lay each node down in the flat DFS-order array at position tin[u]. Every subtree becomes a contiguous range [tin[u], tout[u]], and every "subtree question" becomes an "array range question". From there, Fenwick trees, segment trees, prefix sums, and lazy propagation all become available.

Euler Tour + RMQ — when LCA is the whole problem

If the only query is LCA, and there are very many of them, build the full Euler Tour (2n − 1 entries) and a sparse table over the depth array. Each query is then O(1) — two reads from the sparse table and one from the Euler array. The preprocessing cost O(n log n) is the same as binary lifting, but the per-query cost beats it.

Tarjan offline LCA — when all queries are known in advance

For batches of queries available before processing starts, a single DFS plus a DSU answers each in amortized O(α(n)). The trick is that, while DFS is at node u, every previously-visited subtree is collapsed in the DSU under the node DFS would have to climb back to in order to reach it. When DFS sees a stored query (u, v) with v already visited, the LCA is simply the DSU representative's stored ancestor value.

How the four techniques compose

These four ideas are not mutually exclusive. The persistent-segment-tree solution to SPOJ COT, for example, uses binary lifting to find the LCA and then walks four persistent versions in parallel to extract a k-th value on the path. Many advanced problems combine flattening with binary lifting (e.g. subtree sum plus path distance) — the field is built by composing these pieces, not by choosing among them.

DFS gives us the timestamps. Doubling gives us the jumps. Flattening gives us the ranges. Tarjan amortizes the batches.
