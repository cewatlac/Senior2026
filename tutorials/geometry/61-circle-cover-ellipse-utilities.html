Class 07 — Circle Cover & Ellipse Utilities

What is the smallest circle that holds every point? How many points can a fixed-radius circle cover at once? And is a point inside an ellipse? We build the smallest enclosing circle from its two or three boundary supports, count fixed-radius coverage by an angular sweep (with a brute Class 06 cross-checker), and test ellipse membership by transforming into the ellipse's own frame.

🎯 1 — Problem Definition

Given a set of points, two dual questions arise. The smallest enclosing circle asks for the tightest circle that contains every point. The fixed-radius cover flips it: with a circle of a given radius, how many points can you cover at once by placing it well? Alongside these we add the ellipse membership test — is a point inside, on, or outside an ellipse — which generalizes the circle's inside/on/outside check.

Smallest circle enclosing all n points? Most points coverable by one radius-r circle? Is point p inside / on / outside a circle? Is point p inside / on / outside an ellipse?

The unifying thread is that an optimal circle is pinned down by a few boundary supports: the smallest enclosing circle is fixed by two diametrically opposite points or by three points on its rim. For fixed-radius cover, point-centered candidates handle duplicate or single-location clusters; otherwise, when the best covered set contains at least two distinct locations, there is an optimal placement with two covered points on the boundary. So we enumerate circles defined by a handful of points and test the rest.

Carry-over from Class 06. A radius of 0 is a degenerate point-circle, and a negative radius is invalid. Two identical positive-radius circles meet at infinitely many points, while two identical zero-radius circles are the same single point and share exactly one. The fixed-radius cover here reuses the Class 06 circle-circle intersection contract for placing candidate centers.

🧠 2 — Intuition from Scratch

2.1 — The smallest enclosing circle is held in place by its rim

Imagine the tightest circle around a cloud of points, then try to shrink it. It cannot shrink unless it can move, and it cannot move freely because points on its boundary block it. With a single boundary point the circle could still slide toward that point and shrink; with two it can still rotate about their chord unless they are diametrically opposite; only when two opposite points or three points sit on the rim is the circle fully pinned. So the smallest enclosing circle is always determined by two or three of the input points.

Diagram · Smallest enclosing circle supported by two boundary points

2.2 — A fixed-radius circle slides until two points touch its rim

For the fixed-radius cover, take any circle of radius r covering some points. If only one point is on the boundary (or none), you can drift the circle slightly without losing coverage until a second point lands on the rim. So when the best covered set contains at least two distinct locations, there is an optimal placement with two covered points on its boundary — and those two points, with the fixed radius, determine the candidate center exactly, via the same circle-circle intersection from Class 06. If all covered points are at one location, a point-centered candidate handles the case.

👁 3 — Visual Mental Model

3.1 — Two supports versus three supports

The smallest enclosing circle has exactly one of two shapes: a circle whose diameter is two of the points, or the circumscribed circle of three of the points. The first happens when the spread is dominated by one farthest pair; the second when three points form an acute-enough triangle that no single pair can contain the rest.

Three-point support (circumcircle)

3.2 — Inside, on, or outside — and the EPS band

Every containment question reduces to comparing a distance against a radius. Because floating-point equality is fragile, the boundary is not a razor edge but a thin tolerance band of width set by EPS: just inside it we call the point "on."

Diagram · Inside / on / outside a circle, with the EPS boundary band

🧮 4 — Formal Abstraction

4.1 — Candidate circles from boundary points

Two boundary points a, b give the diameter circle with center (a+b)/2 and radius |a−b|/2. Three boundary points give the circumscribed circle, whose center — the circumcenter — is equidistant from all three and lies at the intersection of the perpendicular bisectors of two sides.

4.2 — The circumcenter from perpendicular bisectors

Translate so a is the origin and write B = b − a, C = c − a. The circumcenter offset (ux, uy) from a solves the two equal-distance conditions, giving

D  = 2 (Bx·Cy − By·Cx)            // = 2 * cross(B, C)
ux = ( Cy·|B|² − By·|C|² ) / D
uy = ( Bx·|C|² − Cx·|B|² ) / D
center = a + (ux, uy)

The denominator D is twice the signed area of the triangle; it vanishes exactly when the three points are collinear, in which case no finite circumcircle exists.

Diagram · Circumcenter as the meeting of two perpendicular bisectors

4.3 — Containment tests

For a circle, the sign of cmp(dist(O, p), r) classifies p as inside, on, or outside. For an ellipse with center c, semi-axes a, b, and rotation θ, transform p into the ellipse's own frame by translating and rotating by −θ, then evaluate

local = rot(p − c, −θ)
v = (local.x / a)² + (local.y / b)²     // v < 1 inside, = 1 on, > 1 outside

4.4 — Ellipse measurements

The area is π·a·b. The linear eccentricity (focal distance from center) is c = √|a² − b²|, with the two foci on the major axis at center ± c. The defining focal property is that every boundary point has the same summed distance 2·(major semi-axis) to the two foci.

Diagram · Ellipse axes, foci, and the focal distance c

💡 5 — Key Observations

  • O1 — Optimal circles are pinned by 2 or 3 points. The smallest enclosing circle has either a diametric pair or three rim points as supports; nothing else is needed to define it.
  • O2 — Incremental construction stays correct in any order. Adding points one at a time and rebuilding only when a point falls outside yields the exact MEC; random order makes it expected linear.
  • O3 — Containment is one comparison. Inside/on/outside is the sign of cmp(dist, r) — the same primitive that drives every enclosure decision.
  • O4 — Fixed-radius cover needs two kinds of candidates. Point-centered candidates handle duplicates and single-location clusters; for distinct support locations, an optimal placement can be chosen with two covered points on the boundary, so the remaining candidates are circle-circle intersections of two radius-r circles (Class 06).
  • O5 — The circumcenter is a linear solve. Two perpendicular-bisector equations give a 2×2 system whose determinant is twice the triangle area; zero area means collinear, no circle.
  • O6 — An ellipse is a circle in disguise. Transform into the ellipse frame and the membership test becomes the unit-circle test on scaled coordinates.
  • O7 — EPS governs every boundary. Whether a point is "on" the circle or ellipse, and whether the MEC just barely contains a point, are all cmp decisions sharing one tolerance.

⚙ 6 — Algorithm / Method Explanation

6.1 — Smallest enclosing circle (incremental Welzl)

Process the points in random order, keeping the MEC of those seen so far. When a new point lies inside or on the current circle, nothing changes. When it lies outside, that point must be on the boundary of the new MEC, so rebuild with it fixed on the rim — nesting the same idea two more levels deep to fix a second and possibly a third boundary point.

Diagram · MEC incremental boundary-update flow

6.2 — Brute force for cross-checking

Deterministically, try every pair as a diameter and every triple as a circumcircle; for each candidate that contains all points, keep the smallest. This is unmistakably correct and serves as an oracle to validate the incremental version on small inputs.

6.3 — Maximum points covered by a fixed-radius circle

For duplicate and single-location cases, first test each input point itself as a candidate center. Then, for each pair of distinct point locations no farther apart than 2r, the radius-r circles passing through both have their centers at the intersection of two radius-r circles centered at the points — a Class 06 circle-circle intersection. Test each candidate center by counting points within r, and keep the maximum.

Diagram · Fixed-radius circle covering several points
Diagram · Two candidate centers for a fixed-radius circle through two points

6.4 — Ellipse membership

Translate the query point by the ellipse center, rotate by −θ so the ellipse becomes axis-aligned, and evaluate the normalized form. The sign of v − 1 gives inside, on, or outside, mirroring the circle test exactly.

Diagram · Ellipse membership by transforming into the ellipse frame

🔒 7 — Invariants

State invariants

  • I1 — The running circle encloses all processed points. After step i of the incremental MEC, every point p[0..i] is inside or on the current circle.
  • I2 — Supports lie on the boundary. Whenever the circle is rebuilt, the points fixed during the rebuild lie exactly on its rim (up to tolerance), never strictly inside.
  • I3 — Containment is sign-exact. pointVsCircle and pointVsEllipse return +1/0/−1 for inside/on/outside under a single shared EPS.

Transition invariants

  • I4 — Rebuilds only shrink-to-fit upward. A point outside the current circle forces a strictly larger circle that now contains it on the boundary; the circle never needlessly grows for an already-contained point.
  • I5 — Candidate validity for fixed-radius cover. A candidate center is used only when the two generating points are within 2r; otherwise no radius-r circle passes through both and the pair is skipped.

Termination invariants

The incremental MEC runs three nested single-pass loops, so it always terminates; with random order the expected total work is linear. The brute force is three fixed nested loops, and the fixed-radius cover is two nested loops with a constant-work candidate step — all finite.

Failure mode — preconditions. An empty point set has no enclosing circle; the routine returns a sentinel with negative radius. circleFrom3 reports found = false for collinear points. A negative cover radius returns 0. Ellipse semi-axes must be positive for the membership test to be meaningful.

📜 8 — Proof of Correctness

8.1 — The smallest enclosing circle is unique

Suppose two distinct circles of the same minimal radius both enclosed all points. Their intersection region (a lens) is contained in a strictly smaller circle — the one centered at the midpoint of the centers with radius reduced by the chord geometry — which would also enclose every point, contradicting minimality. Hence the minimum is unique. Uniqueness is what lets the incremental algorithm commit to each rebuild without backtracking.

8.2 — Why a point outside forces a boundary support

If point p lies outside the MEC of the earlier points, then the MEC of the whole set must include p. It cannot contain p in its strict interior while staying minimal: shrinking toward the others until p reaches the boundary gives a no-larger enclosing circle. So p is on the rim of the new MEC, justifying fixing it as a support and recursing on the rest. The same argument applied twice more bounds the supports at three.

8.3 — The circumcenter formula is exact

The circumcenter is equidistant from a, b, c. Writing B = b−a, C = c−a, the conditions |x|² = |x−B|² and |x|² = |x−C|² linearize to 2B·x = |B|² and 2C·x = |C|². Solving this 2×2 system by Cramer's rule gives the displayed ux, uy with determinant D = 2·cross(B, C). The system is singular precisely when B and C are parallel — the collinear case — where the three points share no finite circumcircle.

8.4 — Fixed-radius cover candidates are complete

Let an optimal radius-r circle cover a maximal set S. If all points of S occupy one location, then a circle centered at that location covers the same set, and this is included by the point-centered candidates. Otherwise, choose two distinct covered locations. The center can be moved within the feasible region that keeps all points of S covered until it reaches the boundary of at least two disk constraints; equivalently, there exists an optimal placement where two distinct covered points lie on the circle boundary. Such centers are exactly the intersections of two radius-r circles centered at input points, which are enumerated. Therefore the maximum count is found.

8.5 — The ellipse transform preserves membership

Rotation and translation are isometries, so a point's position relative to the ellipse is unchanged by moving into the ellipse's frame. In that frame the curve is the axis-aligned ellipse (x/a)² + (y/b)² = 1, whose inside/on/outside is read directly from the sign of v − 1. Hence the transformed test answers the original question exactly.

The randomized MEC is the workhorse: shuffling makes the expected number of rebuilds constant per point. The brute force exists to validate it on small inputs. For maximum cover at a fixed radius, prefer the angular-sweep maxCircleCoverFixedRadius; the cubic maxCoverFixedRadius stays as a simple oracle for cross-checking on small inputs.

📐 10 — Core Template Philosophy

  • Candidates from supports. Enumerate the few circles defined by two or three points; never search the continuous space of all circles.
  • One containment primitive. Inside/on/outside is a single signed compare reused everywhere — MEC rebuilds, cover counts, ellipse tests.
  • Reuse Class 06 for placement. Fixed-radius candidate centers are point-centered candidates plus circle-circle intersections from pairs of distinct locations; do not re-derive them.
  • Shuffle for the bound, not for correctness. The incremental MEC is correct in any order; randomization only buys the expected-linear running time.
  • Validate with a brute oracle. Keep an obviously-correct O(n⁴) MEC to cross-check the fast one on small cases.
  • Transform, don't special-case. An ellipse becomes a unit circle in its own frame — solve the easy problem and map back.

💻 11 — Master / Base Implementation

Standalone, compiles with g++ -std=c++17. The first block repeats just enough of the Class 01–06 substrate (including the trimmed circle-circle classifier) for the file to build on its own; the Class 07 additions follow the divider.

#include <bits/stdc++.h>
using namespace std;

using T     = long double;
using Point = complex<T>;

#define X real()
#define Y imag()

const long double EPS = 1e-12L;
const T PI = acosl(-1.0L);

// ---- rotation helper (typed function, not a macro) ----------------
Point rot(const Point& v, T t) { return v * polar((T)1, t); }

// ---- Class 01-06 substrate ----------------------------------------
template<class U> int cmpInt(const U& a, const U& b = U(0)) { return (a > b) - (a < b); }
template<class U> int cmpFloat(const U& a, const U& b = U(0)) {
    long double d = (long double)a - (long double)b;
    return (d > EPS) - (d < -EPS);
}
template<class U> int cmp(const U& a, const U& b = U(0)) {
    if constexpr (is_floating_point_v<U>) return cmpFloat(a, b);
    else return cmpInt(a, b);
}
T safeSqrt(T x) { if (cmp(x) == 0) return 0; return (T)sqrtl((long double)x); }
// raw-domain clamp for the inverse-trig helpers (no EPS budget; protects against tiny drift only).
T safeAcos(T x) { if (x >  (T)1) x =  1; if (x < (T)-1) x = -1; return (T)acosl((long double)x); }
T safeAsin(T x) { if (x >  (T)1) x =  1; if (x < (T)-1) x = -1; return (T)asinl((long double)x); }
T dot  (const Point& a, const Point& b) { return real(conj(a) * b); }
T cross(const Point& a, const Point& b) { return imag(conj(a) * b); }
T len (const Point& a) { return (T)hypotl((long double)a.X, (long double)a.Y); }
T dist(const Point& a, const Point& b) { return len(a - b); }
bool samePoint(const Point& a, const Point& b) {
    return cmp(a.X, b.X) == 0 && cmp(a.Y, b.Y) == 0;
}
Point perpCCW(const Point& v) { return Point(-v.Y, v.X); }
Point perpCW (const Point& v) { return Point( v.Y,-v.X); }
int orient(const Point& a, const Point& b, const Point& p) {
    return cmp(cross(b - a, p - a));
}

struct Circle { Point c; T r; };

// point vs circle: +1 strictly inside, 0 on boundary, -1 outside.
int pointVsCircle(const Circle& C, const Point& p) {
    return -cmp(dist(C.c, p), C.r);   // dist<r inside, =r on, >r outside
}
bool inOrOn(const Circle& C, const Point& p) { return cmp(dist(C.c, p), C.r) <= 0; }

// Class 06 circle classification (trimmed) for fixed-radius cover.
//   -3 invalid radius, -2 coincident positive-radius (infinite), -1 concentric,
//    0 disjoint/nested, 1 tangent or identical zero-radius point-circles, 2 crossing.
int classifyCircles(const Circle& A, const Circle& B) {
    if (cmp(A.r) < 0 || cmp(B.r) < 0) return -3;
    T d = dist(A.c, B.c);
    if (cmp(d) == 0) {
        if (cmp(A.r, B.r) == 0) { if (cmp(A.r) == 0) return 1; return -2; }
        return -1;
    }
    int outer = cmp(d, A.r + B.r);
    int inner = cmp(d, (T)fabsl((long double)(A.r - B.r)));
    if (outer > 0) return 0;
    if (inner < 0) return 0;
    if (outer == 0 || inner == 0) return 1;
    return 2;
}

// ===================================================================
// CLASS 07 — circle cover & ellipse utilities
// ===================================================================

// Centers of the radius-r circles passing through both a and b.
// Equal-radius circle-circle intersection: returns 0, 1 (tangent), or 2 centers.
int twoCircleCenters(const Point& a, const Point& b, T r, Point& C1, Point& C2) {
    if (cmp(r) < 0) return 0;                      // invalid radius
    if (samePoint(a, b)) {                         // coincident input locations
        if (cmp(r) == 0) { C1 = a; return 1; }     // only the point-circle at a passes through it
        return 0;                                  // r>0: infinitely many centers, no finite unique pair
    }
    T d = dist(a, b);
    if (cmp(d, 2 * r) > 0) return 0;               // too far for one radius-r circle
    T x = d / 2;                                  // equal radii => foot at the midpoint
    T h2 = r*r - x*x;
    Point dir = (b - a) / d, mid = a + dir * x;
    if (cmp(h2) == 0) { C1 = mid; return 1; }     // tangent: one center
    T h = safeSqrt(h2);
    Point nrm = perpCCW(dir);
    C1 = mid + nrm * h;
    C2 = mid - nrm * h;
    return 2;
}

// Smallest circle with a, b as a diameter.
Circle circleFrom2(const Point& a, const Point& b) {
    Point c = (a + b) / (T)2;
    return { c, dist(a, b) / 2 };
}

// Circumscribed circle of a, b, c via perpendicular-bisector intersection.
// found = false when the three points are collinear (no unique circumcircle).
Circle circleFrom3(const Point& a, const Point& b, const Point& c, bool& found) {
    Point B = b - a, C = c - a;
    T D = 2 * cross(B, C);                        // twice the signed triangle area
    if (cmp(D) == 0) { found = false; return { a, 0 }; }
    T B2 = dot(B, B), C2 = dot(C, C);
    T ux = (C.Y * B2 - B.Y * C2) / D;
    T uy = (B.X * C2 - C.X * B2) / D;
    Point center = a + Point(ux, uy);
    found = true;
    return { center, dist(center, a) };
}

// Smallest enclosing circle (incremental Welzl).
// Shuffling gives expected O(n) time; correctness holds for any order.
// An empty input returns a sentinel circle with negative radius.
Circle minEnclosingCircle(vector<Point> p) {
    int n = (int)p.size();
    if (n == 0) return { Point(0, 0), -1 };
    static mt19937 rng(42);                       // fixed seed: reproducible demos
    shuffle(p.begin(), p.end(), rng);
    Circle c = { p[0], 0 };
    for (int i = 1; i < n; i++) {
        if (inOrOn(c, p[i])) continue;
        c = { p[i], 0 };                          // p[i] must be on the new rim
        for (int j = 0; j < i; j++) {
            if (inOrOn(c, p[j])) continue;
            c = circleFrom2(p[i], p[j]);          // p[i], p[j] on the rim
            for (int k = 0; k < j; k++) {
                if (inOrOn(c, p[k])) continue;
                bool ok;
                Circle t = circleFrom3(p[i], p[j], p[k], ok);
                if (ok) c = t;                    // three boundary supports
            }
        }
    }
    return c;
}

// Deterministic brute-force MEC, O(n^4): an oracle for cross-checking.
Circle minEnclosingCircleBrute(const vector<Point>& p) {
    int n = (int)p.size();
    if (n == 0) return { Point(0, 0), -1 };
    if (n == 1) return { p[0], 0 };
    Circle best = { Point(0, 0), -1 };
    bool have = false;
    auto consider = [&](const Circle& c) {
        for (const auto& q : p) if (!inOrOn(c, q)) return;
        if (!have || cmp(c.r, best.r) < 0) { best = c; have = true; }
    };
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            consider(circleFrom2(p[i], p[j]));
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            for (int k = j + 1; k < n; k++) {
                bool ok; Circle c = circleFrom3(p[i], p[j], p[k], ok);
                if (ok) consider(c);
            }
    return best;
}

// Maximum points covered by a circle of fixed radius r. O(n^3).
// Duplicates and single-location clusters are caught by point-centered candidates;
// otherwise an optimal placement has two distinct covered points on its boundary
// (proof 8.4), so the candidate centers are equal-radius circle-circle intersections.
int maxCoverFixedRadius(const vector<Point>& p, T r) {
    if (cmp(r) < 0) return 0;                      // invalid radius
    int n = (int)p.size();
    if (n == 0) return 0;
    auto countAt = [&](const Point& center) {
        int cnt = 0;
        for (const auto& q : p) if (inOrOn({ center, r }, q)) cnt++;
        return cnt;
    };
    int best = 0;
    for (const auto& center : p)                   // point-centered candidates: handle duplicates
        best = max(best, countAt(center));
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            if (cmp(dist(p[i], p[j]), 2 * r) > 0) continue;   // too far for one circle
            Point C1, C2;
            int k = twoCircleCenters(p[i], p[j], r, C1, C2);
            if (k >= 1) best = max(best, countAt(C1));
            if (k >= 2) best = max(best, countAt(C2));
        }
    return best;
}

// ---- ellipse utilities --------------------------------------------
// Semi-axes a (local x) and b (local y), rotated by theta, centered at c.
struct Ellipse { Point c; T a; T b; T theta; };

// +1 inside, 0 on, -1 outside.
int pointVsEllipse(const Ellipse& E, const Point& p) {
    Point local = rot(p - E.c, -E.theta);          // into the ellipse frame
    T v = (local.X * local.X) / (E.a * E.a)
        + (local.Y * local.Y) / (E.b * E.b);
    return -cmp(v, (T)1);                          // v<1 inside, =1 on, >1 outside
}
T ellipseArea(const Ellipse& E) { return PI * E.a * E.b; }
// Linear eccentricity c = sqrt(|a^2 - b^2|); foci at center +/- c on the major axis.
T ellipseFocalDistance(const Ellipse& E) {
    return safeSqrt((T)fabsl((long double)(E.a * E.a - E.b * E.b)));
}

// ---- Extended ellipse toolkit -------------------------------------
// Convert a WORLD point p into the ellipse's LOCAL axis-aligned frame:
// translate by -center, then undo the rotation by -theta.
Point toLocal(const Ellipse& E, const Point& p) {
    return rot(p - E.c, -E.theta);
}
// Inverse: rotate from local frame and add center back.
Point toWorld(const Ellipse& E, const Point& p) {
    return E.c + rot(p, E.theta);
}

// Parametric boundary point: (a cos t, b sin t) in local coords, mapped to world.
// Note: for an ellipse, t is the parameter angle, NOT the polar angle from the center.
Point ellipsePointAt(const Ellipse& E, T t) {
    return toWorld(E, Point(E.a * cosl((long double)t),
                            E.b * sinl((long double)t)));
}

// Tangent direction at parameter t. Derivative of (a cos t, b sin t) is
// (-a sin t, b cos t). We return the rotated WORLD direction (un-normalized).
Point ellipseTangentAt(const Ellipse& E, T t) {
    return rot(Point(-E.a * sinl((long double)t),
                      E.b * cosl((long double)t)), E.theta);
}

// Outward normal at parameter t (un-normalized). For a CCW parameter sweep,
// perpCW(tangent) points outward.
Point ellipseNormalAt(const Ellipse& E, T t) {
    return perpCW(ellipseTangentAt(E, t));
}

// World positions of the two foci.
//   If a >= b: foci sit on the local x-axis at (-c, 0) and (c, 0).
//   If b >  a: the major axis is the local y-axis, so foci sit at (0, +-c).
pair<Point,Point> ellipseFoci(const Ellipse& E) {
    if (cmp(E.a, E.b) >= 0) {
        T c = safeSqrt(E.a * E.a - E.b * E.b);
        return { toWorld(E, Point(-c, 0)), toWorld(E, Point(c, 0)) };
    } else {
        T c = safeSqrt(E.b * E.b - E.a * E.a);
        return { toWorld(E, Point(0, -c)), toWorld(E, Point(0, c)) };
    }
}

// Eccentricity = focal distance / semi-major axis. In [0, 1):
//   0     => circle (a == b).
//   ->1   => extremely elongated.
T ellipseEccentricity(const Ellipse& E) {
    T A = max(E.a, E.b), B = min(E.a, E.b);
    return safeSqrt(A * A - B * B) / A;
}

// A complete ellipse is point-symmetric about its center,
// so the area centroid is exactly the center.
Point ellipseCentroid(const Ellipse& E) { return E.c; }

// ---- Angular-interval sweep for fixed-radius cover ----------------
// Each anchor produces O(n) angular events; sorting them is O(n log n).
// addAngularInterval normalizes [l, r] modulo 2*PI and splits it across 0
// if it wraps around.
T normAngle(T a) {
    T two = 2 * PI;
    a = (T)fmodl((long double)a, (long double)two);
    if (cmp(a) < 0) a += two;
    return a;
}
void addAngularInterval(vector<pair<T,int>>& ev, T l, T r) {
    l = normAngle(l);
    r = normAngle(r);
    if (cmp(l, r) <= 0) {
        ev.push_back({l, +1});
        ev.push_back({r, -1});
    } else {
        ev.push_back({l,         +1});
        ev.push_back({2 * PI,    -1});
        ev.push_back({0,         +1});
        ev.push_back({r,         -1});
    }
}

// Maximum points covered by a circle of fixed radius r. O(n^2 log n).
// For each anchor p[i], every other point within 2r becomes an angular
// interval of valid center directions. The busiest direction wins.
// Boundary rule: at equal angles, start events (+1) are processed BEFORE
// end events (-1), so points lying exactly on the boundary still count.
int maxCircleCoverFixedRadius(const vector<Point>& p, T r) {
    int n = (int)p.size();
    if (n == 0)        return 0;
    if (cmp(r) < 0)    return 0;                        // invalid radius
    if (cmp(r) == 0) {
        // r == 0 collapses the "cover" to "how many points coincide at the chosen center".
        // The arc formula d / (2r) would be 0/0, so handle this case explicitly:
        // return the largest multiplicity of identical points under samePoint().
        int best0 = 0;
        for (int i = 0; i < n; i++) {
            int c = 0;
            for (int j = 0; j < n; j++) if (samePoint(p[i], p[j])) ++c;
            if (c > best0) best0 = c;
        }
        return best0;
    }
    int best = 1;                                       // any single point is always coverable

    for (int i = 0; i < n; i++) {
        vector<pair<T,int>> ev;
        for (int j = 0; j < n; j++) if (i != j) {
            T d = dist(p[i], p[j]);
            if (cmp(d, 2 * r) > 0) continue;            // too far for one radius-r circle
            T base  = (T)atan2l((long double)(p[j] - p[i]).Y,
                                (long double)(p[j] - p[i]).X);
            T delta = safeAcos(d / (2 * r));            // half-angle of the valid arc
            addAngularInterval(ev, base - delta, base + delta);
        }
        sort(ev.begin(), ev.end(),
             [](const pair<T,int>& a, const pair<T,int>& b) {
                 int c = cmp(a.first, b.first);
                 if (c != 0) return c < 0;
                 return a.second > b.second;            // +1 BEFORE -1 at the same angle
             });
        int cur = 1;                                    // anchor p[i] is always covered
        best = max(best, cur);
        for (auto& e : ev) {
            cur += e.second;
            best = max(best, cur);
        }
    }
    return best;
}
Two macros that look like fields but are not. #define X real() makes v.X expand to v.real(); #define Y imag() likewise. rot is a typed helper function, never a macro — and never named rotate (which collides with std::rotate).

⚠ 14 — Common Pitfalls / Bugs

  • P1 — Forgetting to randomize the MEC order. Without a shuffle, adversarial input (e.g. sorted along a line) drives the incremental algorithm to its O(n³) worst case. Shuffle once up front.
  • P2 — Treating collinear triples as a valid circumcircle. Dividing by the zero determinant produces garbage; always check cmp(D) == 0 and fall back to a two-point circle.
  • P3 — Raw < / == on the containment boundary. A point meant to be "on" the circle flickers inside/outside under rounding; always compare with cmp and a shared EPS.
  • P4 — Missing one of the two fixed-radius candidate families. Sampling only grid centers misses the optimum, but sampling only pair-generated circle-circle centers can undercount duplicate/single-location cases. Seed with point-centered candidates first, then add the circle-circle intersection centers from distinct point pairs.
  • P5 — Skipping the 2r feasibility check or the same-point guard. Calling the intersection for far-apart pairs wastes work; calling it for coincident points divides by a zero distance. Guard with the distance test, and handle samePoint(a, b) before any division.
  • P8 — Initializing the cover count from a single pair. Starting best = 1 and only testing pair-generated centers undercounts duplicates: three points stacked at one location with any r ≥ 0 should return 3. Seed best by counting coverage at every input point first.
  • P6 — Negative or near-zero radii unguarded. A negative cover radius must return 0; a zero radius is a point-circle, not an error (Class 06).
  • P7 — Forgetting to rotate into the ellipse frame. Testing (x/a)² + (y/b)² in world coordinates ignores the ellipse's rotation; always apply rot(p − c, −θ) first.

Reflex rules (contest speed)

MEC supports: 2 opposite points, or 3 on the rim. Shuffle ⇒ expected O(n); correctness order-free. Circumcenter: D = 2·cross(B,C); D = 0 ⇒ collinear. Fixed-radius candidates: point centers for duplicate/single-location cases, plus two-point boundary centers. Cover radius < 0 ⇒ 0; radius = 0 ⇒ point-circle. Ellipse = unit circle in its own frame.

Common confusion

"The smallest enclosing circle should pass through the two farthest points." Not always — the farthest pair gives a lower bound, but three points can force a larger circle than any single diameter. Only enumerate or run the incremental algorithm; do not assume the diameter of the farthest pair is the answer.

🌿 16 — Variants & Extensions

16.1 — Smallest enclosing circle of circles (not just points)

If the inputs are disks rather than points, the enclosing circle must contain each disk entirely. What changes: the containment test becomes dist(O, cᵢ) + rᵢ ≤ R, and the supports are tangency conditions rather than point incidences. When to use: bounding clusters with extent, such as covering sensor ranges.

// Does the enclosing circle (O, R) fully contain disk (ci, ri)?
bool containsDisk(const Circle& enc, const Circle& disk) {
    return cmp(dist(enc.c, disk.c) + disk.r, enc.r) <= 0;
}

16.2 — Maximum cover by angular sweep (recommended fixed-radius)

The cubic maxCoverFixedRadius is fine as a small-n oracle, but the workhorse for contest sizes is the angular-sweep version: anchor on each point, and for every other point within 2r compute the arc of center directions for which a radius-r circle through the anchor also covers that point. Sweep those events to find the busiest direction.

Boundary rule. At equal angles, start events (+1) must fire before end events (−1). Otherwise a point that sits exactly on an interval endpoint gets dropped, and the cover misses the optimal direction by one. The sort comparator below encodes this rule.

Radius edge cases. A negative radius is rejected as invalid (return 0). Zero radius is handled separately: with r = 0 the half-angle formula acos(d / (2r)) would be 0 / 0, so before the sweep we short-circuit and return the largest multiplicity of identical points (counted under the same EPS policy as samePoint). For r > 0 the angular sweep runs as described.

Decision map. Use maxCircleCoverFixedRadius by default; fall back to maxCoverFixedRadius only when you need a brute oracle to cross-check the sweep on adversarial inputs.

// Wrap any angle into [0, 2*PI).
T normAngle(T a) {
    T two = 2 * PI;
    a = (T)fmodl((long double)a, (long double)two);
    if (cmp(a) < 0) a += two;
    return a;
}
// Push 2 events for an interval [l, r], or 4 events when it wraps around 0.
void addAngularInterval(vector<pair<T,int>>& ev, T l, T r) {
    l = normAngle(l); r = normAngle(r);
    if (cmp(l, r) <= 0) {
        ev.push_back({l, +1});
        ev.push_back({r, -1});
    } else {
        ev.push_back({l,      +1});
        ev.push_back({2 * PI, -1});
        ev.push_back({0,      +1});
        ev.push_back({r,      -1});
    }
}
// Recommended: O(n^2 log n) maximum cover at fixed radius.
int maxCircleCoverFixedRadius(const vector<Point>& p, T r) {
    int n = (int)p.size();
    if (n == 0 || cmp(r) < 0) return 0;
    int best = 1;                                        // any single point is always coverable
    for (int i = 0; i < n; i++) {
        vector<pair<T,int>> ev;
        for (int j = 0; j < n; j++) if (i != j) {
            T d = dist(p[i], p[j]);
            if (cmp(d, 2 * r) > 0) continue;             // too far for one radius-r circle
            T base  = (T)atan2l((long double)(p[j] - p[i]).Y,
                                (long double)(p[j] - p[i]).X);
            T delta = safeAcos(d / (2 * r));             // half-angle of the valid arc
            addAngularInterval(ev, base - delta, base + delta);
        }
        sort(ev.begin(), ev.end(),
             [](const pair<T,int>& a, const pair<T,int>& b) {
                 int c = cmp(a.first, b.first);
                 if (c != 0) return c < 0;
                 return a.second > b.second;             // start (+1) BEFORE end (-1)
             });
        int cur = 1;                                     // anchor p[i] is always covered
        best = max(best, cur);
        for (auto& e : ev) {
            cur += e.second;
            best = max(best, cur);
        }
    }
    return best;
}

16.3 — Fitting an ellipse / general conic

Beyond membership, one can fit an ellipse to points by least squares on the general conic Ax² + Bxy + Cy² + Dx + Ey + F = 0 with a constraint that selects the ellipse branch. When to use: recovering an ellipse from boundary samples; the membership and area utilities here then interpret the fitted parameters.

// Recover semi-axes and rotation from axis-aligned ellipse half-extents.
// (Full conic fitting is a least-squares solve; this is the read-back step.)
Ellipse ellipseFromAxes(const Point& center, T semiX, T semiY, T theta) {
    return { center, semiX, semiY, theta };
}

16.4 — Full ellipse toolkit (frames, boundary, tangent, foci, eccentricity)

The original Class 07 ships only pointVsEllipse, ellipseArea, and ellipseFocalDistance. Most contest ellipse problems actually need more: a way to walk along the boundary, a tangent/normal at a parameter, and a clean way to switch between the world and the ellipse's own coordinates. The master implementation block already defines them — this section is the conceptual map.

Preconditions for every helper.

  • E.a > 0 and E.b > 0 (positive semi-axes). A zero semi-axis collapses the ellipse to a segment; all of these helpers assume an honest ellipse.
  • E.theta rotates the local x-axis to the desired world direction. The local frame is axis-aligned; the world frame may not be.
  • If E.a == E.b, the ellipse is a circle. Every helper still works — ellipseEccentricity returns 0, the two "foci" coincide with the center, and ellipsePointAt traces a circle.

Coordinate frames. toLocal takes a world point into the ellipse's axis-aligned frame (subtract center, rotate by −θ). toWorld is the inverse (rotate by +θ, add center). Use them whenever you have a routine that only knows how to work on axis-aligned shapes (point-in-ellipse, ray–ellipse intersection, etc.).

Parametric boundary. ellipsePointAt(E, t) returns the world position of (a cos t, b sin t) after rotation and translation. Important: the parameter t is not the polar angle from the center for a non-circular ellipse — it is the angle on the underlying unit circle that gets scaled. If you need an actual world heading, derive it from two boundary samples or from ellipseTangentAt.

Tangent and normal. ellipseTangentAt(E, t) returns the derivative (−a sin t, b cos t) rotated to world; ellipseNormalAt(E, t) applies perpCW to that tangent so it points outward for a CCW parameter sweep. Neither result is normalized; divide by len(...) if you need a unit vector.

Foci. ellipseFoci(E) dispatches on which axis is major. When a ≥ b the major axis is the local x-axis and the foci sit at (±√(a²−b²), 0); when b > a the major axis is the local y-axis and the foci sit at (0, ±√(b²−a²)). The return value pushes both through toWorld, so callers never have to repeat the rotation themselves.

Eccentricity. Defined as e = c / A where A = max(a, b) and c = √(A² − B²). ellipseEccentricity returns 0 for a circle and approaches 1 as the ellipse degenerates toward a segment.

Centroid. A filled ellipse is point-symmetric about its center, so ellipseCentroid(E) simply returns E.c. (Half-ellipses, elliptical arcs, and elliptical sectors have a different centroid — those require an integral over the chosen region.)

Rotated ellipses. All helpers honour E.theta automatically because they route through toLocal/toWorld. If you only ever face axis-aligned ellipses, pass E.theta = 0 and the rotations become identities.

🚀 17 — Problem-Based Applications & Training Drills

Training drill — smallest enclosing circle of n points the canonical MEC pattern, not a contest problem

Scope. A training drill for the smallest enclosing circle, not a full contest problem. It isolates the incremental construction and the inside/on test.

Story. Given n points, output the center and radius of the smallest circle that contains all of them.

Abstraction. Run minEnclosingCircle; the supports are found automatically as points fall outside the running circle.

Why this geometry model fits. The optimum is pinned by two or three boundary points (§8.1–8.2), so the incremental rebuilds reach the exact circle.

Algorithm. Read the points; shuffle; run the three nested passes; print the result with fixed precision.

#include <bits/stdc++.h>
using namespace std;

using T     = long double;
using Point = complex<T>;
#define X real()
#define Y imag()

const long double EPS = 1e-12L;

template<class U> int cmpFloat(const U& a, const U& b = U(0)) {
    long double d = (long double)a - (long double)b;
    return (d > EPS) - (d < -EPS);
}
template<class U> int cmp(const U& a, const U& b = U(0)) {
    if constexpr (is_floating_point_v<U>) return cmpFloat(a, b);
    else return (a > b) - (a < b);
}
T cross(const Point& a, const Point& b) { return imag(conj(a) * b); }
T dot  (const Point& a, const Point& b) { return real(conj(a) * b); }
T len (const Point& a) { return (T)hypotl((long double)a.X, (long double)a.Y); }
T dist(const Point& a, const Point& b) { return len(a - b); }

struct Circle { Point c; T r; };
bool inOrOn(const Circle& C, const Point& p) { return cmp(dist(C.c, p), C.r) <= 0; }

Circle circleFrom2(const Point& a, const Point& b) {
    return { (a + b) / (T)2, dist(a, b) / 2 };
}
Circle circleFrom3(const Point& a, const Point& b, const Point& c, bool& ok) {
    Point B = b - a, C = c - a;
    T D = 2 * cross(B, C);
    if (cmp(D) == 0) { ok = false; return { a, 0 }; }
    T B2 = dot(B, B), C2 = dot(C, C);
    T ux = (C.Y * B2 - B.Y * C2) / D;
    T uy = (B.X * C2 - C.X * B2) / D;
    Point ctr = a + Point(ux, uy);
    ok = true;
    return { ctr, dist(ctr, a) };
}

Circle minEnclosingCircle(vector<Point> p) {
    int n = (int)p.size();
    if (n == 0) return { Point(0, 0), -1 };
    static mt19937 rng(42);
    shuffle(p.begin(), p.end(), rng);
    Circle c = { p[0], 0 };
    for (int i = 1; i < n; i++) {
        if (inOrOn(c, p[i])) continue;
        c = { p[i], 0 };
        for (int j = 0; j < i; j++) {
            if (inOrOn(c, p[j])) continue;
            c = circleFrom2(p[i], p[j]);
            for (int k = 0; k < j; k++) {
                if (inOrOn(c, p[k])) continue;
                bool ok; Circle t = circleFrom3(p[i], p[j], p[k], ok);
                if (ok) c = t;
            }
        }
    }
    return c;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    if (!(cin >> n)) return 0;
    vector<Point> p(n);
    for (int i = 0; i < n; i++) {
        long double x, y; cin >> x >> y; p[i] = Point(x, y);
    }
    Circle c = minEnclosingCircle(p);
    if (cmp(c.r) < 0) { cout << "EMPTY\n"; return 0; }
    cout << fixed << setprecision(6)
         << c.c.X << ' ' << c.c.Y << ' ' << c.r << '\n';
    return 0;
}

Output meaning. EMPTY is printed only for an empty input; otherwise the center and radius are exact to the printed precision.

Pitfalls. Keep the shuffle for the expected-linear bound, and compare containment with cmp so boundary points are treated as covered.

Training drill — maximum points covered by a fixed-radius circle brute oracle / small n; the angular sweep is the recommended contest version

Scope. A pattern drill for the fixed-radius cover, not a standalone contest problem. It shows the candidate-center enumeration that reuses Class 06. For contest sizes use maxCircleCoverFixedRadius (angular sweep, O(n² log n)); the drill code below is the simpler O(n³) brute that doubles as a cross-checker.

Story. Given n points and a radius r, output the maximum number of points that a single radius-r circle can cover, including duplicates at the same location.

Abstraction. This is the brute maxCoverFixedRadius: seed candidate centers from the input points to handle duplicates and single-location clusters, then enumerate pairs within 2r, place candidate centers via equal-radius circle-circle intersection, and count.

Why this geometry model fits. If the best covered set is concentrated at one location, a point-centered candidate captures it. Otherwise, an optimal circle can be chosen so that two distinct covered points touch its rim (§8.4), so pair-generated centers complete the search.

Algorithm. Read points and r; first count coverage at every input point as a candidate center; then for each close-enough pair compute up to two boundary centers; count coverage; report the best.

#include <bits/stdc++.h>
using namespace std;

using T     = long double;
using Point = complex<T>;
#define X real()
#define Y imag()

const long double EPS = 1e-12L;

template<class U> int cmpFloat(const U& a, const U& b = U(0)) {
    long double d = (long double)a - (long double)b;
    return (d > EPS) - (d < -EPS);
}
template<class U> int cmp(const U& a, const U& b = U(0)) {
    if constexpr (is_floating_point_v<U>) return cmpFloat(a, b);
    else return (a > b) - (a < b);
}
T safeSqrt(T x) { if (cmp(x) == 0) return 0; return (T)sqrtl((long double)x); }
T len (const Point& a) { return (T)hypotl((long double)a.X, (long double)a.Y); }
T dist(const Point& a, const Point& b) { return len(a - b); }
bool samePoint(const Point& a, const Point& b) {
    return cmp(a.X, b.X) == 0 && cmp(a.Y, b.Y) == 0;
}
Point perpCCW(const Point& v) { return Point(-v.Y, v.X); }

struct Circle { Point c; T r; };
bool inOrOn(const Circle& C, const Point& p) { return cmp(dist(C.c, p), C.r) <= 0; }

// equal-radius circle-circle intersection centers (Class 06 contract).
int twoCircleCenters(const Point& a, const Point& b, T r, Point& C1, Point& C2) {
    if (cmp(r) < 0) return 0;                      // invalid radius
    if (samePoint(a, b)) {                        // coincident input locations
        if (cmp(r) == 0) { C1 = a; return 1; }    // only the point-circle at a passes through it
        return 0;                                  // r>0: infinitely many centers, no finite unique pair
    }
    T d = dist(a, b);
    if (cmp(d, 2 * r) > 0) return 0;               // too far apart
    T x = d / 2, h2 = r * r - x * x;
    Point dir = (b - a) / d, mid = a + dir * x;
    if (cmp(h2) == 0) { C1 = mid; return 1; }
    T h = safeSqrt(h2);
    Point nrm = perpCCW(dir);
    C1 = mid + nrm * h; C2 = mid - nrm * h;
    return 2;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n; long double r;
    if (!(cin >> n >> r)) return 0;
    vector<Point> p(n);
    for (int i = 0; i < n; i++) {
        long double x, y; cin >> x >> y; p[i] = Point(x, y);
    }
    if (cmp((T)r) < 0) { cout << "INVALID\n"; return 0; }
    if (n == 0) { cout << 0 << '\n'; return 0; }

    auto countAt = [&](const Point& ctr) {
        int cnt = 0;
        for (auto& q : p) if (inOrOn({ ctr, (T)r }, q)) cnt++;
        return cnt;
    };

    int best = 0;
    for (const auto& ctr : p)                      // point-centered candidates: count duplicates
        best = max(best, countAt(ctr));
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++) {
            Point C1, C2;
            int k = twoCircleCenters(p[i], p[j], (T)r, C1, C2);
            if (k >= 1) best = max(best, countAt(C1));
            if (k >= 2) best = max(best, countAt(C2));
        }
    cout << best << '\n';
    return 0;
}

Complexity. O(n³): O(n²) pairs, each candidate counted in O(n). The recommended maxCircleCoverFixedRadius (angular sweep) is O(n² log n).

Output meaning. INVALID flags a negative radius; otherwise the integer is the maximum coverage. For a non-empty set it is at least 1 because each point is counted by its own point-centered candidate, and duplicates at one location are all counted together.

Pitfalls. Guard the same-center and far-apart cases before dividing by d, and treat boundary points as covered with cmp.

📝 18 — Final Summary

The mental model

  • MEC supports: two opposite points or three rim points.
  • Incremental build: rebuild only when a point falls outside.
  • Circumcenter: D = 2·cross(B,C); D = 0 ⇒ collinear.
  • Fixed-radius cover: test point centers, then two-point boundary centers.
  • Containment: sign of cmp(dist, r).
  • Ellipse: transform into its frame, test (x/a)²+(y/b)².

The hard rules

  • Shuffle for expected O(n); correctness is order-free.
  • Check D = 0 before dividing — collinear has no circumcircle.
  • Negative cover radius ⇒ 0; radius 0 is a point-circle.
  • Use cmp/EPS at every boundary.
  • Validate the fast MEC against the brute oracle on small inputs.

Runtime Validation Checklist

Before using these circle-cover and ellipse routines in a contest solution, verify each item below. The Class 01 numeric checklist and the Class 05–06 construction and classification checklists still apply underneath.

Enclosure & cover

  • The point set is shuffled before the incremental MEC so the running time is expected linear.
  • Containment uses cmp(dist, r) with a shared EPS; boundary points count as covered.
  • circleFrom3 checks D = 2·cross(B,C) = 0 for collinear points before dividing.
  • Fixed-radius coverage prefers maxCircleCoverFixedRadius (angular sweep, O(n² log n)) for contest sizes; the brute maxCoverFixedRadius (O(n³)) seeds best from point-centered candidates and is kept as a small-n oracle.
  • twoCircleCenters handles coincident inputs before any division: r = 0 returns the point-circle center, r > 0 returns none, and distinct point pairs generate the true two-boundary centers.
  • A negative cover radius returns 0; a zero radius is treated as a point-circle, not an error.

Ellipse

  • The query point is transformed by rot(p − c, −θ) into the ellipse frame before the membership test.
  • Semi-axes are positive; a = b reduces the ellipse to a circle.
  • Area uses π·a·b and focal distance uses √|a²−b²|.

  • MEC of the unit square (0,0),(2,0),(2,2),(0,2) = center (1,1), radius √2 ≈ 1.4142.
