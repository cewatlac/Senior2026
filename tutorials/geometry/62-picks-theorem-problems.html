Class 08 — Pick's Theorem Problems

A polygon drawn on graph paper hides a beautiful arithmetic identity: its area is determined entirely by counting dots. Pick's theorem ties the area of a lattice polygon to the lattice points strictly inside it and on its boundary. We compute area exactly with the integer shoelace formula, count boundary points with gcd, and recover the interior count — all without ever touching a floating-point number.

🎯 1 — Problem Definition

Pick's theorem applies to a simple polygon whose vertices are all at integer coordinates — a lattice polygon. It relates three integers: the polygon's area A, the number of lattice points strictly inside it I, and the number of lattice points on its boundary B. The theorem states

Pick's theorem. A = I + B/2 − 1 for a simple polygon with integer vertices.

This is remarkable: the continuous quantity (area) is pinned down exactly by two discrete counts. It runs both ways. Given the polygon we can compute A and B directly and then solve for I, which is how the CSES "Polygon Lattice Points" problem is solved.

Area of a lattice polygon? How many points strictly inside? How many points on the boundary? All three from integer arithmetic only
Assumptions, stated up front. The polygon must be simple (edges do not cross and touch only at shared vertices), its vertices must be integer coordinates, and it must have nonzero area (not degenerate to a line or point). Pick's theorem fails if any of these is violated. We handle clockwise and counterclockwise input identically by taking the absolute area.

🧠 2 — Intuition from Scratch

2.1 — Why dots can measure area

Picture the plane tiled by unit cells, one lattice point at each corner. Each interior lattice point "owns" a full unit of area around it; each boundary point owns only a fraction because part of its surrounding cell spills outside the polygon; and going all the way around the boundary you lose exactly one unit of area to the turning. That bookkeeping is precisely A = I + B/2 − 1 — interior points count fully, boundary points count half, and the constant −1 accounts for the full turn around a simple closed curve.

2.2 — Area without trusting fractions

The shoelace formula gives a polygon's area as a sum of cross products of consecutive vertices, divided by two. With integer vertices every cross product is an integer, so the sum before dividing — the doubled area 2A — is always an exact integer. We keep that doubled value and never divide early, sidestepping floating-point error entirely. Pick's theorem cooperates: multiplying through by two turns A = I + B/2 − 1 into 2A = 2I + B − 2, an identity purely among integers.

2.3 — Counting boundary points with gcd

On a single edge from a to b, the lattice points lying on it are evenly spaced. The number of equal steps between the endpoints is gcd(|dx|, |dy|), where dx, dy are the edge's coordinate differences. That is the key fact that turns boundary counting into a few gcd calls.

🧮 4 — Formal Abstraction

4.1 — The shoelace formula (signed and doubled)

For vertices P₀, P₁, …, P_{n−1} in order (indices mod n), the signed doubled area is

2A_signed = Σ (xᵢ · yᵢ₊₁ − xᵢ₊₁ · yᵢ)

The sign encodes orientation: positive for counterclockwise, negative for clockwise. The true area is |2A_signed| / 2. Keeping the doubled, signed integer is the exact-arithmetic foundation for everything else.

4.2 — Boundary lattice points by gcd

An edge from a to b with differences dx = b.x − a.x, dy = b.y − a.y passes through gcd(|dx|, |dy|) + 1 lattice points counting both endpoints, hence gcd(|dx|, |dy|) equal steps. We assign each edge its gcd(|dx|, |dy|) — that counts the start vertex plus the strictly-interior edge points, but not the end vertex, which is the next edge's start. Summing over all edges therefore counts every boundary point exactly once:

B = Σ gcd(|dxᵢ|, |dyᵢ|)        // over all n edges
Diagram · One edge's boundary points counted by gcd(|dx|, |dy|)

4.3 — Pick's theorem in doubled form

Multiply A = I + B/2 − 1 by two to clear the fraction:

2A = 2I + B − 2          ⇒          2I = 2A − B + 2          ⇒          I = (2A − B + 2) / 2

Because 2A and B are integers and the identity guarantees 2A − B + 2 is even, the final division is exact in integer arithmetic.

💡 5 — Key Observations

  • O1 — Doubled area is exactly integer. Each shoelace cross product of integer points is an integer, so 2A needs no float; divide by two only at the very end, if ever.
  • O2 — Orientation is just the sign of 2A. Positive means counterclockwise, negative clockwise; taking the absolute value normalizes both inputs without reordering vertices.
  • O3 — Each edge contributes gcd, not gcd + 1. Counting gcd(|dx|,|dy|) per edge attributes the start vertex to that edge and leaves the end vertex for the next edge, so vertices are never double-counted.
  • O4 — Pick converts one unknown into arithmetic. With 2A and B known, I is a single exact integer division — no geometry search.
  • O5 — The −1 is the global turn. The constant in Pick's theorem comes from going once around a simple closed boundary; it does not depend on the polygon's shape or size.
  • O6 — Overflow, not precision, is the real risk. The danger in integer geometry is a cross product exceeding 64 bits, addressed with a wider accumulator, never rounding.

⚙ 6 — Algorithm / Method Explanation

6.1 — The pipeline: vertices → 2A → B → I

The whole computation is one linear pass that accumulates two quantities, then one arithmetic step:

6.2 — Shoelace area visualization

Each term xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ is twice the signed area of the triangle formed by the origin and the edge (Pᵢ, Pᵢ₊₁). Summing these signed triangles sweeps out the polygon: areas outside cancel, areas inside accumulate, and the total is the signed doubled area.

6.3 — Clockwise versus counterclockwise input

The signed doubled area is positive for counterclockwise vertex order and negative for clockwise. Neither the boundary count nor the interior count depends on orientation, so we take the absolute value of the signed sum and proceed; the polygon need not be re-sorted.

🔒 7 — Invariants

State invariants

  • I1 — Everything stays integer. 2A and B are integers throughout; no floating-point value is ever formed.
  • I2 — The accumulator never overflows. Cross products use a wide enough type (long long, or __int128 for large coordinates) so the running sum is exact.
  • I3 — Boundary count is orientation-free. gcd(|dx|,|dy|) uses absolute differences, so it is identical for an edge traversed either way.

Transition invariants

  • I4 — Each edge adds its start vertex once. Attributing gcd (not gcd + 1) to an edge means the shared vertex between consecutive edges is counted exactly once across the whole loop.
  • I5 — Parity is preserved. 2A − B + 2 is always even for a valid lattice polygon, so the interior division is exact and the result is a non-negative integer.

Termination invariants

Both the area and boundary sums are single passes over n edges, so the computation is one linear loop. Counting arithmetic operations, the pipeline is O(n). In the bit-cost model the per-edge gcd dominates each step, so the exact integer pipeline is O(n log C), with C the maximum coordinate magnitude.

Failure mode — preconditions. If the polygon is not simple, self-intersects, has non-integer vertices, or has zero area, Pick's theorem does not apply and the computed I is meaningless. The code assumes a valid simple integer polygon as a stated precondition.

📜 8 — Proof of Correctness

8.1 — Why the shoelace formula gives the area

Each term xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ equals twice the signed area of the triangle (O, Pᵢ, Pᵢ₊₁) with O the origin — it is the 2D cross product of the two position vectors. Summing over the closed boundary, the contributions outside the polygon cancel in pairs while those inside add up, leaving exactly twice the signed polygon area. The sign is positive when the boundary is traversed counterclockwise.

8.2 — Why an edge contributes exactly gcd(|dx|, |dy|)

Let g = gcd(|dx|, |dy|) for an edge from a to b. Writing dx = g·px, dy = g·py with gcd(|px|,|py|) = 1, the lattice points on the edge are exactly a + k·(px, py) for k = 0, 1, …, g — that is g + 1 points including both ends, because a step of the primitive vector (px, py) never skips a lattice point and never lands on a non-lattice point. So the edge has g equal steps. Assigning the count g to the edge counts its g + 1 points minus the shared end vertex, i.e. the start vertex and the g − 1 strictly-interior edge points.

8.3 — Why summing gcd counts each boundary point exactly once

Around the closed polygon, edge i ends where edge i+1 begins. By counting each edge's start vertex (via the g, not g + 1, attribution) and its strictly-interior points, every vertex is owned by exactly one edge — the one for which it is the start — and every non-vertex boundary point lies on exactly one edge. Therefore B = Σ gcd(|dxᵢ|, |dyᵢ|) counts all boundary lattice points once, with no omissions or duplicates.

8.4 — Pick's theorem and its doubled form

Pick's theorem itself is proved by additivity: it holds for a primitive lattice triangle of area ½, where I = 0 and B = 3. When two lattice polygons are glued along a shared lattice edge, their areas add, and the lattice points on the glued edge stop being boundary points of the combined region; the identity is preserved by this bookkeeping. A simple lattice polygon can be subdivided into primitive lattice triangles after allowing lattice points inside the polygon and on its edges to be used as additional subdivision vertices. Therefore the identity extends to every simple lattice polygon. Doubling gives 2A = 2I + B − 2, and solving for the interior count yields I = (2A − B + 2)/2, which is exact because the right side is even.

📐 10 — Core Template Philosophy

  • Stay in integers. Lattice geometry is exact; introducing double only invites rounding error where none is needed.
  • Keep area doubled. Carry 2A and divide by two only at the final step, so the shoelace sum is always an exact integer.
  • Count boundary with gcd, attribute the start vertex. Use gcd(|dx|,|dy|) per edge — never +1 — so vertices are counted once.
  • Normalize orientation by magnitude. Take |2A| rather than reordering vertices for clockwise input.
  • Guard the accumulator, not the precision. Choose a type wide enough for the largest cross product; reach for __int128 when coordinates are large.
  • State the polygon's preconditions. Simple, integer-vertexed, nonzero area — make these explicit so misuse is obvious.

💻 11 — Master / Base Implementation

Library/template block; compiles as a translation unit with g++ -std=c++17 -c, or after adding a main. Everything is integer-exact: the doubled area is carried in __int128 to guard against overflow on large coordinates, and the interior count is recovered by Pick's theorem with an exact division.

#include <bits/stdc++.h>
using namespace std;

// ===================================================================
// CLASS 08 — Pick's theorem substrate: EXACT INTEGER geometry.
// Coordinates are integers; we keep everything in integers and use
// DOUBLED area so the half in Pick's theorem never needs a fraction.
// ===================================================================
using ll  = long long;
using lll = __int128_t;          // overflow-safe accumulator for large coords

struct P { ll x, y; };

// signed doubled area of a simple polygon via the shoelace formula.
// Returns 2*A with sign: positive if vertices are counterclockwise,
// negative if clockwise. Using __int128 guards against overflow.
lll shoelaceDoubled(const vector<P>& poly) {
    int n = (int)poly.size();
    lll s = 0;
    for (int i = 0; i < n; i++) {
        const P& a = poly[i];
        const P& b = poly[(i + 1) % n];
        s += (lll)a.x * b.y - (lll)b.x * a.y;
    }
    return s;                    // = 2A (signed)
}

// absolute doubled area = |2A|, always non-negative.
lll doubledArea(const vector<P>& poly) {
    lll s = shoelaceDoubled(poly);
    return s < 0 ? -s : s;
}

// boundary lattice points: each edge from a to b contributes
// gcd(|dx|,|dy|) NEW boundary points (counting its start, not its end).
// Summed over all edges, every vertex is counted exactly once.
ll boundaryPoints(const vector<P>& poly) {
    int n = (int)poly.size();
    ll B = 0;
    for (int i = 0; i < n; i++) {
        const P& a = poly[i];
        const P& b = poly[(i + 1) % n];
        ll dx = llabs(b.x - a.x);
        ll dy = llabs(b.y - a.y);
        B += std::gcd(dx, dy);   // gcd(0,0)=0 for a repeated vertex: contributes nothing
    }
    return B;
}

// interior lattice points via Pick's theorem.
// Pick:        A = I + B/2 - 1
// doubled:    2A = 2I + B - 2   =>   2I = 2A - B + 2   =>   I = (2A - B + 2)/2
// 2A - B + 2 is always even, so the division is exact.
lll interiorPoints(const vector<P>& poly) {
    lll twoA = doubledArea(poly);
    ll  B    = boundaryPoints(poly);
    lll twoI = twoA - (lll)B + 2;
    return twoI / 2;
}

// Pick's-theorem area reconstruction from I and B (returns doubled area).
// 2A = 2I + B - 2.  Useful as a sanity check that Pick inverts cleanly.
lll areaDoubledFromPick(lll I, ll B) {
    return 2 * I + (lll)B - 2;
}

// signed-area orientation: +1 CCW, -1 CW, 0 degenerate (zero area).
int orientation(const vector<P>& poly) {
    lll s = shoelaceDoubled(poly);
    return (s > 0) - (s < 0);
}
Why __int128 for the area. A single cross product x·y with coordinates up to 10⁹ reaches 10¹⁸, near the long long limit; summing n of them can overflow 64 bits. Accumulating in __int128 keeps the doubled area exact. If coordinates fit in 10⁴ or so, plain long long is enough.
One rotation helper, never a macro. Although this class is integer-only and does not rotate, the family convention still holds: if a rotation is ever needed it is the typed helper Point rot(const Point& v, T t), never a rotate macro (which would collide with std::rotate).

⚠ 14 — Common Pitfalls / Bugs

  • P1 — Counting gcd + 1 per edge. Adding the end vertex too makes every vertex counted twice, inflating B by n. Attribute only gcd(|dx|,|dy|) to each edge.
  • P2 — Dividing the area by two too early. Halving the shoelace sum before it is complete reintroduces fractions; carry 2A and divide only at the end.
  • P3 — 64-bit overflow on the cross product. With coordinates near 10⁹, a product reaches 10¹⁸ and the running sum can exceed long long; accumulate in __int128.
  • P4 — Forgetting the absolute value for clockwise input. A negative signed area used directly in Pick gives a nonsensical interior count; take |2A|.
  • P5 — Using floating-point area. A double shoelace can round 2A off by one, breaking the exact parity that Pick's division relies on. Stay integer.
  • P6 — Assuming the polygon is simple without checking the source. Pick's theorem silently produces wrong numbers for self-intersecting input; confirm the problem guarantees simplicity.
  • P7 — Mishandling a duplicated closing vertex. Some inputs repeat the first vertex at the end. The zero-length edge is harmless here (gcd(0,0)=0), but be sure not to also add a phantom extra edge.

Reflex rules (contest speed)

Area: keep it doubled, divide last. Boundary: Σ gcd(|dx|,|dy|), never gcd+1. Interior: I = (2A − B + 2)/2, exact. Orientation: just the sign of 2A. Big coords ⇒ __int128 accumulator. Pick needs a simple integer polygon.

Common confusion

"Each edge has gcd + 1 lattice points, so B = Σ (gcd + 1)." The +1 double-counts shared vertices. An edge has gcd + 1 points including both ends, but the end is the next edge's start, so per-edge we add only gcd. The total is B = Σ gcd.

🌿 16 — Variants & Extensions

16.1 — Lattice points strictly inside a single triangle

For one triangle you can apply Pick directly: compute its doubled area and boundary points, then I = (2A − B + 2)/2. When to use: counting interior points of a triangular region without enumerating cells.

// Interior lattice points of triangle (a,b,c) via Pick.
lll triangleInterior(P a, P b, P c) {
    lll twoA = doubledArea({a, b, c});
    ll  B    = boundaryPoints({a, b, c});
    return (twoA - (lll)B + 2) / 2;
}

16.2 — Polygons with holes (Pick for regions with genus)

If a polygon encloses h polygonal holes, the generalized identity is A = I + B/2 − 1 + h, where B counts boundary points of the outer and all inner boundaries and I counts points in the region between them. When to use: annular or perforated lattice regions.

16.3 — Number of lattice points on a segment

A standalone use of the gcd fact: the count of lattice points on a segment from a to b, including both endpoints, is gcd(|dx|,|dy|) + 1. When to use: digital line problems, visibility along a grid, or counting collinear lattice points.

// Lattice points on segment [a,b], inclusive of both endpoints.
ll latticePointsOnSegment(P a, P b) {
    ll dx = llabs(b.x - a.x), dy = llabs(b.y - a.y);
    return std::gcd(dx, dy) + 1;
}

🚀 17 — Problem-Based Applications

CSES — Polygon Area shoelace, output 2A

Problem URL. https://cses.fi/problemset/task/2191

Story. Given the n vertices of a simple polygon in order, output twice its area. Reporting the doubled area keeps the answer an exact integer.

Abstraction. Accumulate the signed shoelace sum, take its absolute value, and print it. No division is needed because the problem asks for 2A.

Why this geometry model fits. The shoelace formula is exact on integer vertices (§8.1), and the absolute value normalizes either orientation (§6.3).

Algorithm. Read the vertices; sum xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ in __int128; print |sum|.

#include <bits/stdc++.h>
using namespace std;
using ll  = long long;
using lll = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    if (!(cin >> n)) return 0;
    vector<ll> x(n), y(n);
    for (int i = 0; i < n; i++) cin >> x[i] >> y[i];

    lll s = 0;
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        s += (lll)x[i] * y[j] - (lll)x[j] * y[i];
    }
    if (s < 0) s = -s;                 // |2A|

    // print __int128
    if (s == 0) { cout << 0 << "\n"; return 0; }
    string out;
    while (s) { out += char('0' + (int)(s % 10)); s /= 10; }
    reverse(out.begin(), out.end());
    cout << out << "\n";
    return 0;
}

Complexity. O(n log C), integer-exact — one linear pass over n edges, with one gcd per edge in O(log C).

Pitfalls. Output is twice the area, so do not divide; accumulate in __int128 for coordinates up to 10⁹.

CSES — Polygon Lattice Points Pick's theorem: print I and B

Problem URL. https://cses.fi/problemset/task/2193

Story. Given a simple polygon with integer vertices, output the number of lattice points strictly inside it and the number on its boundary.

Abstraction. This is Pick's theorem directly: compute |2A| with the shoelace sum, B with the gcd sum, then I = (2A − B + 2)/2.

Why this geometry model fits. Both counts are exact integer quantities (§8.2–8.4); Pick converts the area and boundary into the interior count with one division.

Algorithm. One pass accumulates the signed area and the boundary gcd sum; take the absolute area; solve Pick for I; print I and B.

#include <bits/stdc++.h>
using namespace std;
using ll  = long long;
using lll = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    if (!(cin >> n)) return 0;
    vector<ll> x(n), y(n);
    for (int i = 0; i < n; i++) cin >> x[i] >> y[i];

    lll twoA = 0;
    ll  B    = 0;
    for (int i = 0; i < n; i++) {
        int j = (i + 1) % n;
        twoA += (lll)x[i] * y[j] - (lll)x[j] * y[i];
        ll dx = llabs(x[j] - x[i]);
        ll dy = llabs(y[j] - y[i]);
        B += std::gcd(dx, dy);
    }
    if (twoA < 0) twoA = -twoA;        // |2A|

    // Pick: 2I = 2A - B + 2  ->  I = (2A - B + 2)/2  (exact, even)
    lll I = (twoA - (lll)B + 2) / 2;

    // print I then B
    auto pr = [&](lll v) {
        if (v == 0) { cout << 0; return; }
        string o;
        while (v) { o += char('0' + (int)(v % 10)); v /= 10; }
        reverse(o.begin(), o.end());
        cout << o;
    };
    pr(I); cout << " " << B << "\n";
    return 0;
}

Complexity. O(n log C), integer-exact.

Pitfalls. Print I before B; keep the area doubled; accumulate in __int128 for safety on large coordinates.

Training drill — interior points of a triangle Pick on a single triangle, not a contest problem

Scope. A focused drill applying Pick to one triangle, not a standalone contest problem.

Story. Given three integer vertices, output how many lattice points lie strictly inside the triangle.

Abstraction. Compute the triangle's |2A| and B, then I = (2A − B + 2)/2.

Why this geometry model fits. A triangle is a simple polygon, so Pick applies verbatim.

Algorithm. Reuse the shoelace and gcd sums for three edges; solve Pick.

#include <bits/stdc++.h>
using namespace std;
using ll  = long long;
using lll = __int128_t;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    ll x0, y0, x1, y1, x2, y2;
    if (!(cin >> x0 >> y0 >> x1 >> y1 >> x2 >> y2)) return 0;
    ll X[3] = { x0, x1, x2 }, Y[3] = { y0, y1, y2 };

    lll twoA = 0;
    ll  B    = 0;
    for (int i = 0; i < 3; i++) {
        int j = (i + 1) % 3;
        twoA += (lll)X[i] * Y[j] - (lll)X[j] * Y[i];
        B += std::gcd(llabs(X[j] - X[i]), llabs(Y[j] - Y[i]));
    }
    if (twoA < 0) twoA = -twoA;
    if (twoA == 0) { cout << "DEGENERATE\n"; return 0; }   // collinear: not a triangle

    lll I = (twoA - (lll)B + 2) / 2;
    auto pr = [&](lll v) {
        if (v == 0) { cout << 0; return; }
        string o;
        while (v) { o += char('0' + (int)(v % 10)); v /= 10; }
        reverse(o.begin(), o.end());
        cout << o;
    };
    pr(I); cout << "\n";
    return 0;
}

Complexity. O(1) — three edges.

Output meaning. DEGENERATE is printed when the three points are collinear (zero area, not a valid triangle); otherwise the count of strictly-interior lattice points.

📝 18 — Final Summary

The mental model

  • Pick's theorem: A = I + B/2 − 1.
  • Doubled form: 2A = 2I + B − 2, all integers.
  • Area: shoelace sum of integer cross products, kept doubled.
  • Boundary: B = Σ gcd(|dx|,|dy|), one per edge.
  • Interior: I = (2A − B + 2)/2, an exact division.
  • Orientation: the sign of the signed 2A.

The hard rules

  • Stay integer; never form a double area.
  • Count gcd per edge, never gcd + 1.
  • Take |2A| so clockwise input is fine.
  • Use __int128 when coordinates are large.
  • Pick needs a simple, integer-vertexed, nonzero-area polygon.

Class 08 turns counting into measurement: the shoelace formula gives exact area, gcd counts boundary points, and Pick's theorem recovers the interior — all in pure integer arithmetic. These exact-area and lattice-counting tools carry directly into polygon-cutting and convex-hull area work.

What's next. Class 09 takes up polygon cutting and the convex hull, where the same signed-area machinery decides which side of a line a region falls on and bounds a point set with its tightest convex polygon.

📮 19 — Footer / Contact

Runtime Validation Checklist

Before using these Pick's-theorem routines in a contest solution, verify each item below. The Class 01–07 numeric and geometric checklists still apply underneath where relevant.

Area & boundary

  • The shoelace sum is accumulated in an integer type wide enough for the largest cross product (__int128 for coordinates up to 10⁹).
  • The area is kept doubled and only halved at the very end, if at all.
  • The absolute value of the signed area is taken, so clockwise and counterclockwise input behave identically.
  • Each edge contributes gcd(|dx|,|dy|) to B — never gcd + 1 — so vertices are not double-counted.

Pick & preconditions

  • The interior count uses I = (2A − B + 2)/2, an exact integer division.
  • The polygon is simple, has integer vertices, and has nonzero area before Pick is applied.
  • A repeated closing vertex is handled (its zero-length edge contributes gcd(0,0)=0 and a zero cross product).

Spot checks (run mentally before submitting)

  • Unit square (0,0),(1,0),(1,1),(0,1): 2A = 2, B = 4, I = 0.
  • 4×4 square (0,0),(4,0),(4,4),(0,4): 2A = 32, B = 16, I = 9 (and 16 = 9 + 8 − 1).
  • Right triangle (0,0),(4,0),(0,4): 2A = 16, B = 12, I = 3.
  • Clockwise version of that triangle gives orientation = −1 but the same I = 3.
  • Segment (0,0)→(6,4): gcd(6,4) = 2 steps, so 3 lattice points including endpoints.
  • CSES Polygon Area on a 3×3 square prints 18 (twice the area).
  • CSES Polygon Lattice Points on the 4×4 square prints 9 16.
