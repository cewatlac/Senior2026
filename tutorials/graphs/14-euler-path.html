Euler Path & Euler Tour

Hierholzer's Algorithm · Forward-Star · & Trick · Full Templates

🧩 What is an Euler Path?

Given a graph, we want a walk that uses every edge exactly once. We may visit the same vertex many times, but each edge is traversed at most once.

Euler Path / Trail — uses every edge once, start ≠ end Euler Circuit / Tour — uses every edge once, start = end

Example 1 — Euler Circuit

A ---- B
|      |
|      |
D ---- C

Walk: A → B → C → D → A
All edges used once. Returns to start. ✓

Example 2 — Euler Path

A ---- B ---- C

Walk: A → B → C
All edges used once.
Start ≠ End. ✓

The key difference: a circuit starts and ends at the same vertex. A path does not.

✅ Existence Conditions

Undirected Graph

Each edge contributes 1 to the degree of each endpoint. The condition is purely about parity:

Odd-degree verticesResult
0Euler Circuit exists
2Euler Path exists (those 2 are start/end)
Any other countImpossible

Why? Every time you enter a middle vertex you must leave it — edges consumed in pairs (in + out), so middle vertices must have even degree. Only start/end can be odd.

Also: all vertices with nonzero degree must be in one connected component.

Directed Graph

We track degDif[u] = outDeg[u] − inDeg[u]:

ConditionResult
All vertices: outDeg = inDegEuler Circuit
Exactly one vertex with out−in=+1 (start) and one with in−out=+1 (end), rest balancedEuler Path
Anything elseImpossible

⚙️ Hierholzer's Algorithm

Main idea: Start from a valid start node. Keep following unused edges. When stuck, backtrack. Save edges on the way back. Answer is built in reverse.

Pseudocode

euler(u):
    while u has unused edges:
        take one edge u → v
        mark edge as used
        euler(v)
        add this edge to ans[]   ← saved on return
reverse(ans)

Every edge is processed exactly once → O(E) time.

🌟 Forward-Star Data Structure

We represent edges with three flat arrays:

head[u]  = first edge leaving u     (or -1 if none)
to[e]    = destination of edge e
nxt[e]   = next edge in same start-vertex list

Why it's perfect for Euler

Euler only needs: take the current edge, then advance to next. It does NOT need random erase, sorted access, or balanced BST operations.

Forward-star delivers all of this in O(1) per edge with minimal memory overhead.

addEdge in code

void addEdge(int u, int v) {
    to[ne]  = v;
    nxt[ne] = head[u];
    head[u] = ne++;
}

New edge is always inserted at the front of u's list.

🎩 The & Reference Trick

This single line is the heart of the Euler implementation:

for (int v, &e = head[u]; ~e; ) {
    int ee = e;
    e = nxt[e];   // ← this MODIFIES head[u] directly!
    ...
}

With &e (reference)

int &e = head[u];
// e IS head[u]
e = nxt[e];
// head[u] now points to
// the NEXT edge — edge
// 17 is consumed forever!
✓ Edge consumed correctly

Without &e (copy)

int e = head[u];
// e is a COPY of head[u]
e = nxt[e];
// head[u] is UNCHANGED —
// edge 17 still there
// when recursion returns!
✗ Infinite loop / wrong answer

Why this makes the code O(E)

Each time we advance the loop, we permanently update head[u]. So even after deep recursion returns to u, the already-processed edge is gone. We never revisit it. This is what gives us the clean O(E) complexity.

📄 Master Template — Undirected Euler

vis[ee] = vis[ee^1] = vid ans[i]/2 = original edge index O(E) time
Full C++ Template — Undirected click to expand/collapse 
#include <bits/stdc++.h>
using namespace std;

const int N = 200000 + 5;
const int M = 400000 + 5;  // M*2 for bidirectional
int head[N], to[M*2], nxt[M*2], ne, n;

void init() {
    memset(head, -1, n * sizeof head[0]);
    ne = 0;
}

void addEdge(int u, int v) {
    to[ne] = v; nxt[ne] = head[u]; head[u] = ne++;
}
void addBiEdge(int u, int v) {
    addEdge(u, v); addEdge(v, u);
}

int degArr[N];
int vis[M*2];
int ans[M*2], sz;
int vid = 1;

void euler(int u) {
    for (int v, &e = head[u]; ~e; ) {
        int ee = e;
        e = nxt[e];
        if (vis[ee] == vid) continue;  // ← already used
        vis[ee] = vis[ee ^ 1] = vid;   // ← mark both directions
        v = to[ee];
        euler(v);
        ans[sz++] = ee;                // ← save on return
    }
}

// Returns true if Euler path/circuit exists.
// Sets st = valid start node.
bool buildEulerUndirected(int &st, int m) {
    int odd = 0;
    st = -1;
    for (int i = 0; i < n; i++) {
        if (degArr[i] & 1) { odd++; st = i; }
        if (degArr[i] > 0 && st == -1) st = i;
    }
    if (!(odd == 0 || odd == 2)) return false;
    if (st == -1) { sz = 0; return true; }
    sz = 0; ++vid;
    euler(st);
    if (sz != m) return false;   // ← disconnected check
    reverse(ans, ans + sz);
    return true;
}

// Usage:
// addBiEdge(u, v); degArr[u]++; degArr[v]++;
// int st;
// bool ok = buildEulerUndirected(st, m);
// vertex sequence: st, to[ans[0]], to[ans[1]], ...
// original edge:   ans[i] / 2

Key points

• ee ^ 1 — because edges are added in pairs (0,1), (2,3)... XOR flips the last bit, giving you the reverse direction of the same physical edge.

• vis[ee] == vid — we use a generation counter so we don't need to reset the vis array between test cases.

• sz != m after euler — catches the disconnected graph case.

• reverse(ans, ans+sz) — because edges are saved on return (deepest first), we reverse to get traversal order.

📄 Master Template — Directed Euler

No vis array needed degDif[u] = outDeg[u] − inDeg[u] O(E) time
Full C++ Template — Directed click to expand/collapse 
#include <bits/stdc++.h>
using namespace std;

const int N = 200000 + 5;
const int M = 200000 + 5;
int head[N], to[M], nxt[M], ne, n;

void init() {
    memset(head, -1, n * sizeof head[0]);
    ne = 0;
}
void addEdge(int u, int v) {
    to[ne] = v; nxt[ne] = head[u]; head[u] = ne++;
}

int inDeg[N], outDeg[N];
int ans[M], sz;

void euler(int u) {
    for (int v, &e = head[u]; ~e; ) {
        int ee = e;
        e = nxt[e];   // consume edge immediately via reference
        v = to[ee];
        euler(v);
        ans[sz++] = ee;
    }
}

// Returns true if Euler path/circuit exists.
// Sets st = valid start node.
bool buildEulerDirected(int &st, int m) {
    int plus1 = 0, minus1 = 0;
    st = -1;
    for (int i = 0; i < n; i++) {
        int diff = outDeg[i] - inDeg[i];
        if (diff == 1)       { plus1++; st = i; }
        else if (diff == -1) { minus1++; }
        else if (diff != 0)  { return false; }
        if (outDeg[i] > 0 && st == -1) st = i;
    }
    if (!((plus1==0 && minus1==0) || (plus1==1 && minus1==1)))
        return false;
    if (st == -1) { sz = 0; return true; }
    sz = 0;
    euler(st);
    if (sz != m) return false;
    reverse(ans, ans + sz);
    return true;
}

// Usage:
// addEdge(u, v); outDeg[u]++; inDeg[v]++;
// int st;
// bool ok = buildEulerDirected(st, m);
// vertex sequence: st, to[ans[0]], to[ans[1]], ...
// edge sequence:   ans[i]

Why no vis[] needed in directed version?

In directed graphs, each edge has exactly one direction. The &e = head[u] reference trick already consumes the edge by advancing head[u]. Since we never revisit edges (forward-star only moves forward), no visited array is necessary.

9.3 Word Chain — Lexicographic Order (Catenyms-style)

Each word becomes a directed edge from word[0] to word[last]. An Euler path gives the valid word chain order.

Lexicographic Trick

Forward-star inserts at the front. To get lexicographic traversal order (a, b, c), insert in reverse (c, b, a):

sort(idx, idx+m, [](int a, int b){
    return strcmp(words[a], words[b]) < 0;
});
// Insert in REVERSE sorted order:
for (int k = m-1; k >= 0; k--) {
    int i = idx[k];
    addEdge(words[i][0], words[i][last]);
}
// Recover original word from edge id:
int originalWordIdx = idx[m - 1 - internalEdgeId];

#include <bits/stdc++.h>
using namespace std;
const int N=128, M=1005;
int head[N],to[M],nxt[M],ne,n=128;
char words[M][25]; int idx[M],lenArr[M];
void init(){memset(head,-1,sizeof head);ne=0;}
void addEdge(int u,int v){to[ne]=v;nxt[ne]=head[u];head[u]=ne++;}
int inDeg[N],outDeg[N],ans[M],sz;
void euler(int u){
    for(int v,&e=head[u];~e;){
        int ee=e;e=nxt[e];v=to[ee];euler(v);ans[sz++]=ee;
    }
}
bool buildEuler(int &st,int m){
    int p=0,q=0;st=-1;
    for(int i=0;i<n;i++){
        int d=outDeg[i]-inDeg[i];
        if(d==1){p++;st=i;}
        else if(d==-1)q++;
        else if(d!=0)return false;
        if(outDeg[i]>0&&st==-1)st=i;
    }
    if(!((p==0&&q==0)||(p==1&&q==1)))return false;
    if(st==-1){sz=0;return true;}
    sz=0;euler(st);
    if(sz!=m)return false;
    reverse(ans,ans+sz);return true;
}
int main(){
    int t; cin>>t;
    while(t--){
        int m; cin>>m;
        for(int i=0;i<m;i++){
            cin>>words[i]; idx[i]=i; lenArr[i]=strlen(words[i]);
        }
        sort(idx,idx+m,[](int a,int b){return strcmp(words[a],words[b])<0;});
        init(); memset(inDeg,0,sizeof inDeg); memset(outDeg,0,sizeof outDeg);
        for(int k=m-1;k>=0;k--){
            int i=idx[k];
            int u=words[i][0], v=words[i][lenArr[i]-1];
            addEdge(u,v); outDeg[u]++; inDeg[v]++;
        }
        int st;
        if(!buildEuler(st,m)){cout<<"***\n";continue;}
        for(int i=0;i<m;i++){
            int orig=idx[m-1-ans[i]];
            cout<<words[orig]<<".\n"[i+1==m];
        }
    }
}

⚠️ Pitfalls & Contest Tips

PitfallWhy it happensFix
Forgetting to check connectivityDegree conditions pass but graph is disconnectedAfter euler(), check sz == m
Not reversing ans[]Edges are saved deepest-first (post-order)Always call reverse(ans, ans+sz)
Missing & in &e = head[u]Edge not consumed → infinite recursionMust be int &e = head[u]
Undirected: no vis[ee^1]Reverse direction re-traversedvis[ee] = vis[ee^1] = vid
Wrong start nodeCircuit started at odd-degree nodePick an odd-degree node for path; any node with edges for circuit
Word chain: forgetting reverse insertionForward-star inserts at front, reversing orderSort ascending, insert from back to front
Forgetting ans[i]/2 for undirected edgesInternal ids are doubled (bidirectional storage)Original edge = ans[i] / 2
Stack overflow on large graphsRecursive euler() with huge inputConvert to iterative using explicit stack if needed

⏱ Complexity

Time — O(E)

Each edge is processed exactly once. The &e = head[u] trick ensures we never revisit. Total work is proportional to the number of edges.

Space — O(N + E)

head[], to[], nxt[], vis[], ans[] are all linear. Recursion depth is at most O(E) in the worst case (long chain).

Golden CP Recipe

  1. Build graph with forward-star (addEdge / addBiEdge)
  2. Check Euler existence conditions (degree parity / in-out balance)
  3. Pick correct start node (odd-degree / out−in=+1 / any with edges)
  4. Run recursive euler(start)
  5. Check sz == m for connectivity
  6. Reverse ans[]
  7. Recover vertices / original edges / words as needed
