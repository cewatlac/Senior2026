Same Graph — 3 Ways to Store It

A graph is just nodes + edges. In code we need to store that relationship in memory. There are three common representations. Each makes some operations fast and others slow. The right choice depends on how dense the graph is and which operations you need.

Adjacency Matrix — 2D array, O(n²) space, O(1) edge check
Adjacency List — array of lists, O(n+m) space, best for traversal
Edge List — list of all edges, O(m) space, best for edge-iteration
Forward-Star — flat arrays, O(n+m) space, fastest for competitive programming

We'll use the same 5-node graph throughout so you can compare the three representations side by side.

The reference graph — used in all examples below
5 nodes (0–4) 7 edges

Adjacency Matrix

A 2D array mat[n][n]. mat[u][v] = 1 means edge (u,v) exists. For weighted graphs: mat[u][v] = weight (or 0 / INF if no edge).
Important: In DIRECTED (unweighted) matrix we store only 0/1 — not weights. We show weights only in WEIGHTED mode.
Click any node or any cell in the matrix — row and column highlight.

Edge check (u,v): O(1) — just read mat[u][v]
Space: O(n²) — bad for sparse graphs with millions of nodes
List all neighbors of u: O(n) — must scan entire row

Click a node → see its matrix row/column

Matrix — stored in memory

C++ Code

// Unweighted (Directed or Undirected) — store 0/1
int mat[MAXN][MAXN] = {}; // 0 = no edge, 1 = edge

// Add directed edge u → v
mat[u][v] = 1;

// Add undirected edge (u, v)
mat[u][v] = 1;
mat[v][u] = 1;

// Weighted matrix — store weight
// choose a convention: 0 means no edge (if weights are positive)
mat[u][v] = weight; // directed weighted
mat[v][u] = weight; // for undirected weighted

Adjacency List

An array of lists. adj[u] stores all neighbors of node u. For weighted graphs: store pair<int,int> = (neighbor, weight).
Click any node in the graph or any row in the list — the row highlights and shows exactly what adj[u] contains.

Space: O(n + m) — only stores actual edges
List all neighbors of u: O(deg(u)) — ideal for DFS/BFS
Edge check (u,v): O(deg(u)) — must search u's list

Graph

Click a node → highlight its list row

adj[ ] — stored in memory

    C++ Code

    // Unweighted
vector<int> adj[MAXN];

// Directed edge u → v
adj[u].push_back(v);

// Undirected edge (u, v)
adj[u].push_back(v);
adj[v].push_back(u);

// Weighted
vector<pair<int,int>> adjW[MAXN]; // {neighbor, weight}
adjW[u].push_back({v, w});
adjW[v].push_back({u, w});

    Edge List

    A simple array/vector of all edges. Each entry is a tuple (u, v) or (u, v, w). No adjacency info — just a flat list of every edge in the graph.
    Click any edge in the graph or any row in the list — the row highlights.

    Space: O(m) — most compact
    Edge check: O(m) — must scan all edges
    Best for: Kruskal MST, Bellman-Ford — algorithms that iterate every edge once

    Click an edge → highlight its row in the list

    edges[ ] — stored in memory

    C++ Code

    // Unweighted edge list
vector<pair<int,int>> edges;
edges.push_back({u, v});

// Weighted edge list — tuple (w, u, v)
vector<tuple<int,int,int>> edgesW; // {weight, u, v}
edgesW.push_back({w, u, v});

    Implicit Graphs (State Graphs)

    Sometimes you cannot store an adjacency list/matrix because the graph is HUGE (maybe millions of states). Instead, you treat each state as a node, and you generate its neighbors on-the-fly using a function:
    neighbors(state) → list of next states.

    BFS is common here: shortest number of moves/steps
    We store visited states using a hash/set/map (not visited nodes 0..n-1)
    Examples: 8-puzzle, Ones & Zeros, Getting Zero

    🧩 What is the 8-Puzzle?

    The 8-puzzle is a 3×3 board with 8 numbered tiles (1–8) and one blank cell (0). The only valid move is sliding a tile adjacent to the blank into the blank's position. The goal is to reach the sorted configuration in the minimum number of moves — exactly what BFS solves.

    State = full board configuration (9 digits)
    Move = slide blank ← → ↑ ↓ to an adjacent cell
    BFS = shortest path in number of moves
    9! = 362,880 states possible — too many for a plain array, use a hash map

    Core idea: Instead of building an adjacency list — the state space is far too large — we encode each board configuration as a single integer (the 9 digits concatenated), and generate neighbors on-the-fly by sliding the blank. The hash map replaces the usual visited[] array.

    Example: Encode a 3×3 board into an integer

    We pack digits row-by-row into a single integer code. This makes hashing and storing visited easy. 

    int encode(const vector<vector<int>>& b){
  int code = 0;
  for(auto& row: b)
    for(int x: row)
      code = code*10 + x;
  return code;
}

    Decode + Generate Neighbors (swap the zero)

    Neighbor states are generated by moving the blank (0) left/up/right/down when valid. 

    vector<vector<int>> decode(int code){
  vector<vector<int>> b(3, vector<int>(3));
  for(int i=2; i>=0; --i)
    for(int j=2; j>=0; --j){
      b[i][j] = code % 10;
      code /= 10;
    }
  return b;
}

pair<int,int> findZero(const vector<vector<int>>& b){
  for(int i=0; i<3; ++i)
    for(int j=0; j<3; ++j)
      if(b[i][j]==0) return {i,j};
  return {-1,-1};
}

vector<int> neighbors(int code){
  auto b = decode(code);
  auto [i,j] = findZero(b);

  vector<int> nxt;
  int di=0, dj=-1;
  for(int k=0; k<4; ++k){
    int ni=i+di, nj=j+dj;
    if(0<=ni && ni<3 && 0<=nj && nj<3){
      swap(b[i][j], b[ni][nj]);
      nxt.push_back(encode(b));
      swap(b[i][j], b[ni][nj]);
    }
    swap(di,dj);
    if(k==1) dj=1;
  }
  return nxt;
}

    How BFS looks on an implicit graph

    // BFS on states (implicit graph)
queue<int> q;
unordered_map<int, int> dist;  // dist[state] = steps

q.push(start);
dist[start] = 0;

while(!q.empty()){
  int u = q.front(); q.pop();
  for(int v : neighbors(u)){
    if(!dist.count(v)){
      dist[v] = dist[u] + 1;
      q.push(v);
    }
  }
}

    Forward-Star (Array-Based Adjacency List)

    Instead of vector<vector<int>>, Forward-Star stores all adjacency lists in a few flat arrays. It is the fastest graph representation for competitive programming — O(1) edge insertion, contiguous memory, no dynamic allocation.

    O(1) edge insertion — just increment a counter
    O(n) reset — only clear head[], reuse edge arrays
    O(deg u) neighbor traversal — same as adjacency list
    3 arrays: head[N], to[M], nxt[M] + counter ne

    The Three Arrays

    ArrayPurpose
    head[u]Index of the first edge leaving node u (or -1)
    to[e]Destination of edge e
    nxt[e]Index of the next edge in u's list (or -1)
    neTotal number of edges added so far

    How addEdge(u, v) works

    void addEdge(int u, int v){
    to[ne] = v;         // edge ne goes to v
    nxt[ne] = head[u];  // link to old first edge
    head[u] = ne++;     // this edge is now first
}

    Each new edge is prepended to the linked list. Like inserting at the head of a singly-linked list, but stored in arrays.

    Forward-Star Tricks

    1️⃣ Reverse Edge — e ^ 1

    If you add edges in pairs (u→v then v→u), edge indices are 0,1 then 2,3 then 4,5… The reverse of edge e is always e ^ 1 (XOR with 1). Essential for max-flow algorithms.

    // Add in pairs:
addEdge(u, v);  // edge 0
addEdge(v, u);  // edge 1  →  reverse = 0^1 = 1 ✓
addEdge(a, b);  // edge 2
addEdge(b, a);  // edge 3  →  reverse = 2^1 = 3 ✓

// Get reverse edge:
int rev = e ^ 1;

    2️⃣ Fast Reset — O(n) only

    Only clear head[]. The to[] and nxt[] arrays get overwritten when new edges are added. No need to memset them.

    void init(int n){
    ne = 0;
    memset(head, -1, n * sizeof(head[0]));
    // do NOT clear to[] or nxt[]
}

    3️⃣ Edge Weights — extra array

    Just add a parallel array wt[M]. Same index as to[] and nxt[].

    int wt[M]; // weight of edge e

void addEdge(int u, int v, int w){
    to[ne] = v;
    wt[ne] = w;        // store weight
    nxt[ne] = head[u];
    head[u] = ne++;
}

    4️⃣ Reference Iteration

    Using int &e = head[u] lets you modify the list while traversing. Used in Dinic's max-flow for the "current arc" optimization.

    // Dinic's current-arc optimization
for(int &e = head[u]; ~e; ){
    int v = to[e];
    // ... process edge ...
    e = nxt[e]; // modifies head[u]!
}

    Complete Forward-Star Template

    const int N = 100005;
const int M = 200005;

int head[N], to[M], nxt[M], wt[M], ne;

void init(int n){
    ne = 0;
    memset(head, -1, n * sizeof(head[0]));
}

void addEdge(int u, int v, int w = 0){
    to[ne] = v;
    wt[ne] = w;
    nxt[ne] = head[u];
    head[u] = ne++;
}

void addBiEdge(int u, int v, int w = 0){
    addEdge(u, v, w);
    addEdge(v, u, w);
}

// Traverse neighbors of u:
for(int e = head[u]; e != -1; e = nxt[e]){
    int v = to[e];
    int w = wt[e];
    // process edge u → v with weight w
}

    5️⃣ Encapsulate with #define — Clean DFS / BFS loops

    In contests you write the neighbor-traversal loop over and over. Solution: a single #define makes the code cleaner and faster to write.

    Without #define

    // Repetitive every time you need neighbors
for(int e = head[u]; e != -1; e = nxt[e]){
    int v = to[e];
    int w = wt[e];
    // your logic here
}

    With #define — Encapsulated

    // Define ONCE at the top of your file:
#define neigh(u, e, v) \
    for(int e = head[u], v; ~e && (v = to[e], 1); e = nxt[e])

// Then use everywhere cleanly:
neigh(u, e, v){
    int w = wt[e];
    // your logic with v, e, w
}

    DFS with neigh

    #define neigh(u, e, v) \
    for(int e = head[u], v; ~e && (v = to[e], 1); e = nxt[e])

void dfs(int u){
    vis[u] = 1;
    neigh(u, e, v){
        if(!vis[v]) dfs(v);
    }
}

    BFS with neigh

    #define neigh(u, e, v) \
    for(int e = head[u], v; ~e && (v = to[e], 1); e = nxt[e])

void bfs(int s){
    queue<int> q;
    q.push(s); dist[s] = 0;
    while(!q.empty()){
        int u = q.front(); q.pop();
        neigh(u, e, v){
            if(dist[v] == -1){
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
}
    #define once at the top of your file, then write neigh(u,e,v) instead of the long loop everywhere
    ~e is shorthand for e != -1 (because bitwise NOT of -1 is 0 in C++)

    Grid Traversal — Direction Arrays (di / dj)

    A 2D grid is one of the most common implicit graphs. Each cell (i, j) is a node, and its neighbors are determined by direction arrays di[] / dj[]. No adjacency list needed — neighbors are computed on-the-fly.

    4 Neighbors — Up, Down, Left, Right (most common)
    8 Neighbors — + all 4 diagonals
    Knight Moves — 8 L-shaped jumps (chess)

    Direction Arrays

    4-Direction — Up, Down, Left, Right

    The most common pattern. Used in BFS shortest path on grids, flood fill, connected components.

    int di[] = {-1, 1,  0, 0};  // up, down, stay, stay
int dj[] = { 0, 0, -1, 1};  // stay, stay, left, right

for(int d = 0; d < 4; d++){
    int ni = i + di[d], nj = j + dj[d];
    if(ni >= 0 && ni < n && nj >= 0 && nj < m){
        // (ni, nj) is a valid neighbor
    }
}

    8-Direction — Including Diagonals

    Used when diagonal movement is allowed. Game of Life, image processing, some grid problems.

    int di[] = {-1, -1, -1,  0, 0,  1, 1, 1};
int dj[] = {-1,  0,  1, -1, 1, -1, 0, 1};

// Compact alternative — nested loops:
for(int di = -1; di <= 1; di++)
  for(int dj = -1; dj <= 1; dj++){
    if(di == 0 && dj == 0) continue; // skip self
    int ni = i + di, nj = j + dj;
    // ...
  }

    Knight Moves (Chess)

    The knight has 8 possible L-shaped jumps. Classic BFS problem: minimum moves from A to B.

    int di[] = {-2, -2, -1, -1,  1, 1,  2, 2};
int dj[] = {-1,  1, -2,  2, -2, 2, -1, 1};

    BFS on Grid — Full Template

    int grid[MAXN][MAXN], dist[MAXN][MAXN];
int n, m;
int di[] = {-1, 1,  0, 0};
int dj[] = { 0, 0, -1, 1};

void bfs(int si, int sj){
    memset(dist, -1, sizeof dist);
    queue<pair<int,int>> q;
    dist[si][sj] = 0;
    q.push({si, sj});
    while(!q.empty()){
        auto [ci, cj] = q.front(); q.pop();
        for(int d = 0; d < 4; d++){
            int ni = ci + di[d], nj = cj + dj[d];
            if(ni >= 0 && ni < n && nj >= 0 && nj < m
               && grid[ni][nj] != 1  // not a wall
               && dist[ni][nj] == -1){
                dist[ni][nj] = dist[ci][cj] + 1;
                q.push({ni, nj});
            }
        }
    }
}

    All 4 in C++ — Side by Side

    Reading the same graph input and building all four representations at once.

    int n, m;
cin >> n >> m;

// 1) Matrix
int mat[MAXN][MAXN] = {};

// 2) Adjacency list (weighted pairs)
vector<pair<int,int>> adj[MAXN];

// 3) Edge list (weight first)
vector<tuple<int,int,int>> edges;

// 4) Forward-Star
int head[MAXN], to[MAXM], nxt[MAXM], wt[MAXM], ne;
memset(head, -1, n * sizeof(head[0]));
ne = 0;

for (int i = 0; i < m; i++) {
  int u, v, w;
  cin >> u >> v >> w;

  mat[u][v] = 1;                  // matrix
  adj[u].push_back({v, w});        // adjacency list
  edges.push_back({w, u, v});      // edge list

  // forward-star
  to[ne] = v; wt[ne] = w;
  nxt[ne] = head[u]; head[u] = ne++;
}

    Time & Space Comparison

    n = nodes, m = edges, deg(u) = degree of node u

    Operation Matrix Adj. List Edge List Forward-Star
    Space O(n²) O(n + m) O(m) O(n + m)
    Check edge (u,v) O(1) O(deg u) O(m) O(deg u)
    All neighbors of u O(n) O(deg u) O(m) O(deg u)
    Add edge O(1) O(1) amortized O(1) O(1) always
    DFS / BFS O(n²) O(n + m) Not direct O(n + m)
    Reset graph O(n²) O(n + m) O(m) O(n) only head[]
    Cache friendly Yes (2D) No (scattered vectors) Yes Yes (flat arrays)
    Edge index No No Yes Yes (+ reverse trick)
