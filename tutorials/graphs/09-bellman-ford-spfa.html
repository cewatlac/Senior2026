One Family — Three Layers FRAMEWORK

Bellman-Ford, SPFA, and every difference-constraints application are the same algorithm with three independent axes of variation. The relaxation rule is constant. Everything else is a controlled parameter.

Every variant in this family varies on exactly three axes:
(1) Which edges to relax  ·  (2) In what order / schedule  ·  (3) What graph to build.
The relaxation rule dist[v] = min(dist[v], dist[u] + w) never varies.
Layer 1 — Base Algorithm
Bellman-Ford
Applies the relaxation rule to every edge, in any fixed order, for exactly n−1 rounds. Zero scheduling intelligence — but provably correct on any graph without a negative cycle. This is the master template.

Why SPFA is not a different algorithm click to expand

In the BF master template, the outer for (iter = 0; iter < n-1; iter++) loop drives n−1 exhaustive edge scans. SPFA replaces this with a queue. But crucially, the SPFA loop preserves the same layer structure: at the start of each outer iteration, the queue holds exactly the vertices that improved in the previous layer — analogous to BF's "frontier" after one round. The inner loop drains the entire current layer before advancing. The round counter n-- counts layers exactly as BF counts rounds. The relaxation line inside is byte-for-byte identical. SPFA is BF with a smarter scheduling policy that skips useless scans.

1. The Shortest Path Problem BASE ALGORITHM

Given a directed weighted graph G = (V, E) and a source vertex s, find the minimum-weight path from s to every reachable vertex. Weights can be positive, zero, or negative. One condition must hold: no negative-weight cycle may be reachable from s.

Why? Consider cycle A → B(−1) → A with total weight −1. After one traversal dist[A] = −1, after two dist[A] = −2, after k traversals dist[A] = −k. The question "what is the shortest path?" has no finite answer. The algorithm must detect and report this condition.

📌

Simple vs shortest path: When no negative cycle exists, every shortest path is also simple (visiting no vertex twice), because any cycle either adds nothing or adds positive cost. This equivalence is why n−1 iterations suffice — n−1 is the maximum edge count of a simple path in an n-vertex graph.

2. Negative Edges — Why Dijkstra Fails BASE ALGORITHM

Dijkstra locks each vertex once it is selected as the minimum-distance unvisited vertex. This greedy lock is valid only when all edge weights are non-negative — adding more edges can only increase or maintain path length. A negative edge breaks this monotonicity: a cheaper path can arrive after Dijkstra has already locked the destination.

dist[v] is non-increasing across all relaxation operations — it can only decrease or stay the same. This is the monotonicity property that makes the entire BF family safe.
✗

Dijkstra failure example: A→B(+5), B→C(−7), A→C(+10). Dijkstra locks C with cost 10 before exploring B. Then B→C gives 5+(−7) = −2, but C is already locked — Dijkstra returns 10 instead of −2.

Bellman-Ford's minimal fix: remove locking entirely. Every edge is re-examined every round. Cost: O(nm) instead of O((n+m) log n). Gain: correctness on any weight sign.

🔑

Key transition: The rest of this tutorial asks one question — given that we must re-examine edges, how often and in what order? BF answers: every edge, every round, n−1 rounds. SPFA answers: only edges from recently-improved vertices, one queue layer at a time. Same invariant, different scheduling.

3. Relaxation — The One Rule BASE ALGORITHM

dist[v] = min(dist[v], dist[u] + w(u, v))

The distance invariant: dist[v] is never an overestimate. It starts at ∞ (trivially safe) and can only decrease. After all necessary relaxations have fired, dist[v] equals the true shortest distance. This invariant is maintained by every algorithm in the family — BF, SPFA, and every application — because they all use the same relaxation step.

This is the same operation used in Dijkstra, Floyd-Warshall, and SPFA. The algorithms differ only on three axes: which edges they relax, in what order, and on what graph.

Experiment: Relax edges in different orders. You can always reach the correct answer, but some orderings need more passes. This is why BF scans all edges n−1 times — it handles the worst-case ordering where edges are scanned in reverse propagation order.

📌

Convergence condition: BF converges when a complete scan of all edges produces zero relaxations. SPFA converges when the queue empties before the n-th layer. Both express the same condition: no vertex has new information to propagate.

4. DP Interpretation BASE ALGORITHM

dp[k][v] = minimum weight of any path from s to v using at most k edges

Recurrence:

dp[0][s] = 0;          // base: zero edges to reach source
dp[0][v] = ∞ ∀ v ≠ s  // can't reach anyone else with 0 edges

dp[k][v] = min(
    dp[k-1][v],                            // don't use the k-th edge
    min over all (u,v,w): dp[k-1][u] + w   // extend best (k-1)-edge path
)

Since any simple path in an n-vertex graph uses at most n−1 edges, k = 1…n−1 suffices. This is exactly what the outer loop of Bellman-Ford computes.

📌

Frontier propagation invariant: After k complete BF iterations, dist[v] is correct for every vertex v whose shortest path uses at most k edges. This invariant bridges directly to the "why n−1" argument in §6.

Why the 1D in-place version is correct — and faster

Using a single dist[] array means that when we relax edge u → v in iteration k, we may use dist[u] already improved earlier in the same iteration. This is strictly better — we propagate information faster. Formally: iterations₁D ≤ iterations₂D ≤ n−1. The 1D version is correct and is a performance improvement over strict DP.

5. Correctness Proof BASE ALGORITHM

The proof is a domino chain: the source falls first, then each node falls as soon as the node before it on its shortest path falls.

Inductive proof — step through the domino chain
Step 1 / 8
Press Next to walk through the inductive proof.

Claim: After iteration k (1D in-place), for every vertex v reachable from s by a path with at most k edges, dist[v] equals the optimal path weight.

Base case (k = 0): dist[s] = 0 is correct. All others are ∞ (never an underestimate).

Inductive step: Assume after iteration k−1, dist[v] is correct for all v whose shortest path uses ≤ k−1 edges. Let v have an optimal k-edge path, last edge (u, v, w). By the 1D in-place guarantee, dist[u] is already correct by the time we examine (u, v) in iteration k. The relaxation fires, setting dist[v] to its optimal value. ∎

Correctness in both directions: dist[v] never underestimates (every decrease requires an improving path) and never overestimates after n−1 passes (inductive step guarantees it reaches the correct value).

Extended formal proof click to expand

Why simple paths suffice: Any simple path uses at most n−1 edges. If a shortest path contained a zero-weight cycle, remove it — same cost. If it contained a positive cycle, remove it — cheaper. Therefore optimal paths are always simple, and n−1 iterations always suffice (unless a negative cycle exists).

6. Why n−1 Iterations Is a Tight Bound BASE ALGORITHM

The bound n−1 is tight. Consider a chain graph 0 → 1 → 2 → … → (n−1), with edges presented in reverse order (edge (n−2)→(n−1) first, down to 0→1 last). In iteration 1, only edge 0→1 fires. In iteration k, only node k is settled. Exactly n−1 iterations are required.

⚠

Connection to negative cycles: If a negative cycle exists, distances never stabilise. The n-th iteration catches it: any relaxation that fires after n−1 complete rounds proves distances can still decrease, which is only possible through a negative cycle.

7. Negative Cycle Detection BASE ALGORITHM

The theorem

After n−1 iterations, the frontier propagation invariant guarantees that all shortest paths using at most n−1 edges are correct. A simple path uses at most n−1 edges. Therefore if any relaxation fires in a hypothetical n-th iteration, the improving walk must use ≥ n edges — meaning it revisits at least one vertex — meaning it contains a cycle. Since removing a zero or positive-weight cycle only helps, a cycle that enables further improvement must have negative total weight.

⚠

Reachability nuance: Running BF from a single fixed source only detects negative cycles reachable from that source. To detect any negative cycle in the entire graph, add a super-source with 0-weight edges to all vertices and run BF from it. This is used in Wormholes (§20) and Difference Constraints (§17).

Detection code

// After the standard n-1 loop — append this:
for (const auto& e : edges) {
    if (dist[e.u] < INF && dist[e.u] + e.w < dist[e.v])
        return false;   // negative cycle detected
}
return true;

What this detection does NOT tell you

It tells you a negative cycle exists somewhere reachable from the source. It does not identify which cycle or which vertices are affected. For problems requiring you to output the cycle's vertices, record predecessors and trace back from the vertex that triggered the n-th relaxation.

8. Complexity Analysis BASE ALGORITHM

Time: O(n · m)

O(n · m)

n−1 rounds × m edges per round × O(1) per relaxation. O(nm) is a direct consequence of zero scheduling intelligence. For sparse graphs (m = O(n)): O(n²). For dense graphs (m = O(n²)): O(n³).

O(nm) = O(n rounds) × O(m edges/round)
SPFA reduces effective rounds — see §15

Space: O(n + m)

O(n + m)

O(n) for dist[], O(m) for the edge list. No adjacency matrix required. The 1D optimisation avoids the O(n²) DP table.

Practical note: BF with early termination is often fast on contest inputs — most graphs converge well before n−1 rounds. The O(nm) worst case requires the adversarial chain scenario from §6.

Master Template — Bellman-Ford BASE ALGORITHM

This is the canonical reference for the entire tutorial. Every subsequent variant is derived from this template by changing exactly one thing. Each comment maps back to §3–7.

📌

Robust general-purpose baseline: This template is written with long long distances and a large INF = 1e18 precisely because it serves as a reusable master baseline safe across all problem sizes and weight ranges. When adapting for a specific contest problem you may narrow types as needed — but this version is the one to reach for when in doubt.

// ═══ BF MASTER TEMPLATE ════════════════════════════════════
// Input : n vertices (0-indexed), edge list, source src
// Output: dist[v] = shortest dist from src (INF = unreachable)
// Return: true  = no negative cycle reachable from src
//         false = negative cycle detected
// Complexity: O(nm) time, O(n+m) space
// Template slots:
//   EDGE_SET  — what edges exist (given, or constructed)
//   SCHEDULE  — all edges each round  (→ SPFA: layered queue)
//   STOP_COND — n-1 rounds            (→ SPFA: queue empties before nth layer)
//   QUERY     — dist[] values or cycle detection boolean
// ════════════════════════════════════════════════════════════

struct Edge { int u, v; long long w; };
const long long INF = 1e18;

bool bellmanFord(int n, const vector<Edge>& edges,   // EDGE_SET
                 int src, vector<long long>& dist) {

    dist.assign(n, INF);
    dist[src] = 0;          // dp[0][src] = 0  ← base case

    // SCHEDULE: n-1 rounds over all edges
    for (int iter = 0; iter < n - 1; iter++) {         // STOP_COND
        bool changed = false;
        for (const auto& e : edges) {
            // Guard: never relax through unreachable node (INF + w overflows)
            if (dist[e.u] < INF && dist[e.u] + e.w < dist[e.v]) {
                dist[e.v] = dist[e.u] + e.w;   // THE relaxation rule
                changed = true;
            }
        }
        if (!changed) break;    // early exit: converged before n-1 rounds
    }

    // QUERY: n-th pass = negative cycle detection (§7 theorem)
    for (const auto& e : edges) {
        if (dist[e.u] < INF && dist[e.u] + e.w < dist[e.v])
            return false;       // negative cycle — dist[] invalid for some v
    }
    return true;
}
📌

What to derive from this template: SPFA (§12) changes the SCHEDULE slot only — the outer loop becomes a layered queue. Difference constraints (§17) change the EDGE_SET slot only. Wormholes (§20) add a super-source. Every application in §17–22 is a combination of these two changes — nothing else ever changes.

Line-by-line explanation click to expand

dist.assign(n, INF); dist[src] = 0; — DP base case: k=0, source is 0, all others ∞.

for iter = 0 .. n−2 — n−1 outer rounds. Each round advances the frontier propagation invariant by one hop.

if (dist[e.u] < INF) — critical guard. Without this, INF + (negative w) wraps around and produces phantom relaxations.

if (!changed) break; — 1D convergence is always ≤ n−1 rounds, often much less.

n-th iteration check — any relaxation proves a negative cycle (§7 theorem).

12. SPFA — Bellman-Ford with Layered-Round Scheduling SCHEDULING VARIANT

Context → Mapping: Bellman-Ford's outer loop runs n−1 full edge scans, even when most vertices have not changed since the last round. SPFA eliminates this waste by replacing the fixed outer loop with a queue that tracks only the vertices whose distances just improved — the "active frontier." The queue is processed in discrete layers: each outer iteration drains the entire current layer (one compressed BF round) and collects newly-improved vertices for the next layer.

What stays the same: The relaxation rule if (dist[v] > dist[u] + w) dist[v] = dist[u] + w is byte-for-byte identical to BF. The meaning of dist[] is identical. Any vertex whose distance improves is guaranteed to be re-queued and re-examined — no useful relaxation is ever permanently skipped.

What changes: The outer SCHEDULE changes from "scan all m edges in a fixed order, n−1 times" to "process only active vertices layer by layer, up to n layers." Concretely: the outer for (iter = 0; iter < n-1; iter++) loop is replaced by while (q.size() && n--); the inner edge scan is replaced by draining the current queue layer via int s = q.size(); while (s--). A boolean inQueue[] flag prevents duplicates within a layer.

Why correctness holds: BF fact: after k full rounds, all shortest paths using at most k edges are correct (frontier propagation invariant, §4). SPFA preserves this invariant layer by layer — each outer iteration corresponds to exactly one compressed BF round, because the queue at the start of each iteration holds precisely the vertices that improved in the previous round. The counter n-- counts layers exactly as BF counts rounds. Therefore: if a relaxation still fires when n reaches 0 (the nth layer), some improving walk uses at least n edges. An n-edge walk in an n-vertex graph must revisit a vertex, forming a cycle. Because the distance improved through that cycle, the cycle must have negative total weight. This is the nth-layer negative-cycle proof — structurally identical to BF's nth-iteration proof.

Code delta from Bellman-Ford:

// ─── CHANGED: SCHEDULE slot only ─────────────────────────────
// BF outer loop:
//   for (int iter = 0; iter < n-1; iter++) {
//       for (const auto& e : edges) { ... }
//   }
// Replaced by layered queue:

bool sp(const vector<vector<pair<int,int>>>& adj, int src) {
    int n = (int)adj.size();        // infer n — no explicit parameter
    vector<int> dist(n, 1e9);
    dist[src] = 0;
    queue<int> q;
    q.push(src);
    vector<bool> inQueue(n);
    inQueue[src] = true;

    while (q.size() && n--) {      // outer: n-- counts BF rounds
        int s = q.size();           // freeze current layer size
        while (s--) {               // drain entire current layer
            int u = q.front(); q.pop();
            inQueue[u] = false;
            for (auto [v, w] : adj[u]) {
                int d = dist[u] + w;
                if (dist[v] > d) {  // same relaxation rule as BF
                    dist[v] = d;
                    if (!inQueue[v]) { inQueue[v] = true; q.push(v); }
                    if (!n) return false;  // nth-layer = neg cycle
                }
            }
        }
    }
    return true;  // queue emptied before nth layer: no neg cycle
}
// ─── UNCHANGED: relaxation rule, dist[] semantics, neg-cycle logic

Common pitfall: Processing the queue in arbitrary order instead of layer by layer. The nth-layer negative-cycle proof relies on each outer iteration corresponding to exactly one compressed BF round. If vertices from multiple rounds are mixed in the same iteration, the layer count no longer tracks the edge-hop bound, and the proof breaks. If you change the scheduling semantics, you must also revisit the correctness proof.

Master Template — SPFA SCHEDULING VARIANT

This is the second canonical template in this tutorial. Every application that uses SPFA instead of BF is derived from this template by changing the EDGE_SET slot only — the SCHEDULE loop is never touched. Parity with §9: same invariant, same correctness, different scheduling.

📌

Accepted compact contest style — intentionally preserved: This template uses int distances and 1e9 as INF. This is a deliberate presentation choice, not an algorithmic inconsistency with §9. The BF master template is written in a robust long-long baseline form for safe generic reuse. This SPFA template is kept in its accepted compact contest form to leave the exact layered-round scheduling structure and negative-cycle proof unchanged and easy to read. Same family, same correctness backbone, different code presentation goals.

Invariant

After the queue layer for round k is fully drained, every shortest path using at most k edges is correct in dist[].

This is the same frontier propagation invariant as Bellman-Ford (§4), preserved by the layered structure of the queue. Layer k of the queue holds exactly the vertices that improved in layer k−1 — so draining layer k advances the frontier by one hop, identically to BF's k-th round.

Correctness idea

No useful relaxation is ever permanently skipped. Any vertex v whose distance improves acquires new information to propagate. Because v is pushed to the queue, its outgoing edges will be examined in the next layer. The invariant therefore progresses layer by layer, and after at most n−1 layers all shortest paths are correct — provided no negative cycle exists.

Negative-cycle detection: If the counter n-- reaches 0 and a relaxation still fires (if (!n) return false), some improving walk uses ≥ n edges. An n-edge walk in an n-vertex graph must revisit a vertex, forming a cycle. Because the distance improved, that cycle is negative. This is the nth-layer proof — structurally identical to BF's nth-iteration proof (§7).

Template

// ═══ SPFA MASTER TEMPLATE ══════════════════════════════════
// Input : adj[u] = list of {v, w}, source src
// Output: dist[v] = shortest dist from src (1e9 = unreachable)
// Return: true  = no negative cycle reachable from src
//         false = negative cycle detected (nth-layer proof)
// Complexity: avg O(km) k≈2, worst O(nm) same as BF
// Template slots:
//   EDGE_SET  — adj[][] (given, or constructed from constraints)
//   SCHEDULE  — layered queue  (BF uses fixed round loop instead)
//   STOP_COND — queue empty before nth layer
//   QUERY     — dist[] values or cycle detection boolean
// ════════════════════════════════════════════════════════════

bool sp(const vector<vector<pair<int,int>>>& adj, int src) {
    int n = (int)adj.size();               // infer n from graph

    vector<int> dist(n, 1e9);
    dist[src] = 0;                         // same base case as BF

    queue<int> q;
    q.push(src);
    vector<bool> inQueue(n);
    inQueue[src] = true;

    // SCHEDULE: layered queue — each outer iteration = one BF round
    while (q.size() && n--) {             // STOP_COND: n-- counts rounds
        int s = q.size();                  // freeze current layer size
        while (s--) {                      // drain entire current layer
            int u = q.front();
            q.pop();
            inQueue[u] = false;
            for (auto [v, w] : adj[u]) {
                int d = dist[u] + w;
                if (dist[v] > d) {         // same relaxation rule as BF
                    dist[v] = d;
                    if (!inQueue[v]) {
                        inQueue[v] = true;
                        q.push(v);         // push to NEXT layer
                    }
                    if (!n) return false;  // QUERY: nth-layer = neg cycle
                }
            }
        }
    }
    return true;   // STOP_COND met: queue empty before nth layer
}

Complexity

Avg: O(k · m), k ≈ 2

On random or natural graphs, each vertex enters the queue on average about 2 times — roughly O(2m) = O(m). The layered schedule skips all edges whose source hasn't improved, compressing many BF rounds into zero-cost passes.

Worst: O(n · m) — same as BF

Adversarially constructed graphs can force every vertex into the queue O(n) times. This matches BF's worst case exactly. Anti-SPFA hacks on competitive judges exploit this. No theoretical improvement over BF in the worst case.

✓

Reading this template as BF (§9): while (q.size() && n--) = BF's outer round loop. int s = q.size(); while (s--) = BF's inner edge scan, restricted to active vertices only. if (!n) return false = BF's n-th iteration negative-cycle check. Every structural element of BF has a direct counterpart here.

📌

Why if (!n) return false lives inside the relaxation block: When n-- decrements n to 0, the outer while condition will fail on its next evaluation — but a relaxation could fire before that evaluation completes the current layer. The guard catches this immediately, returning false the instant the first nth-layer improvement is detected rather than waiting for the outer loop to exit.

BF master template (§9) — SCHEDULE slot
for (int iter = 0; iter < n-1; iter++) {
    bool changed = false;
    for (const auto& e : edges) {
        if (dist[e.u] < INF &&
            dist[e.u]+e.w < dist[e.v]) {
            dist[e.v] = dist[e.u]+e.w;
            changed = true;
        }
    }
    if (!changed) break;
}
SPFA replacement — layered queue SCHEDULE
while (q.size() && n--) {  // outer = round counter
    int s = q.size();       // layer size
    while (s--) {           // drain layer
        int u = q.front(); q.pop();
        inQueue[u] = false;
        for (auto [v, w] : adj[u]) {
            int d = dist[u] + w;
            if (dist[v] > d) {
                dist[v] = d;
                if (!inQueue[v]) {
                    inQueue[v] = true;
                    q.push(v);  // next layer
                }
                if (!n) return false; // nth layer
            }
        }
    }
}

15. SPFA Complexity — When to Switch from BF SCHEDULING VARIANT

Average case (practice)

O(k · m), k ≈ 2

On random or natural graphs, each vertex enters the queue on average about 2 times — roughly O(2m) = O(m). Dramatically faster than BF's O(nm) on typical contest inputs.

Worst case (theory)

O(n · m) — same as BF

Adversarially constructed graphs can force the queue to process every vertex O(n) times. Anti-SPFA hacks exist on competitive judges targeting this. SPFA provides no worst-case improvement over BF.

17. Difference Constraints GRAPH CONSTRUCTION

This variant is derived from the master template in §9 by changing the EDGE_SET slot only. The BF loop, relaxation rule, and negative-cycle detection are completely unchanged.

Interactive Demo — Inequality to Edge

Every constraint x_j − x_i ≤ c becomes edge i → j with weight c. Add an infeasible constraint to see how a negative cycle appears.

Inequalities
Constructed edges

Context → Mapping: Given a system of linear inequalities x_j − x_i ≤ c_ij, determine whether a simultaneous solution exists and find one if so. Model each variable as a vertex; each inequality x_j − x_i ≤ c_ij becomes a directed edge i → j with weight c_ij. Add a super-source vertex s with 0-weight edges to every variable vertex (ensures all variables are reachable and establishes 0 as the reference baseline).

What stays the same: The BF master template runs without modification. It has no idea it is "solving inequalities" — it just finds shortest paths on the constructed graph.

What changes: The EDGE_SET is constructed from the inequality system, not given directly. A super-source node (index n) is added with 0-weight edges to all other vertices.

Why correctness holds: Shortest-path distances always satisfy the triangle inequality: dist[j] ≤ dist[i] + w(i→j). This translates directly to x_j − x_i ≤ c_ij. If a negative cycle exists in the constructed graph, BF's n-th iteration fires, signalling that no simultaneous solution exists.

Code delta from §9 master template:

// CHANGED: EDGE_SET — construct from inequalities
// For each inequality x_j - x_i <= c:
+ edges.push_back({i, j, (long long)c});

// CHANGED: add super-source (index n)
+ for (int i = 0; i < n; i++)
+     edges.push_back({n, i, 0LL});

// UNCHANGED: run BF from super-source
bool feasible = bellmanFord(n + 1, edges, n, dist);
// feasible == true  → dist[i] is a valid assignment for x_i
// feasible == false → system is infeasible (negative cycle)

Common pitfall: Forgetting the super-source. Without it, some variables have no path from the source, leaving them at INF. Also: x_j − x_i ≤ c always becomes edge i → j (never j → i).

Practice: POJ 1201 — Intervals — canonical difference-constraints modeling problem.

20. Wormholes (UVA 558) GRAPH CONSTRUCTION

Context → Mapping: Planets are vertices; wormholes are directed edges (negative weight = travels backward in time). The question "can you return to the past?" asks whether any negative-weight cycle is reachable from any starting point. Add a super-source with 0-weight edges to all vertices and run BF from it.

What stays the same: Standard BF negative-cycle detection (n-th iteration). Completely unchanged from §9.

What changes: A super-source is added so all vertices are reachable. The dist[] output is discarded — only the boolean return value matters.

Why correctness holds: The super-source makes every vertex reachable from the BF starting point. Any negative cycle in the original graph is reachable from the super-source. The n-th iteration fires and BF returns false.

Code delta from §9 master template:

// CHANGED: EDGE_SET — add super-source
+ for (int i = 0; i < n; i++)
+     edges.push_back({n, i, 0LL});

// CHANGED: QUERY — only boolean matters, dist[] discarded
bool hasNegCycle = !bellmanFord(n + 1, edges, n, dist);
+ cout << (hasNegCycle ? "possible\n" : "not possible\n");

Common pitfall: Running BF from only one fixed source (e.g., vertex 0). A negative cycle in a disconnected component unreachable from 0 will be missed. The super-source trick is essential for global detection.

Practice: UVA 558 — Wormhole → pure global negative-cycle detection; add a super-source and return the boolean only.

21. King Problem (UVA 515) GRAPH CONSTRUCTION

Context → Mapping: Constraints "person A is between c and d years older than B" (c ≤ x_A − x_B ≤ d) decompose into two difference constraints: x_A − x_B ≤ d (edge B → A, weight d) and x_B − x_A ≤ −c (edge A → B, weight −c). Check feasibility after each new constraint is added.

What stays the same: Difference-constraint graph construction from §17, BF feasibility check, super-source technique. Identical to §17 structurally.

What changes: Each constraint produces two directed edges instead of one. Constraints are added incrementally; BF is re-run after each addition.

Why correctness holds: Both inequality directions must be enforced simultaneously. Adding only one edge enforces the upper bound but not the lower bound — the reverse edge enforces the lower bound as an upper bound on the opposite difference.

Code delta from §17 difference-constraints template:

// CHANGED: EDGE_SET — two edges per bounded-difference constraint
// For c <= x_A - x_B <= d:
+ edges.push_back({B, A, (long long)d});    // x_A - x_B <= d
+ edges.push_back({A, B, (long long)(-c)}); // x_B - x_A <= -c
// UNCHANGED: super-source, BF, feasibility check

Common pitfall: Adding only one edge per constraint. The lower bound is enforced by the reverse edge — missing it allows infeasible assignments through.

Practice: UVA 515 — King → incremental bounded-difference constraints; add two edges per constraint and check feasibility after each addition.

22. Arbitrage & Scheduling GRAPH CONSTRUCTION

Context → Mapping (Arbitrage): Currency exchange rates are multiplicative. Replace each weight with −log(rate). A profitable cycle (product of rates > 1) becomes a negative-weight cycle (sum of −log values < 0), detectable by BF's n-th iteration.

What stays the same: BF negative-cycle detection with super-source (§20 technique).

What changes: Edge weights are −log(exchange_rate). Use floating-point (double) arithmetic.

Why correctness holds: log(r₁·r₂·r₃) = log(r₁)+log(r₂)+log(r₃). Negating: product > 1 ↔ sum of (−log) < 0.

Code delta from §20 master template:

// CHANGED: EDGE_SET — log-transform weights
+ edges.push_back({u, v, -log(rate)});
// Everything else: super-source + BF neg-cycle detection unchanged

Common pitfall: Using integer arithmetic on logarithms. Always use double and add an epsilon guard on the relaxation check.

Practice: UVA 104 — Arbitrage — direct log-transform + cycle-detection match for arbitrage.

Context → Mapping (Scheduling): Constraints "task B starts at least gap after A, at most deadline after A" become two difference constraints. Interpret dist[i] as the start time for task i.

What stays the same: Difference-constraint graph construction, BF feasibility check, two-edges-per-bound rule from §21.

What changes: Interpretation: dist[i] = start time for task i. Constraints express scheduling windows rather than age differences.

Why correctness holds: BF output satisfies all triangle inequalities, which directly encode all scheduling constraints.

Code delta from §21 King template:

// CHANGED: EDGE_SET — scheduling constraint windows
// "B starts at least gap after A": x_B - x_A >= gap
+ edges.push_back({A, B, (long long)(-gap)});
// "B starts at most deadline after A": x_B - x_A <= deadline
+ edges.push_back({A, B, (long long)deadline});

// CHANGED: QUERY — interpret dist[] as start times
if (bellmanFord(n+1, edges, n, dist))
+     for (int i = 0; i < n; i++) start_time[i] = dist[i];

Common pitfall: Mixing up directions when translating constraints. Always: x_j − x_i ≤ c becomes edge i → j with weight c.

The Six Things You Must Know

①
The creative work is graph construction
BF and SPFA are commodities. The question in any new problem is always: can I model the constraints as directed weighted edges such that BF's output answers my question? If yes, apply the template. The algorithm itself never changes.
②
One relaxation rule governs everything
dist[v] = min(dist[v], dist[u]+w) is the only operation this family performs. Every algorithm is a scheduling policy for deciding which (u,v,w) to try and when. The operation itself is invariant.
③
BF = exhaustive schedule, n−1 rounds
Every edge, fixed order, n−1 times. Zero scheduling intelligence — but the frontier propagation invariant guarantees correctness. After k rounds, all ≤k-edge shortest paths are correct.
④
SPFA = layered-round queue scheduling
Replace BF's outer loop with while (q.size() && n--) + inner layer drain. Each outer iteration = one compressed BF round. Only active vertices processed. Same relaxation, same nth-round negative-cycle proof.
⑤
Negative cycle = "distances decrease forever"
If any relaxation fires in the nth round/layer, some improving walk uses ≥n edges — implies a repeated vertex — implies a cycle — and because the distance improved, the cycle is negative. Same proof for BF and SPFA.
⑥
Difference constraints = triangle inequality as solution
x_j − x_i ≤ c becomes edge i→j weight c. Add super-source. BF outputs shortest distances that satisfy all triangle inequalities — exactly your constraints. Negative cycle = infeasible system.
