Shortest Path Problem

ICPC-ready decision framework: frontier expansion, relaxation-based reasoning, and instant algorithm selection.
Theme

🧠 The Frontier Expansion Mental Model

Shortest path algorithms are frontier expansion processes. Imagine a wave spreading outward from the source vertex, discovering vertices in order of their distance. This is the unifying metaphor across all algorithms.

Frontier Expansion: The Core Process

You begin at the source with one certainty: dist[source] = 0. Everything else is unknown (∞). The frontier is the boundary between explored territory (where distances are known or estimated) and unexplored territory.

As you expand the frontier, you propagate distance information through edges. Each edge (u,v) with weight w says: "If you've reached u with distance d, I can help you reach v with distance d+w." You update your estimate: dist[v] = min(dist[v], dist[u] + w).

This propagation continues until the frontier has expanded to cover all reachable vertices. Algorithms differ in how they choose which part of the frontier to expand next.

Start: source at distance 0
Expand frontier outward
Propagate via edge relaxation
End: all reachable vertices discovered

How Algorithms Differ: Frontier Selection Strategy

BFS: Expands frontier level by level. All vertices at distance k discovered before any at distance k+1. Frontier is a wave moving uniformly outward.

0-1 BFS: Expands frontier with priority: free edges (weight 0) processed immediately, costly edges (weight 1) deferred. Frontier has two levels at most.

Dijkstra: Greedily expands closest vertex on frontier. Always chooses minimum-distance unexplored vertex. Frontier shrinks inward from all directions.

Bellman-Ford: Exhaustively relaxes all edges V−1 times. No greedy choice — frontier expands in all directions simultaneously each round.

DAG Relaxation: Expands frontier in topological order. Process vertices such that all predecessors are handled before successors. One-pass frontier sweep.

Floyd-Warshall: Expands frontiers between all pairs simultaneously, allowing intermediate vertices k=0,1,...,V−1 progressively.

⚙️ Relaxation: The Unifying Framework

All shortest path algorithms differ only in how they schedule relaxations. This is the central insight that unifies the entire topic.

The Relaxation Operation

For edge (u,v) with weight w, relaxation attempts to improve the distance to v by routing through u:

dist[v] ← min(dist[v], dist[u] + w)

If the path source→u→v (with cost dist[u]+w) is cheaper than the current best path to v (dist[v]), we update. Otherwise, we keep the existing estimate.

This is frontier expansion in action: we're propagating distance information from u (inside the explored frontier) to v (on or beyond the frontier).

All Algorithms Through the Relaxation Lens

BFS — Relax Edges Level-by-Level

Relaxation pattern: Process vertices in queue order (FIFO). When processing u, relax all edges (u,v).

for each edge (u,v): dist[v] ← min(dist[v], dist[u] + 1)

Key insight: Each edge relaxed exactly once. In unweighted graphs, first discovery = shortest path.

Frontier expansion: Wave expands uniformly, one level at a time.

0-1 BFS — Relax with Deque Priority

Relaxation pattern: Use deque. If edge has weight 0, relax and push to front (process immediately). If weight 1, relax and push to back (defer).

if w=0: dist[v] ← min(dist[v], dist[u]), push_front(v)
if w=1: dist[v] ← min(dist[v], dist[u]+1), push_back(v)

Key insight: Deque maintains two-level frontier: free edges expand frontier immediately, costly edges wait.

Frontier expansion: Free edges create instant jumps; costly edges create measured steps.

Dijkstra — Relax Greedily by Minimum Distance

Relaxation pattern: Extract vertex u with minimum dist[u] from priority queue. Relax all edges (u,v).

u ← extractMin(PQ)
for each edge (u,v): dist[v] ← min(dist[v], dist[u] + w)

Key insight: Each vertex extracted once. Each edge relaxed once (when source vertex extracted).

Frontier expansion: Greedy selection shrinks frontier inward from closest points.

Bellman-Ford — Relax All Edges V−1 Times

Relaxation pattern: Make V−1 passes over entire edge list. In each pass, relax every edge.

for i=1 to V−1:
  for each edge (u,v):
    dist[v] ← min(dist[v], dist[u] + w)

Key insight: Each edge relaxed V−1 times. After k passes, all k-edge paths correctly computed.

Frontier expansion: Exhaustive expansion in all directions each round.

DAG Shortest Path — Relax in Topological Order

Relaxation pattern: Compute topological sort. Process vertices in that order, relaxing outgoing edges.

topo_order ← topological_sort(G)
for u in topo_order:
  for each edge (u,v):
    dist[v] ← min(dist[v], dist[u] + w)

Key insight: Each edge relaxed exactly once, in correct order. No cycles mean no re-examination needed.

Frontier expansion: One-pass sweep following dependency order.

Floyd-Warshall — Relax via Intermediate Vertices

Relaxation pattern: For each intermediate vertex k, relax all pairs (i,j) via k.

for k=0 to V−1:
  for i=0 to V−1:
    for j=0 to V−1:
      dist[i][j] ← min(dist[i][j], dist[i][k] + dist[k][j])

Key insight: Each pair relaxed V times (once per intermediate vertex). DP builds paths incrementally.

Frontier expansion: All-pairs frontiers expand simultaneously through shared intermediates.

Why Relaxation Unifies Everything

By recognizing relaxation as the core operation, you can: (1) Debug any shortest path implementation by checking if relaxations are correct, (2) Understand new algorithms by identifying their relaxation pattern, (3) Optimize by reducing unnecessary relaxations, (4) See the deep similarity between seemingly different approaches.

🔒 Core Invariants (Why Algorithms Work)

An invariant is a property that remains true throughout execution. Understanding invariants is the key to proving correctness and debugging failures.

BFS Invariant

Property: First visit = shortest path

What stays true: The queue contains vertices at distances d and d+1 only. When vertex v enters the queue, dist[v] is final.

Why it guarantees correctness: All edges have weight 1. If v is discovered at distance d, any other path to v must have ≥d edges (since it would need to traverse the same levels). BFS discovers vertices in increasing order of edge count = shortest path in unweighted graphs.

How to verify: Check that queue always has at most 2 distinct distance values. If a vertex at distance d+2 appears before all d+1 vertices are processed, invariant broken.

0-1 BFS Invariant

Property: Deque maintains non-decreasing distances

What stays true: All vertices in deque have distances that are non-decreasing from front to back. At any moment, deque contains at most 2 distinct distance values.

Why it guarantees correctness: Processing from front always extracts minimum-distance vertex among unprocessed. Weight-0 edges don't increase distance, so front insertion maintains order. Weight-1 edges increase distance by exactly 1, so back insertion maintains two-level structure.

How to verify: After each relaxation, check distances in deque are non-decreasing. If dist[front] > dist[back], invariant broken.

Dijkstra's Invariant

Property: Extracted vertex is final

What stays true: For every vertex u extracted from priority queue, dist[u] is the true shortest distance from source. No future relaxation will improve dist[u].

Why it guarantees correctness: Suppose dist[u] is not final when u is extracted. Then there exists a better path P from source to u. Let (x,y) be the first edge in P where y is not yet extracted. We would have relaxed (x,y), setting dist[y] ≤ dist[x] + w(x,y) < dist[u]. But u was extracted before y, so dist[u] ≤ dist[y], contradiction. This proof requires all weights ≥ 0.

How to verify: If a vertex is re-extracted or its distance improves after extraction, invariant broken (happens with negative edges).

Bellman-Ford Invariant

Property: Round k = k-edge shortest paths

What stays true: After k relaxation rounds, dist[v] correctly represents the shortest path from source to v using at most k edges.

Why it guarantees correctness: Base case (k=0): only source has distance 0. Inductive step: if all (k−1)-edge paths are correct, relaxing all edges computes all k-edge paths (each k-edge path = (k−1)-edge path + one edge). After V−1 rounds, all simple paths (which have ≤V−1 edges) are correct. Round V detects negative cycles: if any distance improves, a path with V edges is better than a V−1-edge path, implying a cycle was traversed with negative total weight.

How to verify: After round k, construct any k-edge path and check dist matches. If round V improves any distance, negative cycle exists.

DAG Invariant

Property: Predecessors finalized first

What stays true: When processing vertex u in topological order, all vertices v that have edges v→u have already been processed. Therefore, dist[u] is final.

Why it guarantees correctness: Topological order ensures no backward edges. When u is processed, all possible incoming paths to u have been fully explored (their source vertices were processed earlier). One relaxation pass suffices. Handles negative edges because we never revisit a vertex — the one-pass finality removes the greedy assumption that breaks Dijkstra.

How to verify: Check topological sort is valid. If processing u before some predecessor v (where v→u exists), invariant broken.

Floyd-Warshall Invariant

Property: dist[i][j] via intermediates {0,...,k}

What stays true: After considering intermediate vertices {0,1,...,k}, dist[i][j] is the shortest path from i to j using only vertices in {0,1,...,k} as intermediates (endpoints i,j excluded from intermediate set).

Why it guarantees correctness: Either the shortest i→j path uses vertex k as an intermediate, or it doesn't. If it does, shortest path is dist[i][k] + dist[k][j] (both already correct from previous iteration). If it doesn't, shortest path is dist[i][j] from previous iteration. Taking min covers both cases. After considering all V vertices as intermediates, no restrictions remain — dist[i][j] is globally optimal.

How to verify: After iteration k, verify dist[i][j] is correct when restricted to intermediates ≤k. Check via manual path construction.

⚡ Shortest Path Decision Rules (ICPC Ready)

Use this during contests for instant algorithm selection. Check conditions top-to-bottom — first match wins.

🎯 Single-Source Shortest Path

IF unweighted (all edges weight 1)
→ BFS — O(V+E)
IF weights ∈ {0,1} only
→ 0-1 BFS — O(V+E)
IF graph is DAG (no cycles)
→ DAG Topological Relaxation — O(V+E), handles negative edges
IF all weights ≥ 0
→ Dijkstra — O((V+E)logV) sparse, O(V²) dense
IF negative edges exist
→ Bellman-Ford — O(V·E), detects neg cycles

🎯 All-Pairs Shortest Path

IF dense (E ≈ V²) OR small n (V ≤ 400)
→ Floyd-Warshall — O(V³)
IF sparse (E ≪ V²) AND may have negative edges
→ Johnson's Algorithm — O(V²logV + VE)
IF sparse AND all weights ≥ 0
→ Run Dijkstra V times — O(V(V+E)logV)

📐 Graph Density & Data Structure Selection

Density determines both storage choice and algorithm variant. Understand the thresholds.

Adjacency List vs Matrix: Precise Thresholds

Use Adjacency List when:

• E < V² / 2 (less than half-full)

• Need to iterate over neighbors efficiently

• Memory is constrained

Storage: O(V+E) — each edge stored once/twice

Edge check: O(deg(v)) — must scan list

Neighbor iteration: O(deg(v)) — optimal

Use Adjacency Matrix when:

• E > V² / 2 (more than half-full)

• Need O(1) edge existence queries

• Running Floyd-Warshall

Storage: O(V²) — always, wastes space if sparse

Edge check: O(1) — direct array access

Neighbor iteration: O(V) — must scan row

Floyd-Warshall vs Johnson's: When to Use Which

Complexity Analysis

Floyd-Warshall: O(V³) always, regardless of edge count

Johnson's: O(V² log V + VE) = O(V² log V) if E ≈ V, O(V³) if E ≈ V²

Crossover Point:

When E > V² / log V, Floyd-Warshall becomes competitive

For V=1000: if E > 100,000, Floyd-Warshall likely faster

Choose Floyd-Warshall when:

✓ V ≤ 400 (V³ ≤ 64 million operations)

✓ Graph is dense (E ≥ V² / 4)

✓ Implementation simplicity matters

✓ Need negative cycle detection

✓ Need transitive closure

Choose Johnson's when:

✓ Graph is sparse (E ≈ V or E ≈ V log V)

✓ V > 400 but E is small relative to V²

✓ May have negative edges

✓ Asymptotic optimality matters

✓ Can afford higher implementation complexity

Why these thresholds? Floyd-Warshall has simple operations (one comparison, one addition) that run fast in practice. Johnson's has overhead from Bellman-Ford reweighting and V Dijkstra runs. The crossover happens when the asymptotic advantage of Johnson's overcomes its constant-factor disadvantage.

Why the Greedy Invariant Breaks

Dijkstra's correctness proof relies on: "When vertex u is extracted with distance d, all paths to u have cost ≥d because all edges have weight ≥0."

With negative edges, this fails: A path that arrives later (via more hops) can have lower total cost if it uses a negative edge. In this case, the 2-edge path A→C→B (cost −5) beats the 1-edge path A→B (cost 2), even though it's discovered later.

Fundamental issue: Dijkstra commits to distances too early. Once a vertex is settled, it assumes no future path can improve it. Negative edges violate this assumption.

Dijkstra discovers the worse path first, commits to it, then ignores the better path discovered later.

Why Negative Cycles Break Shortest Path — Mathematical Proof

The Infinite Descent Problem

Graph: Vertices A, B, C. Edges: A→B(3), B→C(4), C→B(−8). Cycle B→C→B has weight 4+(−8)=−4. Source: A. Target: B.

Path Cost Analysis:

PathTraversalsTotal Cost
A→BNo cycle3
A→B→C→BCycle ×13 + (−4) = −1
A→B→(C→B)×2Cycle ×23 + 2(−4) = −5
A→B→(C→B)×3Cycle ×33 + 3(−4) = −9
A→B→(C→B)×kCycle ×k3 + k(−4) = 3 − 4k

Limit Analysis:

cost(k) = 3 − 4k
lim(k→∞) cost(k) = lim(k→∞) (3 − 4k) = −∞

For any proposed "shortest path" with cost C, we can find k large enough such that 3−4k < C. Therefore, no finite minimum exists.

Why Algorithms Cannot Solve This

Bellman-Ford's behavior: After V−1 iterations, dist[B] might be some negative value (e.g., −9). On the V-th iteration, Bellman-Ford would relax the cycle again and find an even better value (e.g., −13). This signals: "a negative cycle exists; shortest path is undefined."

Mathematical impossibility: The problem asks for min{S} where S = {3, −1, −5, −9, ...} is unbounded below. The infimum is −∞, not a real number. Shortest path is undefined.

Bellman-Ford Detection Mechanism

After round V−1: All simple paths (≤V−1 edges) correctly computed. If no negative cycles exist, distances are final.

Round V (detection round): If any distance still improves, this means a path with V edges is better than a path with V−1 edges. Since we can't use V edges without repeating a vertex (pigeonhole principle with V vertices), we must have traversed a cycle. If this cycle improved the distance, it must have negative weight.

Conclusion: If round V finds improvement → negative cycle exists → shortest path = −∞ (undefined).

⚠️ ICPC Competition Pitfalls

These mistakes cost points in contests. Know them before implementation.

Integer Overflow with INT_MAX Infinity
Problem: Using INT_MAX as ∞, then computing dist[u] + w overflows to negative. Fix: Use 1e18 (long long) or 1e9 (int), or check if (dist[u] != INF) before relaxing.
Unreachable Nodes Printed as Zero or Garbage
Problem: Forgetting that unreachable vertices remain at ∞. Fix: After algorithm, check if (dist[v] == INF) print "unreachable" or −1. Never print 0 or uninitialized values.
Multigraph: Wrong Weight in Adjacency Matrix
Problem: Multiple edges between (u,v) with different weights — which to keep in matrix[u][v]? Fix: For shortest path, matrix[u][v] = min(all parallel edge weights). Keeping max, sum, or last-seen gives wrong answers.
Self-Loop with Negative Weight
Problem: Edge u→u with w < 0 is a trivial negative cycle. Fix: If reachable from source, shortest path to u is −∞. Detect and report. Positive/zero self-loops are harmless (ignore them).
Dense Graph with Heap-Based Dijkstra
Problem: Using heap on E ≈ V² gives O(V² log V) > O(V²) of array version. Fix: For dense graphs, use array-based Dijkstra or switch to Floyd-Warshall for all-pairs.
0-1 BFS with Weight ≥ 2
Problem: 0-1 BFS only works when weights ∈ {0,1} exactly. Edge with weight 2 breaks deque invariant. Fix: Verify all weights are 0 or 1 before using 0-1 BFS. Otherwise use Dijkstra.
Floating-Point Precision in Edge Weights
Problem: Comparisons like dist[u] + w < dist[v] fail due to rounding errors. Fix: Use epsilon tolerance: dist[u] + w + EPS < dist[v] where EPS=1e-9. Or scale weights to integers.
Forgetting Negative Cycle Detection
Problem: Running Bellman-Ford or Floyd-Warshall without checking for negative cycles. Fix: Bellman-Ford: run V-th iteration, check if any dist improves. Floyd-Warshall: after completion, check if (dist[v][v] < 0) for any v.
DAG Relaxation Without Verifying Acyclic
Problem: Assuming graph is a DAG without verification. If cycle exists, topological sort fails or produces wrong order. Fix: Run cycle detection first, or verify topological sort succeeded (Kahn's algorithm will detect cycles).
Disconnected Graph Components
Problem: Single-source algorithms only reach source's component. Other components remain at ∞. Fix: Handle in output (print "unreachable") or run algorithm from multiple sources if problem requires.
Stale Entries in Priority Queue (Lazy Deletion)
Problem: After relaxing edge (u,v), old entry for v remains in PQ alongside new entry. Fix: Before processing vertex v from PQ, check if (extracted_dist != dist[v]) skip (stale entry). Only process if values match.
Wrong Parent Tracking for Path Reconstruction
Problem: Forgetting to update parent array during relaxation. Fix: When relaxing (u,v) improves dist[v], set parent[v] = u. Reconstruct path by backtracking from target to source via parent pointers.
Off-by-One in Bellman-Ford Rounds
Problem: Running V rounds instead of V−1 for main algorithm. Fix: Main algorithm needs exactly V−1 rounds. V-th round is ONLY for negative cycle detection, not part of normal computation.

🗺️ Progressive Learning Roadmap

Master shortest path algorithms in hierarchical order. Each level builds on previous understanding.

Level 1: Foundation — BFS

Why start here: BFS is the simplest shortest path algorithm and the purest expression of frontier expansion. Master the queue-based, level-by-level wave and the "first visit = shortest path" invariant. Every algorithm you learn later is a variation of this idea.

Master: Queue implementation, visited tracking, distance array, parent pointers for path reconstruction, multi-source BFS

Relaxation view: Each edge relaxed exactly once. Weight is always 1, so dist[v] = dist[u] + 1. FIFO order guarantees non-decreasing distance processing.

Practice: 15+ problems on: unweighted graphs, grid shortest paths, tree level problems, multi-source BFS

Time investment: 1–2 weeks until you can code BFS blind in under 5 minutes

Level 2: Binary Weights — 0-1 BFS

Why now: Natural next step from BFS. Same O(V+E) complexity but handles binary-weight graphs. Introduces deque-based frontier management — weight-0 edges expand the current level instantly, weight-1 edges create the next level.

Master: Deque operations (push_front / push_back), the two-level invariant, recognizing binary-weight patterns in grid problems (e.g., free moves vs costly moves)

Relaxation view: Same as BFS, but frontier has two priority levels managed by deque. Zero-cost relaxations go to front, unit-cost to back.

Practice: 10+ problems on: grid problems with free moves, obstacle toggling, binary-cost transformations

Time investment: 1 week. Builds directly on BFS intuition

Level 3: General Non-Negative — Dijkstra's Algorithm

Why now: The most common contest algorithm. Generalizes frontier expansion to arbitrary non-negative weights using a priority queue instead of a simple queue. The greedy invariant ("extracted vertex is final") is the key insight to internalize.

Master: Both heap-based O((V+E)logV) for sparse and array-based O(V²) for dense, lazy deletion pattern, early termination, edge cases with duplicate PQ entries

Relaxation view: Each vertex extracted once, each edge relaxed once at extraction time. Priority queue schedules relaxations in optimal order.

Practice: 20+ problems on: weighted graphs, routing, grid with costs, single-source to single-destination optimization

Time investment: 2–3 weeks. Goal: Dijkstra implementation + debugging in under 10 minutes

Level 4: Acyclic Graphs — DAG Shortest Path

Why now: DAGs allow a single-pass frontier sweep in topological order — O(V+E) with any edge weights, including negative. This is the first algorithm that handles negatives and shows that graph structure (acyclic) can compensate for harder weight constraints.

Master: Topological sort (both Kahn's and DFS-based), recognizing DAG structure, longest path via negation, PERT/CPM scheduling

Relaxation view: Each edge relaxed exactly once, in topological order. No re-examination needed because predecessors are always finalized first.

Practice: 10+ problems on: task scheduling, longest path in DAG, PERT charts, DP on DAGs

Time investment: 1 week. The algorithm itself is simple; focus on recognizing DAG applicability

Level 5: Negative Edges — Bellman-Ford

Why now: The general-purpose algorithm for graphs with negative edges and possible cycles. Drops all structural assumptions — no greedy choice, no acyclic requirement. Exhaustive V−1 rounds of relaxation guarantee correctness, and round V detects negative cycles.

Master: V−1 relaxation rounds, negative cycle detection (round V), SPFA optimization, difference constraints formulation

Relaxation view: Every edge relaxed V−1 times. Round k guarantees all k-edge paths are correct (inductive argument).

Practice: 15+ problems on: negative weight graphs, currency arbitrage, difference constraints, negative cycle detection

Time investment: 2 weeks. Understand deeply why Dijkstra fails and Bellman-Ford succeeds with negatives

Level 6: All-Pairs — Floyd-Warshall & Johnson's

Why last: All-pairs problems require computing shortest paths between every pair of vertices. Floyd-Warshall uses a DP approach: O(V³), simple 3-loop, ideal when V ≤ 400 or the graph is dense. Johnson's combines Bellman-Ford reweighting + V Dijkstra runs: O(V²logV + VE), better for sparse graphs with negative edges.

Master (Floyd): 3-loop DP structure (k outermost!), intermediate vertex reasoning, negative cycle detection on diagonal, path reconstruction, transitive closure

Master (Johnson): Reweighting technique, potential functions, why reweighting preserves shortest paths, when it asymptotically beats Floyd-Warshall

Relaxation view: Floyd relaxes all pairs via each intermediate vertex. Johnson reweights to eliminate negatives, then runs V Dijkstra relaxation passes.

Practice: 10+ problems on: small dense graphs (V ≤ 400), all-pairs queries, transitive closure, sparse all-pairs with negative edges

Time investment: 2 weeks total. Floyd-Warshall is straightforward; Johnson's requires understanding the reweighting idea deeply

🎓 Guided Consolidation for Offline Teaching

Designed for lecture recording. Reflective prompts and thought questions — no interaction needed.

📋 Core Concepts Recap

• Shortest path = frontier expansion process — wave propagates from source discovering vertices by distance

• All shortest path algorithms differ only in how they schedule relaxations — this is the unifying principle

• Relaxation dist[v] ← min(dist[v], dist[u]+w) is the core operation across all algorithms

• Each algorithm has an invariant that guarantees correctness: BFS (first visit=final), Dijkstra (extracted=final), Bellman-Ford (round k = k-edge paths)

• Dijkstra's greedy invariant breaks with negative edges — settled vertices can be improved later

• Negative cycles make shortest path undefined (−∞) — detectable but not solvable

• Graph density determines structure choice and algorithm variant — sparse→list, dense→matrix

Answer: BFS — O(V+E) optimal for unweighted

Grid 1000×1000, moving right/down costs 1, left/up costs 0. Shortest path (0,0)→(999,999).

Answer: 0-1 BFS — O(V+E), binary weights {0,1}

DAG, V=10,000, some edges negative, single-source to all.

Answer: DAG topological relaxation — O(V+E), handles negatives, no cycles
