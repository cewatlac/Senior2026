Depth-First Search

Interactive DFS for class presentation: traversal, recursion stack, connected components, directed cycle detection, flood fill, and contest-ready C++ code.
Theme

DFS intuition — go deep first, then backtrack

DFS keeps following one path as far as possible. When it gets stuck, it returns to the last unfinished node and continues from there. In recursion, the call stack acts like the DFS stack.

General graph: use visited[]
Traversal order depends on adjacency order
Disconnected graph: one DFS may miss nodes
Adjacency list DFS: O(n + m)

Core rule

In any graph with cycles, mark nodes as visited before exploring neighbors. Without that, DFS can loop forever.

Competitive-programming style

This page uses the same C++ style as your DFS notes: #include <bits/stdc++.h>, vector<vector<int>> adj, vector<char> vis, and helpers like dfsSize, dfs_tree, and 3-color cycle DFS.

Core Templates

These are the contest-ready DFS templates in the same C++ style used in the graphs DFS/BFS notes: #include <bits/stdc++.h>, vector<vector<int>> adj, vector<char> vis, and 0-indexed input.

Template A — General Graph DFS visited[] is mandatory 
#include <bits/stdc++.h>
using namespace std;

vector<vector<int>> adj;
vector<char> vis;

void dfs(int u){
    vis[u] = 1;
    for(int v : adj[u]){
        if(!vis[v]) dfs(v);
    }
}
Template B — Directed Rooted Tree no visited needed when the input is already a rooted tree 
vector<vector<int>> child;

void dfs_tree(int u){
    for(int v : child[u]){
        dfs_tree(v);
    }
}
Template C — Undirected Tree DFS skip the parent edge 
void dfs_tree(int u, int p){
    for(int v : adj[u]){
        if(v == p) continue;
        dfs_tree(v, u);
    }
}
Template D — Directed Cycle Detection 0 = WHITE, 1 = GRAY, 2 = BLACK 
vector<int> color;

bool dfs_cycle(int u){
    color[u] = 1; // GRAY
    for(int v : adj[u]){
        if(color[v] == 0){
            if(dfs_cycle(v)) return true;
        }else if(color[v] == 1){
            return true; // back edge
        }
    }
    color[u] = 2; // BLACK
    return false;
}
Iterative DFS use this if recursion depth is risky 
void dfs_iter(int src){
    vector<int> st;
    st.push_back(src);
    vis[src] = 1;

    while(!st.empty()){
        int u = st.back();
        st.pop_back();

        for(int v : adj[u]){
            if(!vis[v]){
                vis[v] = 1;
                st.push_back(v);
            }
        }
    }
}

Connected Components

A single DFS only covers one connected component. To solve a disconnected graph, loop over all vertices and start a new DFS whenever you find an unvisited node.

Directed Cycle Detection with 3 Colors

We assign each node one of three colors during DFS. WHITE = not yet visited. GRAY = currently on the recursion stack (we entered but haven't finished). BLACK = fully processed. If we ever find an edge from the current node to a GRAY neighbor, that is a back edge — it closes a loop back into an ancestor still on the stack. That means a cycle exists.

WHITE = unvisited GRAY = in recursion stack BLACK = finished

Graph with a cycle — back edge makes GRAY→GRAY

The key rule

// Inside dfs_cycle(u):
for(int v : adj[u]){
    if(color[v] == 0){           // WHITE → recurse
        if(dfs_cycle(v)) return true;
    } else if(color[v] == 1){    // GRAY → back edge = cycle!
        return true;
    }
    // BLACK → finished ancestor, safe to skip
}
color[u] = 2; // mark BLACK before returning

Problem Patterns from DFS

Before starting any problem: understand the abstract first — what does the problem require? What is the structure? How does DFS solve this problem? Each problem below has an Abstract explaining the idea before the code.

📋 General Steps Before Solving
1️⃣ Model
What are the nodes and edges in this problem?
2️⃣ Goal
What are we computing? count / check / trace?
3️⃣ DFS Role
Does DFS here count / verify / carry state?
4️⃣ Loop
Do we need an outer loop? (disconnected graph)

1) Is it a Tree?

Abstract: Given an undirected graph with n nodes and m edges, determine whether it forms a tree. A graph is a tree if and only if it has exactly n − 1 edges and is fully connected — one DFS from any node must visit all n nodes.

For an undirected graph: it is a tree iff m == n - 1 and the graph is connected. Use dfsCount(0) to verify connectivity.

2) Largest Connected Component

Abstract: Given a disconnected undirected graph, find the size of its largest connected component. A single DFS only covers one component, so you must restart from every unvisited node, track each component's size, and return the maximum.

Loop over all nodes. Whenever you find an unseen node, run dfsSize(i) and maximize the answer.

3) State Propagation on Trees

Abstract: Given a rooted tree where the answer at each node depends on the path from the root, DFS carries extra state downward (e.g. parity, XOR accumulator, depth, ancestor constraints) so each recursive call has the full context it needs to compute a local result.

DFS can carry extra state down the path, like parity flips, path XOR, depth parity, subtree sums, or ancestor constraints.
SPOJ PT07Y — Tree check edge count + DFS connectivity

Abstract: In an undirected graph, being connected is not enough, and having n-1 edges alone is not enough either. A graph is a tree only when both conditions hold together: exactly n-1 edges and one connected component.

int dfsCount(int u){
    vis[u] = 1;
    int cnt = 1;
    for(int v : adj[u])
        if(!vis[v]) cnt += dfsCount(v);
    return cnt;
}
UVA 10608 — Largest connected component outer loop + dfsSize

Abstract: One DFS gives the size of only one component. To get the largest connected component, restart DFS from every still-unvisited node and maximize the returned size.

int dfsSize(int u){
    vis[u] = 1;
    int res = 1;
    for(int v : adj[u])
        if(!vis[v]) res += dfsSize(v);
    return res;
}
CF 430C — Tree parity flips DFS carries path state

Abstract: DFS is not only for visiting nodes. In tree problems, it can carry information along the current root-to-node path. Here the carried state is the parity of flips affecting even-depth and odd-depth nodes.

void dfs(int u, int p, int oddFlip, int evenFlip, int depth){
    int curFlip = (depth % 2 == 0 ? evenFlip : oddFlip);
    int effective = initv[u] ^ curFlip;
    if(effective != targetv[u]){
        ans.push_back(u);
        if(depth % 2 == 0) evenFlip ^= 1;
        else oddFlip ^= 1;
    }
    for(int v : adj[u]) if(v != p) dfs(v, u, oddFlip, evenFlip, depth + 1);
}

Full DFS Toolkit

This is a single compile-ready DFS reference in the same style used in the graphs notes. It includes general DFS, component size, undirected tree DFS with parent, iterative DFS, directed 3-color cycle detection, and grid flood fill using di/dj.

#include <bits/stdc++.h>
using namespace std;

vector<vector<int>> adj;
vector<char> vis;
vector<int> color;

void dfs(int u){
    vis[u] = 1;
    for(int v : adj[u]){
        if(!vis[v]) dfs(v);
    }
}

int dfsCount(int u){
    vis[u] = 1;
    int cnt = 1;
    for(int v : adj[u]){
        if(!vis[v]) cnt += dfsCount(v);
    }
    return cnt;
}

int dfsSize(int u){
    vis[u] = 1;
    int res = 1;
    for(int v : adj[u]){
        if(!vis[v]) res += dfsSize(v);
    }
    return res;
}

void dfs_tree(int u, int p){
    for(int v : adj[u]){
        if(v == p) continue;
        dfs_tree(v, u);
    }
}

void dfs_iter(int src){
    vector<int> st;
    st.push_back(src);
    vis[src] = 1;
    while(!st.empty()){
        int u = st.back();
        st.pop_back();
        for(int v : adj[u]){
            if(!vis[v]){
                vis[v] = 1;
                st.push_back(v);
            }
        }
    }
}

bool dfs_cycle(int u){
    color[u] = 1; // GRAY
    for(int v : adj[u]){
        if(color[v] == 0){
            if(dfs_cycle(v)) return true;
        }else if(color[v] == 1){
            return true; // back edge to GRAY in a directed graph
        }
    }
    color[u] = 2; // BLACK
    return false;
}

string grid[1005];
bool vis2d[1005][1005];
int di[] = {-1, 1, 0, 0};
int dj[] = {0, 0, -1, 1};
int rows, cols;

bool is_valid(int r, int c){
    return r >= 0 && r < rows && c >= 0 && c < cols && grid[r][c] == '.' && !vis2d[r][c];
}

void dfs_grid(int r, int c){
    vis2d[r][c] = true;
    for(int k = 0; k < 4; k++){
        int nr = r + di[k];
        int nc = c + dj[k];
        if(is_valid(nr, nc)) dfs_grid(nr, nc);
    }
}

Pitfalls & Contest Tips

PitfallWhy it happensFix
Forgetting visited[]Cyclic graphs can recurse foreverMark before exploring neighbors
Only starting DFS onceDisconnected graphs leave hidden componentsAdd an outer loop over all vertices
Recursion depth overflowLarge graphs or long chains exceed stack limitsUse iterative DFS with an explicit stack
Wrong cycle logic in undirected graphThe parent edge is always already visitedIgnore the parent: if(v == p) continue;
Assuming one unique traversal orderOrder depends on adjacency orderingSort neighbors only if the problem requires it
Using vector<bool> carelesslyIt is a specialized containervector<char> vis is often simpler

Complexity Summary

Time

On adjacency lists, DFS runs in O(n + m). Every vertex is entered at most once, and every adjacency entry is scanned a constant number of times.

Space

Extra DFS memory is usually O(n) for visited/state arrays plus recursion stack or explicit stack.
