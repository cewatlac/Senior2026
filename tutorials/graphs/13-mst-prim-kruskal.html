1. What Problem Are We Solving?

Given a weighted undirected graph with n vertices and m edges, choose a subset of edges such that:

  • All vertices become connected
  • No cycle appears
  • The total sum of chosen edge weights is minimum possible

That chosen edge set is called a Minimum Spanning Tree (MST).

Why "spanning tree"?

Spanning: it covers all n vertices.
Tree: it is connected and acyclic.

Every spanning tree has exactly n − 1 edges

Fewer than n−1 → cannot connect all vertices. More than n−1 → must contain a cycle.

Disconnected graphs → MSF

If the graph is not connected, a full spanning tree does not exist. The natural generalization is the Minimum Spanning Forest (MSF): one minimum spanning tree per connected component.

💡

Recognition pattern: connect cities / cable cost / bridge lengths / network backbone / keep everyone reachable at minimum budget → think MST immediately.

2. Core Mathematical Properties

All MST algorithms are greedy. They work because of two classical properties.

🔪 Cut Property

Take any cut of the graph: split vertices into two groups A and B. Among all edges crossing from A to B, any minimum-weight crossing edge is safe to include in some MST.

📐

This is the soul of both Prim and Kruskal. Every safe edge they add is justified by the cut property.

🔄 Cycle Property

Take any cycle. If an edge is strictly heavier than every other edge on that cycle, that edge cannot belong to any MST.

⚠️

This explains why Kruskal rejects edges that create cycles — any new cycle-creating edge is not the minimum on its cycle.

Why greedy works here

Many graph problems do not allow greedy decisions. MST does. When you pick a safe edge using the cut property, you are not ruining optimality — there will still exist an MST containing everything you have chosen so far. That is why both Prim and Kruskal are greedy, but still correct.

3. DSU Template

Kruskal repeatedly asks two questions: are u and v already in the same component? and if not, merge their components. That is exactly what DSU is designed for.

🔑

Convention: par[u] < 0 means u is a root. -par[u] is the component size. Non-root nodes store their parent index. Union by size + path compression.

#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;

#define ll long long
const int N = 200000 + 5;

struct DSU {
    int par[N];

    void init(int n) {
        fill(par, par + n, -1);
    }

    int find(int u) {
        if (par[u] < 0) return u;
        return par[u] = find(par[u]);
    }

    bool same(int a, int b) {
        return find(a) == find(b);
    }

    bool join(int a, int b) {
        a = find(a);
        b = find(b);
        if (a == b) return false;

        if (-par[a] < -par[b]) swap(a, b); // a has bigger size
        par[a] += par[b];
        par[b] = a;
        return true;
    }

    int size(int u) {
        return -par[find(u)];
    }
} dsu;
DSU line-by-line explanation click to expand

par[root] is negative

This stores extra information: the size. A bigger component has a more negative value. So if par[a] > par[b], then a is actually the smaller component.

Path compression

return par[u] = find(par[u]); — flattens the tree on every find, making future find() calls much faster. Amortized α(n).

Union by size

if (-par[a] < -par[b]) swap(a, b); — ensures the smaller component attaches to the larger one. This prevents trees from becoming tall.

join() returns bool

Returns true if a merge actually happened (different components), false if they were already connected. This is exactly what Kruskal needs to decide: add edge or skip.

4. Edge List — The Natural Kruskal Structure

Kruskal does not care about adjacency traversal. It only cares about all edges sorted by weight. So the graph is best stored as an edge list:

struct Edge {
    int u, v;
    ll w;

    bool operator < (const Edge& other) const {
        return w < other.w;
    }
};

vector<Edge> edges;
💡

Key insight: different algorithms want different graph representations. Kruskal wants an edge list. Prim wants an adjacency list or matrix. Always match your data structure to your algorithm.

Reading input for Kruskal

cin >> n >> m;
edges.reserve(m);
for (int i = 0; i < m; i++) {
    int u, v; ll w;
    cin >> u >> v >> w;
    --u, --v;  // 0-based
    edges.push_back({u, v, w});
}

Self-loops and parallel edges

Both are fine in the edge list. Kruskal's DSU will naturally reject self-loops (u == v → same component immediately) and will prefer lighter parallel edges.

5. Kruskal Master Template

#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;

#define ll long long
const int N = 200000 + 5;

struct DSU {
    int par[N];
    void init(int n) { fill(par, par + n, -1); }
    int find(int u) {
        if (par[u] < 0) return u;
        return par[u] = find(par[u]);
    }
    bool join(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;
        if (-par[a] < -par[b]) swap(a, b);
        par[a] += par[b];
        par[b] = a;
        return true;
    }
} dsu;

struct Edge {
    int u, v;
    ll w;
    bool operator < (const Edge& other) const {
        return w < other.w;
    }
};

int n, m;
vector<Edge> edges, mst;

ll kruskal() {
    sort(edges.begin(), edges.end());
    dsu.init(n);

    ll cost = 0;
    int taken = 0;
    mst.clear();

    for (auto &e : edges) {
        if (dsu.join(e.u, e.v)) {
            cost += e.w;
            mst.push_back(e);
            taken++;
            if (taken == n - 1) break;
        }
    }

    if (taken != n - 1) return -1; // graph disconnected
    return cost;
}

int main() { _
    cin >> n >> m;
    edges.reserve(m);
    for (int i = 0; i < m; i++) {
        int u, v; ll w;
        cin >> u >> v >> w;
        --u, --v;
        edges.push_back({u, v, w});
    }
    ll ans = kruskal();
    if (ans == -1) cout << "IMPOSSIBLE\n";
    else cout << ans << '\n';
}
Step-by-step idea click to expand

Step 1: Sort edges

We want to try cheaper edges before more expensive ones. Sorting puts them in non-decreasing order of weight.

Step 2: Each vertex starts in its own component

dsu.init(n) initializes all par[i] = -1, meaning n singleton components.

Step 3: Scan edges lightest to heaviest

For each edge (u, v, w): if u and v are already in the same component, adding it would create a cycle → skip. Otherwise, this edge connects two different components → take it, add weight to cost.

Step 4: Stop after n−1 chosen edges

At that point a spanning tree is already built. The if (taken == n-1) break; is a useful early exit optimization.

Step 5: Check connectivity

If we couldn't collect n−1 edges, the graph was disconnected. Return −1 (or handle as a spanning forest).

6. Kruskal Correctness Proof

📐

Claim: Whenever Kruskal adds an edge e, there exists some MST containing all chosen edges so far.

Proof: Suppose Kruskal is currently considering edge e = (u, v, w) and decides to add it. At this moment, u and v are in different DSU components. Think of the cut where one side is the component containing u, and the other side is everything else.

Edge e crosses this cut. Now why is e safe? Because Kruskal processes edges in nondecreasing order. If there were a lighter edge crossing this cut, Kruskal would have processed it earlier — and since its endpoints were also in different components at that time, Kruskal would have taken it. So e is a minimum-weight crossing edge for that cut.

By the cut property, such an edge is safe: some MST contains it. Thus after each accepted edge, there still exists an MST containing all accepted edges. After Kruskal accepts n−1 edges, those edges form a spanning tree. Since they are all contained in some MST, they themselves must be an MST. ∎

8. Prim vs Dijkstra — The Deep Analogy

This is the key bridge to your Dijkstra tutorial. The outer skeleton is almost identical.

In Dijkstra

dist[v] = best path distance from source to v

Relaxation: if (dist[u] + w < dist[v]) dist[v] = dist[u] + w;

In Prim

dist[v] = min edge weight connecting v to current tree

Relaxation: if (w < dist[v]) dist[v] = w;

⚡

The only change in relaxation: Prim does not care about total path cost from source. It only cares about the cheapest single edge that can attach v to the current tree. The outer structure — maintain unfinalized set, extract min, relax neighbors — is identical.

9. Dense Prim — notVis Template

Use dense Prim when the graph is dense, n is small enough for O(n²), or the graph is stored as an adjacency matrix. This uses the same notVis compact-set invariant as your dense Dijkstra template.

#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;

#define ll long long
const ll OO = (1LL << 60);
const int N = 2000 + 5;

int n;
ll adj[N][N];
ll dista[N];
int par[N], notVis[N];

ll prim_dense(int src = 0) {
    int sz = n;

    fill(dista, dista + n, OO);
    fill(par, par + n, -1);
    iota(notVis, notVis + n, 0);

    dista[src] = 0;
    int nxt = src;
    ll mstCost = 0;
    int taken = 0;

    while (nxt != -1) {
        int u = notVis[nxt];
        notVis[nxt] = notVis[--sz];   // remove u from unvisited in O(1)
        nxt = -1;

        mstCost += dista[u];
        taken++;

        ll best = OO;

        for (int i = 0; i < sz; i++) {
            int v = notVis[i];

            if (adj[u][v] < dista[v]) {    // Prim relaxation: edge weight only
                dista[v] = adj[u][v];
                par[v] = u;
            }

            if (dista[v] < best) {
                best = dista[v];
                nxt = i;
            }
        }
    }

    if (taken != n) return -1; // disconnected
    return mstCost;
}

int main() { _
    cin >> n;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) {
            cin >> adj[i][j];
            if (i != j && adj[i][j] == 0) adj[i][j] = OO; // 0 means no edge
        }

    ll ans = prim_dense(0);
    if (ans == -1) cout << "IMPOSSIBLE\n";
    else cout << ans << '\n';
}
Dense Prim — detailed explanation click to expand

dista[v]

Current cheapest edge weight that can connect v to the already-built tree.

par[v]

Which tree vertex gives that cheapest connection. Used for reconstruction.

notVis[] invariant

Stores the current unvisited vertices. When we finalize vertex u, we remove it by swapping with the last element and decrementing sz. We then scan only the live prefix 0..sz-1. This is the same strong invariant as your dense Dijkstra template.

dista[src] = 0

This makes the starting vertex enter the tree first with cost zero. That zero gets added to mstCost — contributing nothing, which is correct.

mstCost += dista[u]

When u is finalized, dista[u] is the weight of the edge by which u joins the tree. For the source, that cost is zero.

10. Sparse Prim — done[] Template

Use sparse Prim when the graph is sparse and stored as an adjacency list. Target complexity: O(m log n).

#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;

#define ll long long
const ll OO = (1LL << 60);
const int N = 200000 + 5;

int n, m;
vector<pair<int,ll>> adj[N];
ll dista[N];
int par[N];
bool done[N];

ll prim_sparse(int src = 0) {
    fill(dista, dista + n, OO);
    fill(par, par + n, -1);
    fill(done, done + n, false);

    priority_queue<pair<ll,int>, vector<pair<ll,int>>, greater<pair<ll,int>>> pq;

    dista[src] = 0;
    pq.push({0, src});

    ll mstCost = 0;
    int taken = 0;

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (done[u]) continue;   // explicit finalized check
        done[u] = true;

        mstCost += d;
        taken++;

        for (auto [v, w] : adj[u]) {
            if (!done[v] && w < dista[v]) {   // Prim relaxation: edge weight only
                dista[v] = w;
                par[v] = u;
                pq.push({dista[v], v});
            }
        }
    }

    if (taken != n) return -1; // disconnected
    return mstCost;
}

int main() { _
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v; ll w;
        cin >> u >> v >> w;
        --u, --v;
        adj[u].push_back({v, w});
        adj[v].push_back({u, w});
    }
    ll ans = prim_sparse(0);
    if (ans == -1) cout << "IMPOSSIBLE\n";
    else cout << ans << '\n';
}

11. done[] vs dist[u] ≠ d — Why It Matters

In sparse Dijkstra, a common stale-state test is:

if (d > dist[u]) continue;  // or: if (d != dist[u]) continue;

For Prim, depending on only this check is not the best approach. Here is why:

⚠️

Fragile approach (dist[u] ≠ d): In Prim, the real logical event is not "is this the latest key?" — it is "has this vertex already been finalized into the tree?" Once a vertex is taken into the MST, any later heap entries for it are meaningless regardless of key equality.

// Less clear intent:
auto [d, u] = pq.top(); pq.pop();
if (d != dista[u]) continue;  // fragile
✓

Robust approach (done[]): The most semantically correct version keeps an explicit bool done[] array. This is clearer, safer, and more faithful to Prim's invariant.

// Explicit and correct:
auto [d, u] = pq.top(); pq.pop();
if (done[u]) continue;  // clear intent
done[u] = true;
💡

The invariant that sparse Prim maintains: done[u] = true means u is permanently inside the MST. For every not-done vertex v, dista[v] is the cheapest edge currently known from the built tree to v. The priority queue may contain many stale entries, but the first time we pop a not-done vertex, that key is the correct minimum connection cost for that vertex.

12. Prim Correctness & Complexity

Correctness

At every step, let T = set of finalized vertices. For each unfinalized v, dista[v] is the minimum weight among all edges from T to v. Prim chooses vertex u with minimum dista[u]. So the edge (par[u], u) is the minimum crossing edge among all edges leaving T.

By the cut property, that edge is safe for some MST. Every finalized vertex joins via a safe edge. After all vertices are finalized, we have chosen exactly one entering edge for every non-source vertex → n−1 edges → spanning tree. Since every chosen edge was safe → it is an MST. ∎

Note: the heap only makes extraction faster. It does not change the logic. Dense and sparse Prim make the same greedy choice.

Complexity

VersionTimeMemory
Dense (notVis)O(n²)O(n²)
Sparse (done[])O(m log n)O(n+m)

Dense: each of n extractions scans all remaining unvisited vertices. Sparse: each relaxation pushes one heap state; m total relaxations × log n each.

14. Reconstructing the Actual MST Edges

In Kruskal

Whenever you accept an edge, push it into the mst vector.

vector<Edge> mst;
// inside loop:
if (dsu.join(e.u, e.v)) {
    cost += e.w;
    mst.push_back(e);  // record MST edge
}

In Prim

For every vertex v ≠ src, the chosen MST edge is (par[v], v).

// After prim_sparse():
for (int v = 0; v < n; v++) {
    if (par[v] != -1)
        cout << par[v]+1 << ' ' << v+1 << '\n';
}

Each vertex except the source enters the tree exactly once. At that moment, par[v] = u is set. That parent edge is the MST edge for v.

15. Important Edge Cases

🔗 Disconnected Graph

Full MST does not exist. Kruskal takes fewer than n−1 edges. Prim finalizes fewer than n vertices. Handle by printing IMPOSSIBLE or computing a minimum spanning forest.

➖ Negative Weights

MST algorithms do not require nonneg weights — a key difference from Dijkstra. Negative edge weights are completely fine. The tree just minimizes total sum.

✓

Both Kruskal and Prim handle negative weights correctly.

🔁 Self-loops

Self-loops can never help an MST. In Kruskal, u == v means same component immediately → rejected by DSU. In Prim, a self-loop never connects a new vertex. No special handling needed.

Variant 1 Standard MST by Kruskal
CSES 1675

Story: Undirected weighted graph. Connect all vertices with minimum total cost.

Abstraction: Given G=(V,E) with n vertices and m weighted edges, find a subset T⊆E of exactly n−1 edges forming a connected acyclic subgraph with minimum total weight.

Why MST fits: The goal is global minimum-cost connectivity with no redundant edges — the textbook MST definition.

Algorithm: Kruskal — sort edges ascending, use DSU to greedily accept safe edges and reject cycle-forming ones.

🔗 CSES 1675 — Road Reparation 
#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;
#define ll long long
const int N = 200000 + 5;
struct DSU {
    int par[N];
    void init(int n) { fill(par, par + n, -1); }
    int find(int u) { return par[u] < 0 ? u : par[u] = find(par[u]); }
    bool join(int a, int b) {
        a = find(a), b = find(b);
        if (a == b) return false;
        if (-par[a] < -par[b]) swap(a, b);
        par[a] += par[b]; par[b] = a; return true;
    }
} dsu;
struct Edge { int u, v; ll w; bool operator<(const Edge& o) const { return w < o.w; } };
int main() { _
    int n, m; cin >> n >> m;
    vector<Edge> edges(m);
    for (auto &e : edges) { cin >> e.u >> e.v >> e.w; --e.u, --e.v; }
    sort(edges.begin(), edges.end());
    dsu.init(n);
    ll cost = 0; int taken = 0;
    for (auto &e : edges)
        if (dsu.join(e.u, e.v)) { cost += e.w; if (++taken == n-1) break; }
    if (taken != n-1) cout << "IMPOSSIBLE\n";
    else cout << cost << '\n';
}

Complexity: O(m log m) for sorting + O(m α(n)) for DSU operations ≈ O(m log m) overall. Space: O(n+m).

Variant 2 Standard MST by Prim Sparse
CSES 1675

Story: Same MST problem, graph given as adjacency list.

Abstraction: Undirected weighted graph G=(V,E) as adjacency list. Find minimum weight spanning tree.

Why MST fits: Same as Variant 1 — minimize total connection cost across all vertices.

Algorithm: Prim Sparse — grow one tree from source using a min-heap; the critical relaxation is w < dista[v] (edge weight only, not dist[u]+w).

🔗 CSES 1675 — Road Reparation 
#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;
#define ll long long
const ll OO = (1LL << 60);
const int N = 200000 + 5;
int n, m;
vector<pair<int,ll>> adj[N];
ll dista[N]; int par[N]; bool done[N];
int main() { _
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v; ll w; cin >> u >> v >> w; --u, --v;
        adj[u].push_back({v,w}); adj[v].push_back({u,w});
    }
    fill(dista, dista+n, OO); fill(par, par+n, -1); fill(done, done+n, false);
    priority_queue<pair<ll,int>,vector<pair<ll,int>>,greater<>> pq;
    dista[0] = 0; pq.push({0,0});
    ll cost = 0; int taken = 0;
    while (!pq.empty()) {
        auto [d,u] = pq.top(); pq.pop();
        if (done[u]) continue;
        done[u] = true; cost += d; taken++;
        for (auto [v,w] : adj[u]) if (!done[v] && w < dista[v]) {
            dista[v] = w; par[v] = u; pq.push({w,v});
        }
    }
    if (taken != n) cout << "IMPOSSIBLE\n";
    else cout << cost << '\n';
}

Complexity: O(m log n) — m relaxations each pushing one heap entry, O(log n) per push/pop. Space: O(n+m).

Why MST fits: Kruskal naturally sweeps edges in weight order. Sorting both edges and queries by threshold allows a single coordinated pass — the DSU state after processing all edges ≤ X gives the answer for query X.

Algorithm: Kruskal-style sweep — sort edges and queries jointly by threshold; advance the edge pointer until the threshold is exceeded, then record the current taken count for that query.

🔗 CF 1661E — Offline Queries (DSU sweep template) 
#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);
using namespace std;
#define ll long long
const int N = 200000 + 5;
struct DSU {
    int par[N];
    void init(int n) { fill(par, par + n, -1); }
    int find(int u) { return par[u] < 0 ? u : par[u] = find(par[u]); }
    bool join(int a, int b) {
        a = find(a), b = find(b);
        if (a == b) return false;
        if (-par[a] < -par[b]) swap(a, b);
        par[a] += par[b]; par[b] = a; return true;
    }
} dsu;
struct Edge { int u, v; ll w; bool operator<(const Edge& o) const { return w < o.w; } };
struct Query { ll x; int id; bool operator<(const Query& o) const { return x < o.x; } };
int main() { _
    int n, m, q; cin >> n >> m >> q;
    vector<Edge> edges(m);
    for (auto &e : edges) { cin >> e.u >> e.v >> e.w; --e.u, --e.v; }
    vector<Query> qs(q);
    for (int i = 0; i < q; i++) { cin >> qs[i].x; qs[i].id = i; }
    sort(edges.begin(), edges.end()); sort(qs.begin(), qs.end());
    dsu.init(n);
    vector<int> ans(q); int j = 0, taken = 0;
    for (auto& qu : qs) {
        while (j < m && edges[j].w <= qu.x) { if (dsu.join(edges[j].u, edges[j].v)) taken++; j++; }
        ans[qu.id] = taken;
    }
    for (int a : ans) cout << a << '\n';
}

Complexity: O((m+q) log(m+q)) for sorting + O(m α(n)) for DSU. Space: O(n+m+q).

Variant 16 Incremental MSF — Add Edges and Report Cost
LightOJ 1123

Story: Edges are added one by one. After each addition, report the current minimum spanning forest cost.

Abstraction: Process q edge insertions online; after each insertion, output the MSF cost of the current graph.

Why MST fits: Each insertion potentially changes the MSF. The simplest correct approach is to re-run Kruskal from scratch with all edges seen so far — optimal for small inputs; more advanced approaches use link-cut trees.

Algorithm: Incremental Kruskal — maintain a sorted growing edge list; after each insertion, re-run full Kruskal. (For large inputs, use a more advanced offline MST data structure.)

Variant 17 Minimum Bottleneck Spanning Tree
MST property

Story: Minimize the maximum edge weight in the spanning tree.

Abstraction: Given G=(V,E), find a spanning tree T that minimizes max{w(e) : e ∈ T}. This is the minimum bottleneck spanning tree (MBST).

Why MST fits: Every MST is also an MBST — a classical theorem. Kruskal adds edges in ascending weight order; the last accepted edge is always the bottleneck of the MST it produces.

Algorithm: Kruskal — record the weight of the last accepted edge as the bottleneck; everything else is standard.

Variant 18 Node Activation Cost — Super Source Modeling
CF 1245D

Story: Each city can either build its own power station at cost c[i], or connect via edges. Find the cheapest global plan.

Abstraction: Given G=(V,E) where each vertex has an optional self-activation cost c[i], find the minimum total cost to make all vertices active — either by paying c[i] directly or by connecting to an already-active vertex via edges.

Why MST fits: Adding a super source S connected to each vertex with weight c[i] transforms the problem into a standard MST on n+1 vertices. An S–i edge being in the MST means vertex i activates itself; a regular edge means it connects to the network.

Algorithm: Kruskal on the augmented graph with the super source — add n extra edges and run standard Kruskal on n+1 vertices.

Variant 19 Clustering — Stop Kruskal at k Components
POJ 2349

Story: Partition n points into exactly k clusters, minimizing total internal connection cost.

Abstraction: Given G=(V,E), find a minimum spanning forest with exactly k trees (k components), minimizing the total weight of edges chosen.

Why MST fits: The MSF with k components is obtained by running Kruskal and stopping when the number of components drops to k — the first k−1 edges that would complete the full MST are simply left out.

Algorithm: Kruskal with component counter — augment DSU with a comps field that decrements on each merge; stop when comps == k.

#include <bits/stdc++.h>

using namespace std;
#define ll long long
const int N = 200000 + 5;
struct DSU {
    int par[N], comps;
    void init(int n) { fill(par, par + n, -1); comps = n; }
    int find(int u) { return par[u] < 0 ? u : par[u] = find(par[u]); }
    bool join(int a, int b) {
        a = find(a); b = find(b);
        if (a == b) return false;
        if (-par[a] < -par[b]) swap(a, b);
        par[a] += par[b]; par[b] = a; comps--; return true;
    }
} dsu;
struct Edge { int u, v; ll w; bool operator<(const Edge& o) const { return w < o.w; } };

}

Variant 20 Cheapest Extra Edge to Connect MSF Components
Post-Kruskal DSU

Story: You built a minimum spanning forest on a disconnected graph. Now find the cheapest additional edge that connects two different forest components.

Abstraction: Given a disconnected G=(V,E), after building the MSF, find the minimum weight edge (u,v) such that u and v are in different MSF components — i.e., the cheapest bridge between two components.

Why MST fits: The DSU state after Kruskal fully encodes component membership. Scanning all non-tree edges for the cheapest cross-component one is a direct post-Kruskal query on the DSU.

Algorithm: Kruskal (MSF) to build the forest, then scan all edges for the minimum-weight one crossing two different DSU components.

🔗 CSES 1676 — Road Construction (connect MSF components) 
#include <bits/stdc++.h>
#define _ ios::sync_with_stdio(false); cin.tie(nullptr);

#define ll long long
const ll OO = (1LL << 60);
const int N = 200000 + 5;
struct DSU {
    int par[N];
    void init(int n) { fill(par, par + n, -1); }
    int find(int u) { return par[u] < 0 ? u : par[u] = find(par[u]); }
    bool join(int a, int b) {
        a = find(a), b = find(b);
        if (a == b) return false;
        if (-par[a] < -par[b]) swap(a, b);
        par[a] += par[b]; par[b] = a; return true;
    }
} dsu;
struct Edge { int u, v; ll w; bool operator<(const Edge& o) const { return w < o.w; } };

    vector<Edge> edges(m);
    for (auto &e : edges) { cin >> e.u >> e.v >> e.w; --e.u, --e.v; }
    sort(edges.begin(), edges.end());
    dsu.init(n);
    for (auto &e : edges) dsu.join(e.u, e.v); // build MSF
    // Find cheapest cross-component edge
    ll best = OO;
    for (auto &e : edges)
        if (dsu.find(e.u) != dsu.find(e.v))
            best = min(best, e.w);
    if (best == OO) cout << "NO EXTRA EDGE\n";
    else cout << best << '\n';
}

Complexity: O(m log m) for sorting + O(m α(n)) for DSU. Space: O(n+m).

Common Mistakes

Kruskal Mistakes

  • ⚠ Using adjacency list for Kruskal when edge list is natural
  • ⚠ Forgetting to sort edges before the main loop
  • ⚠ Comparing DSU parents directly instead of roots — always call find()
  • ⚠ Forgetting the disconnected graph case (check taken == n-1)
  • ⚠ Using int for weights when ll is needed (overflow)

Prim Mistakes

  • ⚠ Writing Dijkstra relaxation dist[u]+w instead of Prim's w
  • ⚠ In sparse Prim, depending only on d == dist[u] for stale detection — use done[u]
  • ⚠ Applying Prim on a directed graph — MST is for undirected only
  • ⚠ In dense Prim, forgetting that notVis stores vertex IDs, not positions
  • ⚠ Initializing dista[src] = OO instead of 0
