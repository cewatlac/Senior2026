1. What is a Strongly Connected Component?

In a directed graph, a strongly connected component (SCC) is a maximal set of vertices such that every vertex can reach every other vertex in that set.

💡

If u and v belong to the same SCC, there exists a path u → v AND a path v → u. "Maximal" means you cannot add any more vertex and preserve this property.

SCC Example — 8-node directed graph
SCC #0 {0,1,2} SCC #1 {3,4} SCC #2 {5,6} SCC #3 {7}

Why SCC Matters

Once you compress every SCC into one super-node, the graph turns into a DAG — the condensation graph. This single fact powers an enormous amount of contest graph logic.

2. Why the Condensation Graph is Always a DAG

Suppose the condensation had a directed cycle:

C₁ → C₂ → ··· → Cₖ → C₁

Pick any vertex from C₁. Because there is a path through the cycle and back, every component on the cycle can reach every other one. But then they were never different SCCs — contradiction. So the condensation is acyclic.

⚠

Key consequence: The condensation is always a DAG regardless of the original graph's structure. This is what makes SCC compression so powerful — DAGs are much easier to reason about.

3. Tarjan's Algorithm

Tarjan is the cleanest one-pass SCC algorithm. A single DFS discovers all SCCs in O(N + M). It is also the best base for 2-SAT.

🏆

Why prefer Tarjan? One DFS pass, no reverse graph, directly produces SCC IDs in the topological order needed for 2-SAT assignment.

3.1 The Three Key Arrays

ArrayMeaning
dfsT[u]DFS discovery time of node u (increments with each new visit)
low[u]Smallest discovery time reachable from u via tree edges downward, then optionally one back-edge staying in the active DFS region
cmpId[u]−1 = not yet assigned to any SCC; otherwise = the SCC id of u

We also maintain a stack of currently active (not-yet-finished) vertices. When an SCC root is found, we pop everything down to and including the root.

3.2 The Key Invariant

While node u is still in the recursion stack, low[u] tells us the "highest point" the current SCC connects back to.

🔑

If low[u] == dfsT[u], then u is the root of an SCC. Pop everything from the stack down to u — they all belong to that SCC.

How low[] updates — three edge cases
🌳

Tree edge u→v (unvisited):
low[u] = min(low[u], low[v]) — after returning from v

↩

Back edge u→v (v active, cmpId[v]==-1):
low[u] = min(low[u], low[v]) — v is still in our DFS region

✗

Cross edge u→v (v done, cmpId[v]!=-1):
Skip entirely — v belongs to a finished SCC, cannot affect us

3.3 Why cmpId[v] == -1 is a Clever Trick

Many textbooks maintain a separate boolean inStack[v]. Tarjan's implementation can avoid this with:

cmpId[v] == −1 ⟺ v is still active in the current DFS

The moment a node is popped into an SCC, its cmpId is set to a non-negative id. So cmpId[v] == -1 perfectly captures "v has been visited but not yet finished." One array does the work of two.

✓

This is exactly the trick used in the templates in this tutorial — it is cleaner and just as correct as a separate inStack array.

3.4 Tarjan Correctness Proof

Claim 1 — The popped block is exactly one SCC click to expand proof

They are strongly connected: Every popped vertex x is in the DFS subtree of u. Since low[x] cannot escape above u, all vertices remain tied into the region rooted at u. By the DFS-tree and back-edge structure, u reaches all of them, and each has a path back into the active ancestor chain through u.

No outside vertex belongs: Any vertex not in the popped block either cannot reach back to u inside the active region, or is already assigned to another SCC. It cannot be part of this SCC.

✓

Therefore: the popped block is exactly one SCC.

Claim 2 — Every SCC is found exactly once click to expand proof

Pick any SCC S, and let u be the first vertex discovered in S. No vertex in S has a smaller discovery time than u. Since all vertices in S reach each other, DFS from u eventually visits all of them, and each can reach back to u or a descendant of u in the active chain. Therefore low[u] = dfsT[u], and when u finishes, exactly S is popped.

Since popped vertices are permanently assigned (cmpId set), no SCC is counted twice.

✓

Tarjan is correct.

3.5 Complexity

Each vertex is discovered once, pushed once, and popped once. Each edge is examined once. Total: O(N + M) time and space.

3.6 Tarjan Master Template — Forward-Star Style

C++ Template
Code Notes
#include <bits/stdc++.h>
using namespace std;

#define adj(head, u, e, v) for (int e = head[u], v; ~e && ((v = to[e]), 1); e = nxt[e])

const int N = 200000 + 5;
const int M = 400000 + 5;

int head[N], to[M], nxt[M], ne;
int dfsT[N], low[N], cmpId[N], stk[N], sz, curT, cmpCnt;
int n;

void initGraph(int nNodes){
    n = nNodes; ne = 0;
    memset(head, -1, n * sizeof(head[0]));
}

void addEdge(int u, int v){
    to[ne] = v; nxt[ne] = head[u]; head[u] = ne++;
}

void tarjanDfs(int u){
    dfsT[u] = low[u] = curT++;
    stk[sz++] = u;

    adj(head, u, e, v){
        if(dfsT[v] == -1){           // unvisited → tree edge
            tarjanDfs(v);
            low[u] = min(low[u], low[v]);
        } else if(cmpId[v] == -1){   // visited & still active → back edge
            low[u] = min(low[u], low[v]);
        }
        // cmpId[v] != -1 → finished SCC, skip
    }

    if(low[u] == dfsT[u]){           // u is SCC root
        do {
            cmpId[stk[--sz]] = cmpCnt;
        } while(stk[sz] != u);
        cmpCnt++;
    }
}

void tarjan(){
    curT = sz = cmpCnt = 0;
    memset(dfsT, -1, n * sizeof(dfsT[0]));
    memset(cmpId, -1, n * sizeof(cmpId[0]));
    for(int i = 0; i < n; i++)
        if(dfsT[i] == -1) tarjanDfs(i);
}
📌

The adj macro iterates the forward-star adjacency list. ~e is e != -1 via bitwise NOT.

⚠

Recursive DFS may stack-overflow for N ≥ 200 000 on some judges. Convert to iterative if needed.

💡

SCC IDs are assigned in reverse topological order of the condensation. This is exactly the order needed for 2-SAT value assignment.

🔢

cmpCnt after running = total number of SCCs in the graph.

4. Kosaraju's Algorithm

Kosaraju uses two DFS passes and requires building the reverse graph. Same O(N+M) complexity, but preferred when teaching because the intuition is easy to follow.

The Three Steps

  1. DFS the original graph, push each node after it fully finishes (post-order).
  2. Reverse all edges to get the transpose graph.
  3. Process nodes in reverse finishing order on the transpose — each new DFS gives exactly one SCC.

Why Finishing Order Works

If there is an edge SCC A → SCC B in the condensation, the finish time of any vertex in A is larger than that of any vertex in B during DFS on the original graph. So processing in reverse finishing order always starts a DFS from a "source" SCC of whatever remains. On the reversed graph, that DFS cannot leak into another unassigned SCC.

Kosaraju Master Template forward-star style 
#include <bits/stdc++.h>
using namespace std;

#define adj(head, u, e, v) for (int e = head[u], v; ~e && ((v = to[e]), 1); e = nxt[e])

const int N = 200000 + 5;
const int M = 400000 + 5;

int head[N], rhead[N], to[M], rto[M], nxt[M], rnxt[M], ne, rne;
bool vis[N];
int stk[N], topOrder, cmpId[N];
int n;

void initGraph(int nNodes){
    n = nNodes; ne = rne = 0;
    memset(head, -1, n * sizeof(head[0]));
    memset(rhead, -1, n * sizeof(rhead[0]));
}

void addEdge(int u, int v){
    to[ne] = v; nxt[ne] = head[u]; head[u] = ne++;
    rto[rne] = u; rnxt[rne] = rhead[v]; rhead[v] = rne++;
}

void dfs1(int u){
    vis[u] = true;
    adj(head, u, e, v){ if(!vis[v]) dfs1(v); }
    stk[topOrder++] = u;          // post-order
}

void dfs2(int u, int c){
    cmpId[u] = c;
    adj(rhead, u, e, v){ if(cmpId[v] == -1) dfs2(v, c); }
}

int kosaraju(){
    memset(vis, 0, n * sizeof(vis[0]));
    topOrder = 0;
    for(int i = 0; i < n; i++) if(!vis[i]) dfs1(i);
    memset(cmpId, -1, n * sizeof(cmpId[0]));
    int cmpCnt = 0;
    for(int i = topOrder - 1; i >= 0; i--){
        int u = stk[i];
        if(cmpId[u] == -1) dfs2(u, cmpCnt++);
    }
    return cmpCnt;
}

5. Tarjan vs Kosaraju

Use Tarjan when:
  • You want one DFS pass
  • You need a direct 2-SAT base
  • You want no reverse graph
  • Contest code must be minimal
Use Kosaraju when:
  • Teaching — proof is more intuitive
  • Reverse edges already available
  • 2× memory is acceptable
⚡

Both are O(N + M) time and space. For 2-SAT in competitive programming, Tarjan is the standard choice.

6. 2-SAT & the Implication Graph

A 2-SAT formula is a conjunction of clauses, each with exactly two literals:

(a ∨ b) ∧ (¬c ∨ d) ∧ ···

2-SAT is solvable in polynomial time via the implication graph. The key transformation:

(a ∨ b) ≡ (¬a → b) ∧ (¬b → a)

Every clause becomes two directed implications. Each variable x has two nodes: x (true) and ¬x (false). Node encoding: false-literal of variable i = 2*i, true-literal = 2*i+1.

Implication Graph Builder — step by step
Formula: (x₁ ∨ x₂) ∧ (¬x₁ ∨ x₃) ∧ (¬x₂ ∨ ¬x₃). Press "+ Add Clause" to add implications one at a time.

6.2 The Contradiction Criterion

⚠

Theorem: The 2-SAT formula is satisfiable if and only if, for every variable x, the nodes x and ¬x belong to different SCCs.

Proof — why same SCC means contradiction click to expand

If x and ¬x are in the same SCC, both x → ¬x and ¬x → x hold.

  • If x is set true, implication forces ¬x true — contradiction.
  • If x is set false (¬x true), implication forces x true — contradiction.

Conversely, if no variable's two literals share an SCC, the condensation DAG is consistent and we can assign values along topological order without violations.

6.3 Which SCC Order Gives the Assignment?

With the Tarjan numbering in this tutorial, if there is an edge C → D in the condensation, then cmpId[C] > cmpId[D] (C is popped earlier, gets a higher ID). Therefore:

value(x) = (cmpId[x] < cmpId[¬x])
🔑

Intuition: Set the literal whose SCC is "earlier in topological order" (lower ID in our numbering) to true. This respects all implications — you never force a false literal to be true against an implication.

ConditionResult
cmpId[x] == cmpId[¬x]UNSATISFIABLE — contradiction
cmpId[x] < cmpId[¬x]Set x = true
cmpId[x] > cmpId[¬x]Set x = false

7. Tarjan 2-SAT Master Template

Full Template
Helper Reference
#include <bits/stdc++.h>
using namespace std;

#define adj(head, u, e, v) for (int e = head[u], v; ~e && ((v = to[e]), 1); e = nxt[e])

const int MAXV = 400000 + 5;
const int MAXE = 4000000 + 5;

int head[MAXV], to[MAXE], nxt[MAXE], ne;
int dfsT[MAXV], low[MAXV], cmpId[MAXV], stk[MAXV], sz, curT, cmpCnt;
int vars, nodes;

void init2SAT(int nVars){
    vars = nVars; nodes = 2 * vars; ne = 0;
    memset(head, -1, nodes * sizeof(head[0]));
}

inline int var(int x)  { return 2 * x; }    // false-literal node for variable x (0-based)
inline int neg(int x)  { return x ^ 1; }    // flip literal (XOR bit-0)

void addEdge(int u, int v){ to[ne]=v; nxt[ne]=head[u]; head[u]=ne++; }

void addOr(int a, int b)   { addEdge(neg(a),b); addEdge(neg(b),a); }
void addImp(int a, int b)  { addEdge(a,b); }
void forceTrue(int a)      { addEdge(neg(a),a); }
void forceFalse(int a)     { addEdge(a,neg(a)); }
void addXor(int a, int b)  { addOr(a,b); addOr(neg(a),neg(b)); }
void addEqual(int a, int b){ addImp(a,b); addImp(b,a); addImp(neg(a),neg(b)); addImp(neg(b),neg(a)); }

void tarjanDfs(int u){
    dfsT[u] = low[u] = curT++;
    stk[sz++] = u;
    adj(head, u, e, v){
        if(dfsT[v]==-1){ tarjanDfs(v); low[u]=min(low[u],low[v]); }
        else if(cmpId[v]==-1){ low[u]=min(low[u],low[v]); }
    }
    if(low[u]==dfsT[u]){ do{ cmpId[stk[--sz]]=cmpCnt; }while(stk[sz]!=u); cmpCnt++; }
}

bool solve2SAT(vector<int>& ans){
    curT=sz=cmpCnt=0;
    memset(dfsT,-1,nodes*sizeof(dfsT[0]));
    memset(cmpId,-1,nodes*sizeof(cmpId[0]));
    for(int i=0;i<nodes;i++) if(dfsT[i]==-1) tarjanDfs(i);

    ans.assign(vars,0);
    for(int i=0;i<vars;i++){
        if(cmpId[var(i)]==cmpId[neg(var(i))]) return false; // contradiction
        ans[i]=(cmpId[var(i)]<cmpId[neg(var(i))]);          // assignment rule
    }
    return true;
}

Variable Encoding

Variable i (0-based): false-node = 2i, true-node = 2i+1

Flipping a literal: x ^ 1 works because 2i and 2i+1 differ only in bit 0.

FunctionLogical meaningEdges added
addOr(a, b)a ∨ b¬a→b, ¬b→a
addImp(a, b)a → b (≡ ¬a ∨ b)a→b
forceTrue(a)a must be true¬a→a
forceFalse(a)a must be falsea→¬a
addXor(a, b)exactly one of a, b is truea∨b and ¬a∨¬b
addEqual(a, b)a and b have same valuea↔b (4 edges)
📌

To get the true-literal of variable i, use var(i)|1 or neg(var(i)). To get the false-literal, use var(i).

8. Common 2-SAT Modeling Patterns

Let a, b be already-encoded literal nodes (either true- or false-literal of any variable).

OR clause: a ∨ b

addOr(a, b);

If ¬a (a is false), b must be true. If ¬b, a must be true. Both directions encoded.

Implication: a ⟹ b

addImp(a, b);
// same as: addOr(neg(a), b)

One directed edge. If a is true, b must also be true.

Forced assignment

forceTrue(a);    // ¬a → a
forceFalse(a);   // a → ¬a

XOR: exactly one true

addXor(a, b);
// addOr(a,b) + addOr(neg(a),neg(b))

Equality: same truth value

addEqual(a, b);
// a→b, b→a, ¬a→¬b, ¬b→¬a
💡

All roads lead to implications. A clause (a ∨ b) simply says: if you choose not-a, you must choose b.

9. Weighted / Max-SAT Notes

✅ Standard 2-SAT

Polynomial (O(N+M)), SCC-based. Covered by this tutorial.

✗ General Max-SAT

NP-hard in general. SCC alone does not solve it.

⚡ Contest Combos

Binary search + 2-SAT feasibility; min-cut for weighted variants; monotone reductions.

📌

ICPC advice: Use 2-SAT as a feasibility oracle inside a larger optimization, not as a standalone optimizer for weighted clauses.

10.1 POJ 1236 — Network of Schools  ↗ VJudge

A pure SCC condensation problem. Build the condensation DAG and count sources (in-degree 0) and sinks (out-degree 0).

1

Min schools to reach all: = number of source SCCs (in-degree 0)

2

Min edges to make fully connected: = max(sources, sinks) — or 0 if already one SCC

Why max(sources, sinks)? proof sketch

Each added edge can reduce at most one source deficiency and one sink deficiency. So you need at least max(sources, sinks) edges. This bound is achievable by pairing source SCCs with sink SCCs and adding a chain of edges.

        addOr(lit(a,s1),lit(b,s2));
    }
    memset(dfsT,-1,nodes*sizeof(dfsT[0]));
    memset(cmpId,-1,nodes*sizeof(cmpId[0]));
    curT=sz=cmpCnt=0;
    for(int i=0;i<nodes;i++) if(dfsT[i]==-1) tarjanDfs(i);
    vector<char> ans(n);
    for(int i=0;i<n;i++){
        if(cmpId[2*i]==cmpId[2*i+1]){cout<<"IMPOSSIBLE\n";return 0;}
        ans[i]=(cmpId[2*i]<cmpId[2*i+1]?'+':'-');
    }
    for(int i=0;i<n;i++){if(i)cout<<' ';cout<<ans[i];}cout<<'\n';
}

10.4 POJ 3648 — Wedding  ↗ VJudge

Each couple gets a boolean: husband/wife on bride's side. Pair constraints become clauses. Couple 0's wife is forced to the bride's side.

💍

Key trick: addOr(w(0), w(0)) is equivalent to forceTrue(w(0)) — encoding a forced literal as a degenerate OR clause.

AC-Ready Code click to expand 
#include <bits/stdc++.h>
using namespace std;
#define adj(head,u,e,v) for(int e=head[u],v;~e&&((v=to[e]),1);e=nxt[e])
const int MAXV=5000,MAXE=200000;
int head[MAXV],to[MAXE],nxt[MAXE],ne;
int dfsT[MAXV],low[MAXV],cmpId[MAXV],stk[MAXV],sz,curT,cmpCnt;
int nVars,nodes;
inline int h(int i){return 2*(2*i);}
inline int w(int i){return h(i)^1;}
void initGraph(int n2){nodes=n2;ne=0;memset(head,-1,nodes*sizeof(head[0]));}
void addEdge(int u,int v){to[ne]=v;nxt[ne]=head[u];head[u]=ne++;}
void addOr(int a,int b){addEdge(a^1,b);addEdge(b^1,a);}
void addXor(int a,int b){addOr(a,b);addOr(a^1,b^1);}
void tarjanDfs(int u){
    dfsT[u]=low[u]=curT++;stk[sz++]=u;
    adj(head,u,e,v){
        if(dfsT[v]==-1){tarjanDfs(v);low[u]=min(low[u],low[v]);}
        else if(cmpId[v]==-1){low[u]=min(low[u],low[v]);}
    }
    if(low[u]==dfsT[u]){do{cmpId[stk[--sz]]=cmpCnt;}while(stk[sz]!=u);cmpCnt++;}
}
int main(){
    int n,m;
    while(scanf("%d%d",&n,&m),n||m){
        nVars=n; initGraph(4*n);
        for(int i=0;i<n;i++) addXor(h(i),w(i));
        addOr(w(0),w(0));
        while(m--){
            int a,b;char sa,sb;
            scanf("%d%c %d%c",&a,&sa,&b,&sb);
            int x=(sa=='h'?h(a):w(a));
            int y=(sb=='h'?h(b):w(b));
            addOr(x,y);
        }
        memset(dfsT,-1,nodes*sizeof(dfsT[0]));
        memset(cmpId,-1,nodes*sizeof(cmpId[0]));
        curT=sz=cmpCnt=0;
        for(int i=0;i<nodes;i++) if(dfsT[i]==-1) tarjanDfs(i);
        bool bad=false;
        for(int i=0;i<nodes;i+=2) if(cmpId[i]==cmpId[i^1]){bad=true;break;}
        if(bad){puts("bad luck");continue;}
        for(int i=1;i<n;i++){
            if(i>1) printf(" ");
            bool hSide=(cmpId[h(i)]<cmpId[h(i)^1]);
            printf("%d%c",i,hSide?'h':'w');
        }
        puts("");
    }
}

10.5 UVA 10319 — Manhattan  ↗ VJudge

Each street direction is a boolean. Route constraints encode as 2-SAT clauses. When row and column both differ, two possible move orders exist — at least one must work, giving an OR constraint.

#include <bits/stdc++.h>
using namespace std;
#define adj(head,u,e,v) for(int e=head[u],v;~e&&((v=to[e]),1);e=nxt[e])
const int N=4005,M=4000005;
int head[N],to[M],nxt[M],ne;
int dfsT[N],low[N],cmpId[N],stk[N],sz,curT,cmpCnt,n;
void initGraph(){ne=0;memset(head,-1,n*sizeof(head[0]));}
void addEdge(int u,int v){to[ne]=v;nxt[ne]=head[u];head[u]=ne++;}
void addOr(int a,int b){addEdge(a^1,b);addEdge(b^1,a);}
void addSoP(int a,int b,int x,int y){addOr(a,x);addOr(a,y);addOr(b,x);addOr(b,y);}
void tarjanDfs(int u){
    dfsT[u]=low[u]=curT++;stk[sz++]=u;
    adj(head,u,e,v){
        if(dfsT[v]==-1){tarjanDfs(v);low[u]=min(low[u],low[v]);}
        else if(cmpId[v]==-1){low[u]=min(low[u],low[v]);}
    }

}
int main(){
    int T;scanf("%d",&T);
    while(T--){
        int r,c,m;scanf("%d%d%d",&r,&c,&m);
        n=2*(r+c);initGraph();
        while(m--){
            int r1,c1,r2,c2;scanf("%d%d%d%d",&r1,&c1,&r2,&c2);
            --r1;--c1;--r2;--c2;
            if(r1==r2&&c1==c2) continue;
            if(r1==r2){int v=2*r1;if(c2<c1)v^=1;addOr(v,v);}
            else if(c1==c2){int v=2*(r+c1);if(r2<r1)v^=1;addOr(v,v);}
            else{
                bool dr=(c2<c1),dc=(r2<r1);
                int R1=2*r1+dr,R2=2*r2+dr;
                int C1=2*(r+c1)+dc,C2=2*(r+c2)+dc;
                addSoP(R1,C2,R2,C1);
            }
        }
        memset(dfsT,-1,n*sizeof(dfsT[0]));
        memset(cmpId,-1,n*sizeof(cmpId[0]));

10.6 UVA 1086 — The Ministers' Major Mess  ↗ VJudge

Each bill is a boolean. Each minister requires more than half their opinions to hold. For ministers with k ≤ 2 opinions: all opinions are forced. For k = 3 or 4: if any opinion fails, all others must hold — giving "¬vote[p] → vote[q] for all q ≠ p" implications.

⚡

After solving, to determine if each variable is definitely true/false/unknown, test forcing it both ways and checking satisfiability each time.

AC-Ready Code click to expand 
#include <bits/stdc++.h>
using namespace std;
const int MAXN=262144;
struct TwoSAT{
    int n;
    vector<int> g[MAXN];
    int vfind[MAXN],findIdx,stk[MAXN],stkIdx,inStk[MAXN],visited[MAXN],contract[MAXN],sccCnt;
    int scc(int u){
        inStk[u]=visited[u]=1;stk[++stkIdx]=u;vfind[u]=++findIdx;
        int mn=vfind[u];
        for(int v:g[u]){if(!visited[v])mn=min(mn,scc(v));if(inStk[v])mn=min(mn,vfind[v]);}
        if(mn==vfind[u]){do{inStk[stk[stkIdx]]=0;contract[stk[stkIdx]]=u;}while(stk[stkIdx--]!=u);sccCnt++;}
        return mn;
    }
    void init(int _n){n=_n;for(int i=0;i<n;i++)g[i].clear();}
    void addEdge(int u,int v){g[u].push_back(v);}
    bool solvable(){
        memset(visited,0,n*sizeof(visited[0]));memset(inStk,0,n*sizeof(inStk[0]));sccCnt=0;
        for(int i=0;i<n;i++){if(visited[i])continue;stkIdx=0;findIdx=0;scc(i);}
        for(int i=0;i<n;i+=2) if(contract[i]==contract[i^1])return false;
        return true;
    }
    void computeChoice(){
        if(!solvable())return;
        for(int i=0;i<n;i+=2){
            int val=0;
            g[i].push_back(i^1);if(solvable())val|=1;g[i].pop_back();
            g[i^1].push_back(i);if(solvable())val|=2;g[i^1].pop_back();
            printf("%c","-yn?"[val]);
        }puts("");
    }
}sat;
int main(){
    int n,m,tc=0;char s[20];
    while(scanf("%d%d",&n,&m)==2&&n){
        sat.init(2*n);
        for(int i=0;i<m;i++){
            int k;scanf("%d",&k);
            vector<int> vote(k);
            for(int j=0;j<k;j++){int x;scanf("%d%s",&x,s);--x;vote[j]=2*x+(s[0]=='y');}
            if(k<=2){for(int x:vote)sat.addEdge(x^1,x);}
            else{for(int p=0;p<k;p++)for(int q=0;q<k;q++)if(p!=q)sat.addEdge(vote[p]^1,vote[q]);}
        }
        printf("Case %d: ",++tc);
        if(!sat.solvable())puts("impossible");
        else sat.computeChoice();
    }
}

Knowledge Summary

✅ Core Checklist

  • SCC = maximal mutually reachable set
  • Condensation graph is always a DAG
  • Tarjan: one DFS, three arrays (dfsT, low, cmpId)
  • low[u]==dfsT[u] ⟹ u is an SCC root
  • cmpId[v]==-1 replaces inStack[v]
  • Kosaraju: post-order + reversed graph
  • 2-SAT: (a∨b) → ¬a→b, ¬b→a
  • Conflict ⟺ x and ¬x in same SCC
  • Value: x true iff cmpId[x] < cmpId[¬x]

⚠ Common Mistakes

  • Updating low from a cross-edge to a finished SCC
  • Using dfsT instead of low when checking SCC root
  • Wrong assignment direction — check cmpId numbering
  • Checking satisfiability after extracting assignment
  • Stack overflow with large N and deep recursion
  • Not popping the SCC root itself from the stack
  • Forgetting to init cmpId to -1 before each run

🔑 Key Theorems to Memorize

  • Tarjan and Kosaraju both run in O(N+M)
  • Condensation is always a DAG — no exceptions
  • 2-SAT satisfiability is linear-time decidable
  • Min broadcasts = #source SCCs in condensation
  • Min edges to make SCC = max(sources, sinks)

📚 Problem Taxonomy

  • Source/sink counting on DAG → POJ 1236
  • Sink SCC extraction → SPOJ CAPCITY
  • Direct 2-SAT → CSES 1684 Giant Pizza
  • Implication modeling → POJ 3648 Wedding
  • Grid direction 2-SAT → UVA 10319
  • Forced assignment detection → UVA 1086
